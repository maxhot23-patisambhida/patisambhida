# ปฏิสัมภิทามรรค — ห้องสมุดธรรมดิจิทัล

เว็บอ่านและค้นคว้าปฏิสัมภิทามรรคแบบ static จาก PDF ทั้ง 8 เล่ม (ไม่ต้องมี backend)

## สร้างข้อมูลใหม่

ใช้ Python ที่มี `pypdf`:

```powershell
python scripts\build_content.py
```

ผลลัพธ์จะอยู่ที่ `web\data\catalog.json` และ `web\data\book-XX.json`

## เปิดเว็บ

```powershell
python -m http.server 4173 --bind 127.0.0.1 --directory web
```

แล้วเปิด `http://127.0.0.1:4173/`

## ความสามารถ

- **ห้องสมุด** — hero ด้านบน (ชื่อคัมภีร์ · ห้องสมุดธรรมดิจิทัล · พุทธสุภาษิต · ปุ่มเริ่มอ่าน/อ่านต่อ · สถิติจำนวนเล่ม/หน้า/ตัวอักษร) มี **ลายน้ำภาพพระธาตุ** จางๆ หลัง hero (ไฟล์ `web/assets/hero-stupa.png` — CSS ครอป/เบลอ/ลดความทึบให้เอง; ถ้าไม่มีไฟล์จะเป็นกระดาษงาช้างล้วน) แล้วตามด้วยแดชบอร์ดทั้ง 8 เล่ม แถบความคืบหน้ารายเล่ม, "อ่านต่อจากครั้งก่อน", หัวข้อที่ดูล่าสุด
- **หน้าอ่าน** — ตัวอักษร serif ไทยอ่านสบาย เปลี่ยนหน้าได้ด้วยปุ่ม / ช่องเลขหน้า / slider / ลูกศรซ้าย-ขวา
- **สารบัญอัตโนมัติ** — ปุ่มสารบัญในหน้าอ่าน เปิด drawer แสดงหัวข้อ 2 ชั้น (หัวข้อหลักตามโครงสร้างคัมภีร์ / ทุกหน้า) พร้อมช่องกรองและป้ายประเภท (สารบัญ, มาติกา, นิทเทส, ภาณวาระ, วาระ, ข้อบาลี, ข้อธรรม)
- **อ่านตามหัวข้อ** — รวมข้อความช่วงหน้าของหัวข้อเดียวกันไว้ในหน้าเดียว พร้อม pill เลขหน้าลิงก์กลับโหมดรายหน้า และ citation แบบช่วงหน้า (หัวข้อยาวเกิน 30 หน้าแสดงบางส่วนพร้อมลิงก์อ่านต่อ)
- **ดัชนีธรรม** — คำหลัก 55 คำใน 6 หมวด นับจำนวนหน้าที่พบแยกตามเล่ม + snippet ตัวอย่าง (สแกนครั้งแรกครั้งเดียว แล้ว cache ใน localStorage) กดคำเพื่อค้นทันที
- **ค้นหาแบบ command palette** — กด `Ctrl+K` ค้นทั้ง 8 เล่ม, scope ทั้งหมด/เล่มนี้/ที่คั่นไว้, กรองรายเล่ม, ไฮไลต์คำค้น, คำค้นล่าสุด, คัดลอก citation จากผลค้นหา
- **ค้นหาแบบ normalized** — เลขไทย=อารบิก, "กํา"="กำ", ช่องว่างแทรกจาก PDF ("ท า"="ทำ", "นั ้น"="นั้น") โดยยังไฮไลต์ตำแหน่งในข้อความต้นฉบับได้ถูกต้อง
- **เครื่องมืออ่าน** — ปรับขนาดตัวอักษร, โหมดกลางคืน, focus mode (`Esc` ออก), คัดลอกการอ้างอิง
- **Citation มาตรฐาน** — `ปฏิสัมภิทามรรค เล่ม X, "ชื่อเล่ม", หน้า Y, ไฟล์ PDF: ..., ลิงก์: ...` (หัวข้อใช้ หน้า Y-Z) มีทุกที่: หน้าอ่าน หัวข้อ ที่คั่นหน้า ผลค้นหา
- **ที่คั่นหน้า / ความคืบหน้า / คำค้นและหัวข้อล่าสุด** — เก็บใน localStorage ทั้งหมด
- รองรับมือถือเต็มรูปแบบ (bottom nav + pager ลอย)

## โครงสร้าง URL

- `#/` ห้องสมุด
- `#/book/{slug}/{page}` หน้าอ่าน เช่น `#/book/book-01/12`
- `#/section/{slug}/{index}` อ่านตามหัวข้อ เช่น `#/section/book-01/2`
- `#/bookmarks` ที่คั่นหน้า
- `#/map` ดัชนีธรรม

ลิงก์รูปแบบเก่า `?book=&page=` ยัง redirect มารูปแบบใหม่ให้อัตโนมัติ

## โครงสร้างโค้ด

- `app.js` — state, routing, views, palette, localStorage
- `knowledge.js` — ชั้นความรู้: normalize/ค้นหา, สกัดสารบัญ (`buildToc`), แบ่งหัวข้อ (`buildSections`), ดัชนีธรรม (`buildTermIndex`), diagnostics (`diagnoseBook`)
- `overrides/toc-overrides.json` — แก้สารบัญทับ heuristic (add/remove รายเล่ม)
- `overrides/term-overrides.json` — aliases และคำเพิ่มของดัชนีธรรม
- `overrides/editorial-overrides.json` — ชั้นแก้ไขข้อความของแอดมิน (bold/italic/color/replace/heading/image-block/layout) แสดงต่อผู้อ่านทุกคน
- `editorial-images/` — ไฟล์รูปภาพประกอบที่อ้างจาก image-block (ดู README ในโฟลเดอร์)

## เครื่องมือตรวจทาน (สำหรับผู้ดูแล)

- **ตรวจสารบัญ** — ใน TOC drawer เลือกแท็บ "ตรวจสารบัญ": เห็น diagnostics ของเล่ม (จำนวน headings/หัวข้อหลัก/การกระจายประเภท), หัวข้อทุกตัวพร้อมช่วงหน้า ความเชื่อมั่น และธงเตือนหัวข้อยาวเกิน 30 หน้า กด "ตัดออก" เพื่อทดลองตัดแบบเห็นผลทันที แล้ว "คัดลอก override JSON" ไปวางใน `overrides/toc-overrides.json`
- **วิธี override สารบัญ** — `{"book-05": {"add": [{"page": 100, "title": "ชื่อหัวข้อ", "type": "นิทเทส"}], "remove": [{"page": 1, "title": "คำขึ้นต้น"}]}}` — `add` ใช้แบ่งหัวข้อที่ยาวเกินได้ทันที, `remove` จับคู่ด้วย page + title ขึ้นต้น
- **ดัชนีธรรม exact/broad** — สลับ "แบบกว้าง" (substring, "มรรค" นับรวม "อริยมรรค") กับ "เฉพาะคำโดด" (ตรวจขอบเขตคำ) ได้ที่หัวหน้าดัชนี
- **`#/quality`** — รายงานคุณภาพข้อมูลต่อเล่ม: คุณภาพ TOC, หัวข้อยาวผิดปกติ, หน้าว่าง/สั้น, ช่องว่างแทรกกลางคำที่เหลือ, คำดัชนีที่กว้างเกินไป (ลิงก์จากท้ายหน้าดัชนีธรรม)

## ข้อจำกัดของ extraction ที่ควรปรับมือภายหลัง

- `headings` ใน pageData คือ "บรรทัดเน้น" จาก PDF — กติกาแยกหัวข้อหลักเป็น heuristic ใน `MAJOR_RULES` (knowledge.js) อาจหลุด/เกินได้ ปรับ regex เพิ่มได้ตรงนั้น
- เล่ม 5: หัวข้อ `[๖๖]` ที่เคยยาว ~170 หน้า ถูกแบ่งแล้วเป็น 22 หัวข้อย่อยตามหมวดธรรม (ขันธ์/อายตนะ/อาการ ๓๒/ธาตุ/อินทรีย์/ปฏิจจสมุปบาท/สัจจะ/กิจญาณ/อาทีนวะ/อนุปัสสนา/สันติบทญาณ) ผ่าน `overrides/toc-overrides.json` — ใช้เป็นตัวอย่างการ override เล่มอื่นได้
- เล่ม 6: หัวข้อ `[เทียบข้อ ๓๙]` ที่ยาว ๗๙ หน้า แบ่งเป็น 5 ช่วงตามลำดับอรรถ "เอกัตเต…อัฏฐัง" (๑–๔๒), หัวข้อ `[ข้อ ๓๐/๓๖/๓๘]` แบ่งครึ่งตามเลขอรรถ, และช่วง `[ข้อ ๓๗]` ใช้ `remove` ตัด noise major (ประโยคจบด้วย "นิทเทส/ภาณวาระ" ที่ heuristic จับผิด หน้า 137–148) ผลลัพธ์: หัวข้อยาวสุดเหลือ 21 หน้า
- ขอบเขตหัวข้อใช้กติกา conservative: เริ่มที่หน้าที่พบหัวข้อหลัก จบก่อนหัวข้อหลักถัดไป — หัวข้อที่เริ่มกลางหน้าจะรวมเนื้อความท้ายหัวข้อก่อนหน้าด้วย
- ดัชนีธรรมนับ "หน้าที่พบคำ" แบบ substring (เช่น "มรรค" นับรวม "อริยมรรค") ยังไม่แยกบริบท
## Notes and Highlights (ผู้อ่าน)

- Select text in the page reader, then choose highlight or note.
- Personal annotations are stored in localStorage under `psm.annotations`.
- The `#/notes` page lists all highlights and notes, grouped by book.
- Notes can be exported/imported as JSON for backup or moving between browsers.
- This is an annotation layer only; it does not change `web/data` or the PDF-derived source text.
- โน้ต/ไฮไลท์เป็น **ส่วนตัวของผู้อ่านแต่ละคน** ไม่แชร์ และไม่กระทบข้อความที่ผู้อ่านคนอื่นเห็น
- **สลับไฮไลท์แบบ ebook** — ลากเลือกข้อความที่ยังไม่ไฮไลท์ → ปุ่ม **🖍 ไฮไลท์**; ลากเลือกข้อความที่ไฮไลท์แล้ว → ปุ่ม **🗑 เอาไฮไลท์ออก** (คลิกเดียว; ถ้าช่วงนั้นมีโน้ตด้วยจะถามยืนยัน)
- ในหน้า `#/notes` มีปุ่มลบรายตัว, ปุ่ม **ลบไฮไลท์ทั้งหมด** (ลบเฉพาะไฮไลท์ที่ไม่มีโน้ต — โน้ตยังอยู่ครบ) และ **สำรองโน้ต / กู้คืนโน้ต** (ดาวน์โหลด/อ่านไฟล์ JSON — รูปแบบเดิม ไม่เปลี่ยน เป็นภาษาผู้อ่านแทน "ส่งออก/นำเข้า JSON")

## โหมดแอดมิน — แก้ไขข้อความ (Editorial, Phase 1)

ระบบแก้ไขข้อความระดับ "ชั้นซ้อน" (annotation layer) สำหรับเจ้าของเว็บคนเดียว — ไม่แตะ `web/data`, ไม่มี backend

- **เข้าโหมด** — กด `Ctrl+Shift+E` กรอกรหัสผ่าน (ตั้งใน `app.js` → `ADMIN_PASSCODE`, ค่าตั้งต้น `patisambhida-2569`) เก็บสถานะใน `localStorage.psm_admin` จนกว่าจะ "ออกจากโหมดแอดมิน"
- **รหัสผ่านเป็น "ประตูเบา"** — เว็บ static เปิดเผยซอร์สได้ จึงกันผู้อ่านทั่วไปเท่านั้น ไม่ใช่ระบบความปลอดภัยจริง
- **สร้างการแก้ไข** — ลากเลือกข้อความในหน้าอ่าน ป๊อปอัปจะเพิ่ม **ตัวหนา / ตัวเอียง / แก้ข้อความ**, แถบ **สี preset คลิกเดียว** (📘 คำสำคัญ · 🟣 คำบาลี · 🟢 คำอธิบาย · 🔴 ตรวจสอบ) และแถบ **หัวข้อ** (🔠 หัวข้อใหญ่ `heading-lg` 1.8em · 🔡 หัวข้อกลาง `heading-md` 1.4em · ▫️ หัวข้อเล็ก `heading-sm` 1.2em — ขนาด preset เท่านั้น ไม่มีขนาดอิสระ) — ผู้อ่านทั่วไปเห็นแค่ ไฮไลท์/เพิ่มโน้ต
- **อักษรไทย** — ตัวหนาใช้ `font-weight:700` + `-webkit-text-stroke` ให้ชัด, ตัวเอียงใช้ `font-style: oblique 12deg` (อักษรไทยไม่มีหน้าตัวเอียง จึงบังคับเอียงสังเคราะห์)
- **ป๊อปอัปเครื่องมือ** — ลากย้ายได้ที่แถบหัว, ปรับขนาดที่มุมขวาล่าง, ปุ่มย่อ (–) → ยุบเป็นปุ่มลอย ✎ คลิกเพื่อกางใหม่ — ตำแหน่ง/ขนาด/สถานะย่อ จำใน `localStorage.psm.edPopup` และคงอยู่ในจอเสมอ (clamp ขอบจอ); โน้ต/แก้ข้อความ ซ่อนไว้ก่อน กดปุ่มจึงกาง (compact mode) ลดการบังเนื้อหา
- **เก็บแบบเดียวกับ TOC override** — ไฟล์ `overrides/editorial-overrides.json` = baseline ที่ผู้อ่านทุกคนเห็น, แอดมินแก้ใน `localStorage.psm.editorial` (overlay: entries + tombstone) แล้วกด **ส่งออก Editorial** เอา JSON ไปวางในไฟล์
- **offset-based** — ใช้ start/end อิงข้อความต้นฉบับของหน้า (สถาปัตยกรรมเดียวกับระบบไฮไลท์)
- **แถบเครื่องมือแอดมิน** (มุมซ้ายล่าง) — บรรทัดบนสรุปสถานะ `แก้ไข Editorial: X · ส่งออกล่าสุด <วันที่ พ.ศ.>` (เก็บเวลาใน `localStorage.psm.edLastExport`), บรรทัดล่าง: รายการแก้ไข / ส่งออก Editorial / นำเข้า Editorial / ออกจากโหมด — ผู้อ่านทั่วไปไม่เห็นแถบนี้เลย (แสดงเฉพาะหลังเข้าโหมดแอดมินสำเร็จ)
- **ส่งออก Editorial** — ดาวน์โหลดไฟล์ JSON **และ** คัดลอกขึ้นคลิปบอร์ดอัตโนมัติ พร้อม toast "คัดลอก Editorial JSON แล้ว" + คำใบ้ "วางลงใน overrides/editorial-overrides.json แล้ว deploy"
- **แผงรายการแก้ไข** — ดูทุก entry แยกตามเล่ม พร้อมป้ายความหมาย `BOLD / ITALIC / IMPORTANT / PALI / COMMENTARY / REVIEW / REPLACE` (สี preset map กลับเป็นป้ายอัตโนมัติ) และ แก้ไข (replace = แก้ข้อความ, color = เปลี่ยนสี) / ลบ / กระโดดไปหน้านั้น พร้อม **สถิติ** จำนวนแต่ละชนิด + รวมทั้งหมด
- **ย้อนกลับได้ทุกขั้น** — `Ctrl+Z` ย้อน, `Ctrl+Shift+Z` ทำซ้ำ (ครอบคลุม ตัวหนา/เอียง/สี/แก้ข้อความ/ลบ/ล้างหน้า/นำเข้า) เก็บประวัติในหน่วยความจำ (ไม่ persist ข้าม reload)
- **ลบรูปแบบเร็ว** — ลากเลือกข้อความที่มี editorial อยู่ ป๊อปอัปจะมีปุ่ม **🗑 ลบรูปแบบ** ลบ entry ที่ทับช่วงนั้นทันที (ไม่ต้องเปิดแผง)
- **ล้างหน้านี้** — ปุ่มในแผงลบ editorial ทั้งหน้าปัจจุบันในคลิกเดียว
- **กันพลาด** — ลบ `replace` และ "ล้างหน้านี้" มี dialog ยืนยันก่อน; ทุกการลบย้อนด้วย `Ctrl+Z` ได้
- **แสดงต่อผู้อ่านทุกคน** — entries จากไฟล์ render ทั้งในหน้าอ่านและโหมดอ่านตามหัวข้อ ขณะที่โน้ต/ไฮไลท์ส่วนตัวยังทำงานแยกตามเดิม
- โครงสร้าง entry: `{ "id", "type": "bold|italic|color|replace|heading-*|image-block", "page", "start", "end", "color"? , "replacement"?, "image"?, "caption"? }`
- **ขอบเขต Phase 1** — ยังไม่มี footnote, cross-reference, multi-admin, review/approval workflow, cloud sync

### Phase 2 — รูปภาพประกอบ (image-block)

แทรกแผนภาพ/รูปประกอบ (เช่น ขันธ์ ๕, อริยสัจ ๔, มรรค ๘, ปฏิจจสมุปบาท) คั่นกลางเนื้อหา เป็นชั้น editorial เหมือน heading/replace — ไม่แตะ `web/data`, ไม่มี CMS/อัปโหลด/drag-drop

- **แทรกรูป** — ลากเลือกตำแหน่งในข้อความ → ปุ่ม **🖼 แทรกรูปภาพ** → ไดอะล็อกกรอก **ที่อยู่รูป (URL / path)** + **คำบรรยายใต้ภาพ** พร้อมพรีวิวสด — รูปผูกกับจุดเริ่มของช่วงที่เลือก (`start == end`)
- **ที่อยู่รูป** — รองรับ URL ภายนอก (`https://…`), path ในเว็บ (`/editorial-images/khandha-5.webp`) และ `data:image/*` — กรองทิ้ง scheme อันตราย (`javascript:` ฯลฯ); วางไฟล์ภาพไว้ที่ `web/editorial-images/` (มี `khandha-5.svg` เป็นตัวอย่าง/แม่แบบ — SVG ต้องมี `width`/`height` ไม่งั้นยุบเป็น 0)
- **การแสดงผล** — `<figure>` กลางคอลัมน์ ขอบมน เงาบาง คำบรรยายใต้ภาพ ขนาด responsive (`max-width:100%`); ผู้อ่านทุกคนเห็น ไม่มีปุ่มแอดมิน (โหมดแอดมินมีกรอบ dashed บางๆ ให้รู้ว่าเป็นชั้น editorial)
- **แผง/สถิติ** — แสดงป้าย **IMAGE** + thumbnail, แก้ไข (เปิดไดอะล็อกเดิมแบบ prefill) / ลบ / กระโดดไปหน้า; สถิติเพิ่มบรรทัด **ภาพประกอบ**
- **export/import + undo/redo** — image-block round-trip ผ่านไฟล์ได้ครบ (`image`/`caption`) และย้อน/ทำซ้ำผ่านระบบประวัติเดิม
- **โครงสร้าง** — `{ "id", "type": "image-block", "page", "start", "end": start, "image": "/editorial-images/…", "caption": "…" }`

### Phase 1.4 — จัดวาง/ระยะ (layout overrides)

กู้โครงสร้างหน้าจากต้นฉบับหนังสือ (จัดกึ่งกลาง/ชิดขวา/ย่อหน้า/ระยะ) เป็นชั้น editorial ระดับ block — preset เชิงความหมายเท่านั้น ไม่มีค่าตัวเลขอิสระ, ไม่มี contenteditable/rich text

- **ปุ่มในป๊อปอัป** (แถว `.ed-layouts`) — 📍 จัดกึ่งกลาง · ⬅ ชิดซ้าย · ➡ ชิดขวา · ⇥ ย่อหน้า · ⬆ ระยะบน · ⬇ ระยะล่าง — ลากเลือกข้อความแล้วคลิกเดียว (เหมือน heading/สี preset)
- **ชนิด + การแสดงผล** — `align-center` → `text-align:center;display:block` · `align-left/right` ทำนองเดียวกัน · `indent` → `text-indent:2em` · `spacing-top` → `margin-top:1.5em` · `spacing-bottom` → `margin-bottom:1.5em` (ทุกชนิดเป็น block; รวมกับ heading ได้ในบล็อกเดียว)
- **แผง/สถิติ** — ป้าย **CENTER / LEFT / RIGHT / INDENT / SPACE-TOP / SPACE-BOTTOM** + นับแยกชนิด; ไม่มีปุ่มแก้ไข (ไม่มีพารามิเตอร์) มีแค่ ลบ/กระโดดไปหน้า
- **export/import + undo/redo** — round-trip ผ่านไฟล์ได้ (เก็บแค่ id/type/page/start/end) และย้อน/ทำซ้ำผ่านระบบประวัติเดิม
- **โครงสร้าง** — `{ "id", "type": "align-center|align-left|align-right|indent|spacing-top|spacing-bottom", "page", "start", "end" }`
- **ข้อจำกัด** — เลือกช่วงที่สะอาด (ไม่ทับ bold/โน้ต) ต่อหนึ่งบล็อก เพราะ engine ซอยตามขอบ format เหมือน heading; ถ้าทับกันมากบล็อกอาจถูกแยกหลายชิ้น

### Cache busting

`index.html` ผูกเวอร์ชันกับ asset: `styles.css?v=editorial-v2`, `app.js?v=editorial-v2` (และ import `knowledge.js?v=editorial-v2` ใน app.js) — เมื่อ deploy โค้ดใหม่ให้ **bumpเลขเวอร์ชัน** (เช่น `editorial-v3`) ผู้ใช้จะได้ไฟล์ใหม่ทันทีโดยไม่ต้อง Ctrl+F5
