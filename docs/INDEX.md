## 1. ภาพรวมโครงการ (Project Overview)

โครงการ **Patisambhidamagga Knowledge Architecture (PKA)** มุ่งสร้าง **สำนักงานใหญ่ดิจิทัลขององค์กรองค์ความรู้พระธรรม (Digital Knowledge Headquarters)** ที่ทำให้องค์ความรู้จากปฏิสัมภิทามรรคและพระไตรปิฎก **ถูกต้อง ตรวจสอบได้ และเชื่อมโยงกัน** โดยมีมนุษย์เป็นผู้รับผิดชอบสูงสุดเสมอ

PKA **ไม่ใช่** "เว็บไซต์อ่านหนังสือ" และ **ไม่ใช่** "ระบบหลายตัวแทน (Multi-Agent System)" แต่เป็น **องค์กรองค์ความรู้ที่มีระเบียบ** ซึ่งจัดเอกสารเป็นชั้น ตั้งแต่วิสัยทัศน์ลงไปจนถึงองค์ความรู้จริง

```
   Vision           (วิสัยทัศน์ — PKA-VIS)
        ▼
   Standards        (มาตรฐาน — PKA-STD)
        ▼
   Architecture     (สถาปัตยกรรม — PKA-ARC)
        ▼
   Domain           (แบบจำลองโดเมน — PKA-DOM)
        ▼
   Reference        (แหล่งอ้างอิง — PKA-REF)
        ▼
   Operations       (ปฏิบัติการ/องค์กร — PKA-OPS)
        ▼
   Knowledge        (องค์ความรู้จริง — PKA-KNW)
        ▼
   การนำไปปฏิบัติ    (ชั้นปฏิบัติ — เฟสอนาคต)
```

> **อ่านก่อนเสมอ:** วิสัยทัศน์ [PKA-VIS-001](vision/PKA-VIS-001-วิสัยทัศน์สำนักงานใหญ่ดิจิทัล.md) และเอกสารแม่ [PKA-STD-001](standards/PKA-STD-001-ธรรมนูญโครงการ.md) เป็นรากฐานที่เอกสารอื่นทั้งหมดต้องไม่ขัดแย้ง

3. **เปิดเอกสารรายฉบับ** — จากทะเบียนในหัวข้อ 4
4. **เคารพธรรมาภิบาล** — ผู้ช่วยอัจฉริยะเป็นผู้ช่วย ไม่ใช่ผู้อนุมัติ และไม่ใช่แหล่งความจริง (ดูหัวข้อ 15)

## 15. สรุปธรรมาภิบาล (Governance Summary)

| หลักการ | สาระ |
|---|---|
| **มนุษย์เป็นผู้อนุมัติ (Human Approve)** | การอนุมัติองค์ความรู้เพื่อเผยแพร่เป็นอำนาจของมนุษย์เสมอ |
| **ผู้ช่วยอัจฉริยะเป็นผู้ช่วย (AI Assist)** | AI ช่วยค้นหา อธิบาย และเชื่อมโยง แต่ไม่สร้างหลักธรรมและไม่ตัดสินเนื้อหา |
| **AI ไม่ใช่แหล่งความจริง (AI is not Source of Truth)** | ผลผลิตของ AI อยู่ในลำดับชั้นต่ำสุด ต้องสืบย้อนไปยังแหล่งที่สูงกว่าได้เสมอ |
| **ตรวจสอบย้อนกลับได้ (Traceability)** | ทุกองค์ความรู้ต้องสืบย้อนถึงแหล่งข้อมูลต้นทางได้ |

> รายละเอียดธรรมาภิบาลอยู่ใน [PKA-STD-001](standards/PKA-STD-001-ธรรมนูญโครงการ.md), [PKA-OPS-001](operations/PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md) และ [PKA-OPS-005](operations/PKA-OPS-005-เมทริกซ์การมอบหมายขีดความสามารถ.md)

## 16. นโยบายการบำรุงรักษา (Maintenance Policy)

ดัชนีนี้เป็น **ประตูหลัก** จึงต้องคงความถูกต้องเสมอ — **ต้องอัปเดต INDEX.md ทุกครั้ง** เมื่อ

1. **เพิ่มเอกสาร** — เพิ่มในทะเบียนเอกสารหลัก (หัวข้อ 4) และสรุปตระกูล (หัวข้อ 3)
2. **ลบหรือปลดระวางเอกสาร** — ปรับสถานะและทะเบียนให้ตรง
3. **เปลี่ยน Version** — อัปเดตคอลัมน์ Version ในทะเบียน
4. **เปลี่ยน Status** — อัปเดตสถานะในทะเบียนและในหัวข้อ 8

> **ความสอดคล้อง:** เมื่อใดที่ INDEX.md ขัดกับ [Documentation Portal](README.md) หรือเอกสารต้นทาง ให้ยึดเอกสารต้นทางเป็นหลักและแก้ไขดัชนีให้ตรง — INDEX.md เป็น *ดัชนี* ไม่ใช่ *แหล่งความจริง*
