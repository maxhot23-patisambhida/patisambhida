## 1. วัตถุประสงค์

เอกสารฉบับนี้มีหน้าที่กำหนด **โครงสร้างขององค์ความรู้ (Knowledge Structure)** ที่ระบบของโครงการจะรองรับ กล่าวคือ เป็นการวางกรอบว่า องค์ความรู้พระธรรมจะถูกมองเป็น *อะไร* มี *ชนิดใดบ้าง* และเชื่อมโยงกันด้วย *ความสัมพันธ์แบบใด* ในเชิงโครงสร้าง

เป้าหมายของเอกสารคือสร้าง **กรอบความคิดร่วม** ที่ทำให้การวิเคราะห์เนื้อหาคัมภีร์ในขั้นต่อไปดำเนินไปบนพื้นฐานเดียวกัน โดยยังไม่ก้าวล่วงเข้าไปในเนื้อหาพระธรรม

เอกสารฉบับนี้ **ไม่ใช่**:

- การออกแบบฐานข้อมูลหรือโครงสร้างการจัดเก็บ
- แผนการพัฒนาหรือการ Implement ใด ๆ
- การวิเคราะห์หรืออธิบายเนื้อหาพระธรรม องค์ธรรม หรือหมวดธรรม

เอกสารฉบับนี้ **คือ**: การกำหนด *กรอบ* ของโดเมนองค์ความรู้ เพื่อให้เอกสารชุด Domain ฉบับถัด ๆ ไปนำไปเติมรายละเอียด

## 2. ขอบเขต

### 2.1 อยู่ในขอบเขต

- หลักการของแบบจำลองโดเมนองค์ความรู้
- คำจำกัดความของคำศัพท์เชิงโครงสร้างที่ใช้ร่วมกัน
- การจำแนก **ชนิดขององค์ความรู้** ในเชิงโครงสร้าง (โดยยังไม่ระบุเนื้อหาจริง)
- การจำแนก **ชนิดของความสัมพันธ์** ในเชิงโครงสร้าง
- การอ้างถึงระดับของข้อมูลและหลักการอ้างอิงตามเอกสารแม่

### 2.2 อยู่นอกขอบเขต

- เนื้อหาพระธรรม องค์ธรรม และหมวดธรรมทุกชนิด
- การออกแบบฐานข้อมูล โครงสร้างการจัดเก็บ หรือรูปแบบไฟล์
- รายละเอียดการพัฒนา เครื่องมือ หรือเทคโนโลยี
- การกำหนดรายการ (instance) ขององค์ความรู้หรือความสัมพันธ์จริง
- การตีความหรือการชี้ขาดทางหลักธรรม

---

## 3. หลักการของแบบจำลองโดเมน

แบบจำลองโดเมนของโครงการตั้งอยู่บนหลักการต่อไปนี้ ซึ่งทั้งหมดสืบเนื่องจากเอกสารแม่ PKA-STD-001

1. **แบบจำลองเป็นกรอบ ไม่ใช่เนื้อหา** — แบบจำลองโดเมนกำหนด *ภาชนะ* และ *ความสัมพันธ์ของภาชนะ* เท่านั้น ส่วนเนื้อหาพระธรรมที่จะบรรจุลงไปเป็นเรื่องของขั้นถัดไป
2. **เป็นกลางต่อเนื้อหา (Content-Neutral)** — กรอบต้องไม่ผูกติดกับองค์ธรรมใดองค์ธรรมหนึ่ง และต้องรองรับองค์ความรู้ได้หลากหลายชนิดอย่างเสมอภาค
3. **ทุกองค์ความรู้ต้องตรวจสอบย้อนกลับได้** — ทุกหน่วยในแบบจำลองต้องสามารถสืบกลับไปยัง **แหล่งข้อมูลหลัก (Primary Source)** ได้เสมอ
4. **แหล่งข้อมูลหลักเพียงแห่งเดียว** — องค์ความรู้แต่ละหน่วยมีต้นทางที่ถือเป็นต้นฉบับเพียงแห่งเดียว (Single Source of Truth)
5. **แยกองค์ความรู้ออกจากการแสดงผล** — โครงสร้างองค์ความรู้ต้องเป็นอิสระจากวิธีนำเสนอ
6. **ความสัมพันธ์ต้องมีหลักฐานรองรับ** — การเชื่อมโยงทุกเส้นต้องสืบที่มาได้ ไม่สร้างความสัมพันธ์ขึ้นโดยปราศจากหลักฐาน
7. **เปิดให้ขยายได้อย่างมีระเบียบ** — กรอบต้องรองรับการเพิ่มชนิดองค์ความรู้และชนิดความสัมพันธ์ในอนาคต โดยไม่ทำลายโครงสร้างเดิม
8. **ความเรียบง่ายมาก่อน** — เลือกกรอบที่เรียบง่ายที่สุดเท่าที่ยังครอบคลุมความต้องการได้

## 4. คำจำกัดความ

ในเอกสารฉบับนี้และเอกสารชุด Domain ที่จะตามมา ให้ใช้คำศัพท์ตามความหมายดังนี้

| คำศัพท์ (ไทย) | คำอังกฤษ | ความหมาย |
|---|---|---|
| **องค์ความรู้** | Knowledge | สิ่งที่ระบบรับรู้และจัดการในฐานะหน่วยของความรู้ |
| **หน่วยองค์ความรู้** | Knowledge Unit | หน่วยย่อยที่สุดขององค์ความรู้ที่ระบบจัดการเป็นเอกเทศได้หนึ่งหน่วย |
| **ความสัมพันธ์** | Relationship | การเชื่อมโยงระหว่างหน่วยองค์ความรู้ตั้งแต่สองหน่วยขึ้นไป |
| **ข้อมูลกำกับ** | Metadata | ข้อมูลที่อธิบายหน่วยองค์ความรู้หรือความสัมพันธ์ เช่น ที่มา ระดับ สถานะ |
| **ระดับความน่าเชื่อถือ** | Authority Level | ตำแหน่งของหน่วยองค์ความรู้ในลำดับชั้นแหล่งข้อมูลตามเอกสารแม่ |
| **แหล่งข้อมูลหลัก** | Primary Source | ต้นทางสูงสุดที่ถือเป็นต้นฉบับขององค์ความรู้ |
| **สายการสืบทอดข้อมูล** | Data Lineage | เส้นทางที่องค์ความรู้เดินทางจากแหล่งข้อมูลหลักจนถึงรูปแบบที่นำเสนอ |
| **แบบจำลองโดเมน** | Domain Model | กรอบเชิงโครงสร้างที่อธิบายชนิดขององค์ความรู้และความสัมพันธ์ |
| **ชนิดขององค์ความรู้** | Knowledge Type | ประเภทเชิงโครงสร้างของหน่วยองค์ความรู้ |
| **ชนิดของความสัมพันธ์** | Relationship Type | ประเภทเชิงโครงสร้างของการเชื่อมโยง |

> หมายเหตุ: คำจำกัดความข้างต้นเป็นเชิงโครงสร้าง ไม่ใช่นิยามทางพระธรรม

## 5. ชนิดขององค์ความรู้ที่ระบบรองรับ

ส่วนนี้กำหนด **ประเภทเชิงโครงสร้าง** ของหน่วยองค์ความรู้ที่ระบบจะรองรับ โดย *ยังไม่ระบุ* ว่าเนื้อหาจริงในแต่ละประเภทคืออะไร การจัดประเภทนี้เป็นเพียงการวางช่องว่างให้เนื้อหามาบรรจุในภายหลัง

| รหัสชนิด | ชนิดขององค์ความรู้ (เชิงโครงสร้าง) | คำอธิบายกรอบ |
|---|---|---|
| K-1 | **หน่วยเนื้อหาต้นทาง** (Source Unit) | หน่วยที่บรรจุเนื้อหาจากแหล่งข้อมูลหลักโดยตรง |
| K-2 | **หน่วยคำอธิบาย** (Explanatory Unit) | หน่วยที่ขยายความหน่วยอื่น โดยต้องระบุที่มาและระดับ |
| K-3 | **หน่วยโครงสร้าง/หมวดหมู่** (Structural Unit) | หน่วยที่ทำหน้าที่จัดกลุ่มหรือลำดับหน่วยอื่น |
| K-4 | **หน่วยคำศัพท์/นิยาม** (Term Unit) | หน่วยที่นิยามคำศัพท์ที่ใช้อ้างถึงในระบบ |
| K-5 | **หน่วยอ้างอิง** (Reference Unit) | หน่วยที่ชี้ไปยังแหล่งข้อมูลหลักหรือหน่วยอื่นเพื่อการสืบย้อน |

หลักที่ใช้กับทุกชนิด:

- ทุกหน่วยต้องมี **ข้อมูลกำกับ** อย่างน้อยคือ ที่มา ระดับความน่าเชื่อถือ และสถานะ
- รายการประเภทข้างต้นเป็น *ฉบับร่าง* และอาจปรับปรุงในเอกสาร Domain ฉบับถัดไป
- เอกสารฉบับนี้ **ไม่** ระบุว่าเนื้อหาใดเป็นของชนิดใด — เป็นเพียงการวางกรอบประเภท

## 6. ชนิดของความสัมพันธ์ที่ระบบรองรับ

ส่วนนี้กำหนด **ประเภทเชิงโครงสร้าง** ของความสัมพันธ์ระหว่างหน่วยองค์ความรู้ โดยยังไม่ระบุความสัมพันธ์จริงระหว่างเนื้อหาใด ๆ

| รหัสชนิด | ชนิดของความสัมพันธ์ (เชิงโครงสร้าง) | คำอธิบายกรอบ |
|---|---|---|
| R-1 | **ความเป็นลำดับชั้น** (Hierarchy) | หน่วยหนึ่งบรรจุหรืออยู่ใต้อีกหน่วยหนึ่ง |
| R-2 | **การอ้างอิง** (Reference) | หน่วยหนึ่งชี้ไปยังอีกหน่วยหนึ่งเพื่อการสืบย้อน |
| R-3 | **การขยายความ** (Elaboration) | หน่วยหนึ่งอธิบายหรือขยายความอีกหน่วยหนึ่ง |
| R-4 | **การเชื่อมโยง/เทียบเคียง** (Association) | หน่วยสองหน่วยมีความเกี่ยวข้องกันในเชิงความหมาย |
| R-5 | **สายการสืบทอด/ที่มา** (Provenance) | หน่วยหนึ่งสืบทอดหรือมีที่มาจากอีกหน่วยหนึ่ง |

หลักที่ใช้กับทุกชนิด:

- ความสัมพันธ์ทุกเส้นต้องมี **หลักฐานรองรับ** และต้องสืบที่มาได้
- ความสัมพันธ์อาจมี **ข้อมูลกำกับ** ของตัวเอง เช่น ที่มาและระดับความน่าเชื่อถือ
- รายการประเภทข้างต้นเป็น *ฉบับร่าง* และอาจปรับปรุงในเอกสาร Domain ฉบับถัดไป

## 7. ระดับของข้อมูล

หน่วยองค์ความรู้ทุกหน่วยต้องมี **ระดับความน่าเชื่อถือ** กำกับ โดยอ้างอิงลำดับชั้นแหล่งข้อมูล (Data Authority) ที่กำหนดไว้ในเอกสารแม่ [PKA-STD-001 §10](../standards/PKA-STD-001-ธรรมนูญโครงการ.md)

```
   ระดับ A : แหล่งข้อมูลหลัก (สูงสุด)
        ▼
   ระดับ B
        ▼
   ระดับ C
        ▼
   ระดับ D : คำอธิบายของโครงการ
        ▼
   ระดับ E : ผลสรุปจากผู้ช่วยอัจฉริยะ (ต่ำสุด)
```

หลักการที่เกี่ยวข้องกับแบบจำลองโดเมน:

- ทุกหน่วยองค์ความรู้ต้อง **ระบุระดับ** ของตนเองได้เสมอ
- ระดับที่สูงกว่ามีน้ำหนักเหนือระดับที่ต่ำกว่าเมื่อเกิดความขัดแย้ง
- ผลสรุปจากผู้ช่วยอัจฉริยะอยู่ในระดับต่ำสุดเสมอ และ **ไม่มีวัน** เป็นแหล่งข้อมูลหลัก
- รายละเอียดของแต่ละระดับเป็นไปตามเอกสารแม่ และจะถูกขยายความใน [PKA-STD-004](../standards/PKA-STD-004-มาตรฐานพระธรรมและลำดับชั้นแหล่งข้อมูล.md) — เอกสารฉบับนี้เพียง *อ้างถึง* ไม่ได้นิยามใหม่

---

## 8. หลักการอ้างอิง

แบบจำลองโดเมนยึดหลักการอ้างอิงตามเอกสารแม่ และมาตรฐานการอ้างอิง [PKA-STD-003](../standards/PKA-STD-003-มาตรฐานการอ้างอิง.md) ดังนี้

1. **ทุกหน่วยองค์ความรู้ต้องสืบย้อนกลับได้** ไปยังแหล่งข้อมูลหลักเสมอ
2. **สายการสืบทอดข้อมูลต้องไม่ขาดตอน** — หากสายขาด ณ จุดใด ถือว่าหน่วยนั้นยังไม่สมบูรณ์
3. **ความสัมพันธ์ต้องมีที่มา** — การอ้างอิงและการเชื่อมโยงทุกชนิดต้องระบุหลักฐานได้
4. **แยกเนื้อหาต้นทางออกจากคำอธิบาย** — ผู้ใช้ต้องแยกได้เสมอว่าหน่วยใดเป็นเนื้อหาต้นทาง หน่วยใดเป็นคำอธิบาย
5. รายละเอียดของรูปแบบและองค์ประกอบการอ้างอิงเป็นไปตาม PKA-STD-003 — เอกสารฉบับนี้เพียงกำหนดว่าแบบจำลองโดเมน *ต้องรองรับ* การอ้างอิงเหล่านั้น

---

## 9. สิ่งที่ยังไม่กำหนด

เพื่อความชัดเจน เอกสารฉบับนี้ **จงใจยังไม่กำหนด** สิ่งต่อไปนี้ และจะกำหนดในเอกสารหรือขั้นถัดไป

- เนื้อหาพระธรรม องค์ธรรม และหมวดธรรมใด ๆ ทั้งสิ้น
- รายการจริง (instance) ของหน่วยองค์ความรู้และความสัมพันธ์
- การจับคู่ว่าเนื้อหาใดเป็นองค์ความรู้ชนิดใด หรือสัมพันธ์กันแบบใด
- รายละเอียดของข้อมูลกำกับในแต่ละชนิด (เกินกว่าระดับกรอบ)
- การออกแบบฐานข้อมูล โครงสร้างการจัดเก็บ หรือรูปแบบไฟล์
- เครื่องมือ เทคโนโลยี และรายละเอียดการ Implement
- กฎการตรวจสอบความถูกต้องเชิงลึกของแต่ละชนิด

> สิ่งที่ยังไม่กำหนดเหล่านี้ไม่ใช่ความบกพร่อง แต่เป็นเจตนาของการออกแบบ "กรอบก่อนเนื้อหา"

## 10. แผนการพัฒนาเอกสารชุด Domain

เอกสาร PKA-DOM-001 ฉบับนี้เป็น **กรอบตั้งต้น** ของเอกสารชุด Domain ทั้งหมด

> **เงื่อนไขสำคัญ:** เมื่อเอกสารฉบับนี้ (PKA-DOM-001) ได้รับการอนุมัติแล้วเท่านั้น จึงจะสามารถจัดทำ **PKA-DOM-002**, **PKA-DOM-003** และ **PKA-DOM-004** ในลำดับถัดไปได้

| รหัส | ขอบเขตที่คาดไว้ (เบื้องต้น) | สถานะ |
|---|---|---|
| **PKA-DOM-001** | กรอบของแบบจำลองโดเมน (เอกสารฉบับนี้) | Draft 0.1 |
| **PKA-DOM-002** | แค็ตตาล็อกชนิดขององค์ความรู้แบบละเอียด (ขยายจาก §5) | ยังไม่เริ่ม (รออนุมัติ DOM-001) |
| **PKA-DOM-003** | แค็ตตาล็อกชนิดของความสัมพันธ์แบบละเอียด (ขยายจาก §6) | ยังไม่เริ่ม (รออนุมัติ DOM-001) |
| **PKA-DOM-004** | จุดเริ่มการวิเคราะห์โครงสร้างองค์ความรู้จากคัมภีร์ (เริ่มแตะเนื้อหา) | ยังไม่เริ่ม (รออนุมัติ DOM-001) |

ขอบเขตข้างต้นเป็นการคาดการณ์เบื้องต้น และอาจปรับเมื่อ DOM-001 ได้รับการอนุมัติ
