## 1. วัตถุประสงค์ (Purpose)

เอกสารฉบับนี้กำหนด **วงจรชีวิตมาตรฐาน** ของตัวแทนงาน (Agent) เพื่อให้

1. ตัวแทนงานทุกตัว **มีสถานะที่ชัดเจน** และทราบได้เสมอว่าอยู่ขั้นใดของวงจรชีวิต
2. การเกิด การใช้งาน การระงับ และการปลดระวางตัวแทนงานเป็นไปอย่าง **มีระเบียบและมีร่องรอย**
3. มนุษย์เป็นผู้กำกับการเปลี่ยนผ่านที่สำคัญของตัวแทนงานเสมอ

## 2. ขอบเขต (Scope)

**อยู่ในขอบเขต**

- สถานะทั้งหกของวงจรชีวิตตัวแทนงาน และเงื่อนไขการเปลี่ยนผ่าน
- กฎของการเปลี่ยนผ่านและร่องรอยที่ต้องบันทึก

**อยู่นอกขอบเขต**

- ตัวแทนงานรายตัวและสถานะปัจจุบันของแต่ละตัว (อยู่ในสารบบ `catalog/`)
- เครื่องมือ เทคโนโลยี กรอบงาน ฐานข้อมูล หรือการเขียนโปรแกรมใด ๆ

## 3. ภาพรวมวงจรชีวิต (Lifecycle Overview)

ตัวแทนงานหนึ่งตัวผ่านสถานะตามลำดับดังนี้

```
   ข้อเสนอ (Proposal)
        ▼
   ออกแบบแล้ว (Designed)
        ▼
   อนุมัติแล้ว (Approved)
        ▼
   ปฏิบัติการ (Operational)
        ▼
   ระงับชั่วคราว (Suspended)
        ▼
   ปลดระวาง (Retired)
```

> ขั้น **ระงับชั่วคราว (Suspended)** เป็นสถานะที่ตัวแทนงานอาจกลับเข้าสู่ **ปฏิบัติการ (Operational)** ได้อีก ส่วน **ปลดระวาง (Retired)** เป็นสถานะสิ้นสุดที่ไม่นำตัวแทนงานกลับมาใช้ในรหัสเดิม

## 4. ขั้นข้อเสนอ (Proposal)

- ตัวแทนงานเริ่มต้นในฐานะ **ข้อเสนอ** ว่าควรมีบทบาทนี้ในองค์กรหรือไม่
- ระบุ **ภารกิจ ความจำเป็น และสำนักงานที่จะสังกัด** ในระดับแนวคิด ยังไม่ลงรายละเอียดพิมพ์เขียวเต็ม
- ข้อเสนอต้องแสดงว่าบทบาทนี้สอดคล้องกับหลัก "หนึ่งตัวแทนงาน หนึ่งภารกิจ" และไม่ทับซ้อนกับตัวแทนงานเดิม

> ขั้นนี้ตอบคำถามว่า *"องค์กรควรมีตัวแทนงานนี้หรือไม่"*

## 5. ขั้นออกแบบแล้ว (Designed)

- เมื่อข้อเสนอได้รับการยอมรับ ตัวแทนงานเข้าสู่ขั้น **ออกแบบแล้ว**
- กรอก **พิมพ์เขียวตัวแทนงาน** ให้ครบทุกองค์ประกอบตาม [PKA-AGT-002](PKA-AGT-002-พิมพ์เขียวตัวแทนงาน.md)
- ตรวจให้ความรับผิดชอบ อำนาจ และข้อจำกัดสอดคล้องกับ [PKA-AGT-003](PKA-AGT-003-มาตรฐานบทบาทและความรับผิดชอบ.md)

> ขั้นนี้ตอบคำถามว่า *"ตัวแทนงานนี้คืออะไรอย่างครบถ้วน"*

## 6. ขั้นอนุมัติแล้ว (Approved)

- ตัวแทนงานที่ออกแบบครบเข้าสู่การพิจารณาเพื่อ **อนุมัติ**
- การอนุมัติให้บทบาทเข้าสู่องค์กรเป็น **อำนาจของมนุษย์ผู้รับผิดชอบ** ตามธรรมาภิบาลใน [PKA-AGT-001](PKA-AGT-001-ธรรมนูญตัวแทนงาน.md)
- เมื่ออนุมัติแล้ว ตัวแทนงานพร้อมเข้าสู่การปฏิบัติการ แต่ยังไม่เริ่มทำงานจนกว่าจะเปลี่ยนสถานะ

> ขั้นนี้ตอบคำถามว่า *"องค์กรรับรองให้บทบาทนี้มีอยู่จริงแล้วหรือยัง"*

## 7. ขั้นปฏิบัติการ (Operational)

- ตัวแทนงานที่อนุมัติแล้วเข้าสู่ **ปฏิบัติการ** — รับงานผ่านเหตุการณ์และทำภารกิจตามบทบาท
- ขณะปฏิบัติการ ตัวแทนงานอยู่ใต้ **อำนาจแทรกแซงของมนุษย์** เสมอ และผลงานที่กระทบความถูกต้องต้องผ่านการทบทวนของมนุษย์
- ผลงานวัดด้วยดัชนีและเกณฑ์ความสำเร็จในพิมพ์เขียว โดยความถูกต้องเป็นดัชนีสูงสุด

> ขั้นนี้คือช่วงที่ตัวแทนงาน *ทำงานจริง* ภายใต้การกำกับขององค์กร

## 8. ขั้นระงับชั่วคราว (Suspended)

- ตัวแทนงานอาจถูก **ระงับชั่วคราว** เมื่อพบปัญหา เมื่อบทบาทยังไม่จำเป็นชั่วคราว หรือเมื่อมนุษย์ผู้รับผิดชอบสั่งระงับ
- ขณะระงับ ตัวแทนงาน **หยุดรับงาน** แต่ยังคงนิยามและพิมพ์เขียวไว้
- ตัวแทนงานที่ระงับสามารถ **กลับเข้าสู่ปฏิบัติการ** ได้เมื่อเหตุของการระงับหมดไป โดยมนุษย์ผู้รับผิดชอบเป็นผู้ตัดสิน

> ขั้นนี้เป็นการ *พัก* ไม่ใช่การ *จบ* — บทบาทยังคงอยู่ รอการกลับมาหรือการปลดระวาง

## 9. ขั้นปลดระวาง (Retired)

- ตัวแทนงานเข้าสู่ **ปลดระวาง** เมื่อบทบาทไม่จำเป็นอีกต่อไป หรือถูกแทนที่ด้วยบทบาทใหม่
- ตัวแทนงานที่ปลดระวาง **หยุดทำงานถาวร** แต่คงบันทึกไว้เพื่อการอ้างอิงและการตรวจสอบย้อนกลับ
- **ไม่นำรหัสตัวแทนงานที่ปลดระวางกลับมาใช้ซ้ำ** สอดคล้องกับหลักการตั้งชื่อขององค์กร

> ขั้นนี้เป็นสถานะสิ้นสุดของวงจรชีวิต — บทบาทจบลงอย่างมีร่องรอย

## 10. กฎของการเปลี่ยนผ่าน (Transition Rules)

1. **เปลี่ยนผ่านไปตามลำดับ** — ตัวแทนงานเดินตามลำดับขั้น ไม่ข้ามขั้นที่จำเป็น (เช่น ต้องออกแบบก่อนอนุมัติ)
2. **การอนุมัติเป็นของมนุษย์** — การเปลี่ยนผ่านเข้าสู่ขั้นอนุมัติและการกลับเข้าปฏิบัติการจากการระงับ เป็นการตัดสินของมนุษย์ผู้รับผิดชอบ
3. **ระงับแล้วกลับได้ ปลดระวางแล้วจบ** — Suspended กลับสู่ Operational ได้ แต่ Retired เป็นสถานะสิ้นสุด
4. **ทุกการเปลี่ยนผ่านมีร่องรอย** — บันทึกว่าใครเปลี่ยนสถานะใด เมื่อใด และด้วยเหตุใด
5. **เคารพอำนาจแทรกแซง** — มนุษย์สามารถสั่งระงับหรือปลดระวางตัวแทนงานได้ทุกเมื่อตามอำนาจแทรกแซงใน [PKA-AGT-001](PKA-AGT-001-ธรรมนูญตัวแทนงาน.md)

```
   Proposal ─▶ Designed ─▶ Approved ─▶ Operational ─┬─▶ Suspended ─┐
                                          ▲          │             │
                                          └──────────┘ (กลับมาได้)  │
                                                                    ▼
                                                                 Retired (สิ้นสุด)
```

## 11. ความสัมพันธ์กับเอกสารอื่น

- **[PKA-AGT-001 — ธรรมนูญตัวแทนงาน](PKA-AGT-001-ธรรมนูญตัวแทนงาน.md)** — ธรรมาภิบาล อำนาจแทรกแซง และการอนุมัติของมนุษย์ที่วงจรชีวิตนี้ยึดถือ
- **[PKA-AGT-002 — พิมพ์เขียวตัวแทนงาน](PKA-AGT-002-พิมพ์เขียวตัวแทนงาน.md)** — สิ่งที่ต้องกรอกในขั้นออกแบบแล้ว
- **[PKA-AGT-003 — มาตรฐานบทบาทและความรับผิดชอบ](PKA-AGT-003-มาตรฐานบทบาทและความรับผิดชอบ.md)** — หลักที่ต้องตรวจในขั้นออกแบบ
- **[PKA-OPS-001 — ธรรมนูญสำนักงานดิจิทัล](../operations/PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md)** — หลักการมนุษย์ในวงจรการตัดสินใจในระดับองค์กร

## 12. นิยามคำศัพท์

| คำศัพท์ | ความหมาย |
|---|---|
| **ข้อเสนอ (Proposal)** | สถานะที่ตัวแทนงานเป็นเพียงข้อเสนอว่าควรมีบทบาทนี้หรือไม่ |
| **ออกแบบแล้ว (Designed)** | สถานะที่กรอกพิมพ์เขียวตัวแทนงานครบทุกองค์ประกอบแล้ว |
| **อนุมัติแล้ว (Approved)** | สถานะที่มนุษย์ผู้รับผิดชอบรับรองให้บทบาทเข้าสู่องค์กร |
| **ปฏิบัติการ (Operational)** | สถานะที่ตัวแทนงานทำภารกิจจริงภายใต้การกำกับ |
| **ระงับชั่วคราว (Suspended)** | สถานะพักที่ตัวแทนงานหยุดรับงานแต่ยังคงนิยามไว้ และกลับมาได้ |
| **ปลดระวาง (Retired)** | สถานะสิ้นสุดที่ตัวแทนงานหยุดถาวร โดยไม่นำรหัสกลับมาใช้ซ้ำ |
| **การเปลี่ยนผ่าน (Transition)** | การเปลี่ยนสถานะของตัวแทนงานจากขั้นหนึ่งสู่อีกขั้น พร้อมร่องรอย |
