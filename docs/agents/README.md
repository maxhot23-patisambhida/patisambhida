# agents/ — พิมพ์เขียวตัวแทนงาน (PKA-AGT)

โฟลเดอร์นี้บรรจุเอกสารตระกูล **พิมพ์เขียวตัวแทนงาน (`PKA-AGT-*`)** ซึ่งเป็น **ชั้นพิมพ์เขียวตัวแทนงาน (Agent Blueprint Layer)** ของโครงการ — กำหนดว่า **ตัวแทนงาน (Agent) หนึ่งตัวคืออะไร ประกอบด้วยอะไร แบ่งหน้าที่อย่างไร สื่อสารอย่างไร และมีวงจรชีวิตอย่างไร** เพื่อให้บุคลากรดิจิทัลทั้งหมดในสำนักงานใหญ่ดิจิทัลของ PKA มีพิมพ์เขียวมาตรฐานเดียวกัน

## หน้าที่ของตระกูล AGT

- ตระกูล AGT อธิบาย **"ตัวแทนงานในฐานะบุคลากรดิจิทัล"** — หลักการ บทบาท ความรับผิดชอบ การสื่อสาร และวงจรชีวิตของตัวแทนงาน
- ตระกูล AGT เป็น **พิมพ์เขียวและหลักการเชิงแนวคิด (Conceptual Architecture)** **ไม่ใช่** การจัดวางตัวแทนงานจริง ไม่ใช่ชุดคำสั่ง และ **ไม่กล่าวถึงเทคโนโลยี กรอบงาน ฐานข้อมูล หรือการเขียนโปรแกรมใด ๆ**
- หลักการสำคัญที่สุดของตระกูลนี้คือ **ตัวแทนงานไม่ใช่ปัญญาประดิษฐ์** — ตัวแทนงานคือบทบาทขององค์กร ส่วนปัญญาประดิษฐ์เป็นเพียงผู้ปฏิบัติหน้าที่ และมนุษย์เป็นผู้รับผิดชอบสูงสุดเสมอ

## ความสัมพันธ์กับตระกูล OPS

- ตระกูล AGT **ไม่ใช่ส่วนหนึ่งของ PKA-OPS** แต่เป็นตระกูลใหม่ที่ **เชื่อมระหว่างสถาปัตยกรรมองค์กร (OPS) กับสถาปัตยกรรมตัวแทนงานในอนาคต (OPS-008)**
- **รับนิยามตั้งต้นจาก OPS** — แนวคิดตัวแทนงานถูกนิยามครั้งแรกในระดับองค์กรที่ [PKA-OPS-001](../operations/PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md) ส่วน AGT ขยายให้เป็นพิมพ์เขียวที่สมบูรณ์
- **เคารพโครงสร้างของ OPS** — ตัวแทนงานทุกตัวต้องสังกัดสำนักงานและอยู่ในสายการบังคับบัญชาตาม [PKA-OPS-002](../operations/PKA-OPS-002-โครงสร้างองค์กรและสายการบังคับบัญชา.md)
- ทุกฉบับต้องอ้างอิง [PKA-STD-001 (เอกสารแม่)](../standards/PKA-STD-001-ธรรมนูญโครงการ.md)
