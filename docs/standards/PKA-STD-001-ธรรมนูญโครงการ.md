## 1. บทนำ

โครงการ **Patisambhidamagga Knowledge Architecture (PKA)** เป็นโครงการที่มุ่งสร้างสถาปัตยกรรมความรู้สำหรับการศึกษาพระธรรม โดยมีรากฐานจากคัมภีร์ปฏิสัมภิทามรรค และเชื่อมโยงกับพระไตรปิฎกอย่างเป็นระบบ

ปฏิสัมภิทามรรคเป็นคัมภีร์ที่ลึกซึ้ง มีโครงสร้างของหลักธรรมที่ละเอียดและสัมพันธ์กันอย่างซับซ้อน ผู้ศึกษาจำนวนมากเข้าถึงได้ยาก เนื่องจากเนื้อหามีความเชื่อมโยงข้ามหมวด ข้ามคัมภีร์ และต้องอาศัยการอ้างอิงย้อนกลับไปยังแหล่งข้อมูลต้นทางอยู่เสมอ

โครงการนี้จึงเกิดขึ้นเพื่อทำให้การศึกษาพระธรรมเป็นเรื่องที่ *เข้าถึงได้* *ตรวจสอบได้* และ *เชื่อมโยงได้* โดยยึดความถูกต้องของพระธรรมเป็นที่ตั้งเหนือสิ่งอื่นใด

เอกสารฉบับนี้วางรากฐานทางความคิดและหลักการที่ทุกส่วนของโครงการต้องยึดถือร่วมกัน เพื่อให้การพัฒนาในทุกระยะเดินไปในทิศทางเดียวกัน และไม่หลุดออกจากเจตนารมณ์ดั้งเดิม

## 2. ปรัชญาของโครงการ

โครงการนี้เริ่มต้นจากความเชื่อพื้นฐานชุดหนึ่งเกี่ยวกับความสัมพันธ์ระหว่างพระธรรมกับเครื่องมือที่มนุษย์สร้างขึ้น ความเชื่อเหล่านี้เป็นรากของทุกการตัดสินใจในโครงการ

**พระธรรมไม่ควรถูกจำกัดด้วยรูปแบบของเอกสาร** เนื้อหาพระธรรมดำรงอยู่เหนือภาชนะที่บรรจุมัน ไม่ว่าจะเป็นใบลาน หน้ากระดาษ หรือสิ่งใดในยุคปัจจุบัน ภาชนะเป็นเพียงสิ่งชั่วคราวที่ช่วยส่งต่อ ส่วนพระธรรมคือสิ่งที่ต้องรักษาให้คงความบริสุทธิ์ โครงการจึงไม่ยึดติดกับรูปแบบใดรูปแบบหนึ่ง แต่ยึดที่การรักษาแก่นของเนื้อหาให้ส่งต่อได้อย่างไม่คลาดเคลื่อน

**เครื่องมือควรช่วยให้เข้าถึงพระธรรม ไม่ใช่แทนที่พระธรรม** สิ่งที่โครงการสร้างขึ้นเป็นเพียงสื่อกลาง มีหน้าที่นำพาผู้ศึกษาไปสู่เนื้อหาต้นทาง ไม่ใช่มายืนแทนเนื้อหานั้น ผู้ศึกษาต้องสามารถเดินผ่านเครื่องมือไปถึงแหล่งข้อมูลต้นทางได้เสมอ และต้องไม่มีวันเข้าใจผิดว่าคำอธิบายของเครื่องมือคือตัวพระธรรมเอง

**ความถูกต้องสำคัญกว่าความสะดวก** ความสะดวกเป็นสิ่งพึงปรารถนา แต่เมื่อใดที่ความสะดวกต้องแลกมาด้วยความคลาดเคลื่อนของพระธรรม โครงการเลือกความถูกต้องเสมอ การเข้าถึงง่ายที่นำพาความเข้าใจผิด ย่อมเลวร้ายกว่าการเข้าถึงยากที่รักษาความถูกต้องไว้ได้

**เป้าหมายคือทำให้การศึกษาพระธรรมเข้าถึงง่ายขึ้น โดยไม่ลดทอนความถูกต้อง** ความง่ายกับความถูกต้องไม่จำเป็นต้องขัดกัน หน้าที่ของโครงการคือหาหนทางที่ทั้งสองสิ่งนี้ไปด้วยกันได้ ทำให้สิ่งที่ลึกซึ้งเข้าถึงได้โดยไม่ทำให้สิ่งที่ลึกซึ้งกลายเป็นสิ่งตื้นเขิน

**โครงการนี้เป็นสะพานเชื่อมองค์ความรู้ ไม่ใช่ผู้สร้างองค์ความรู้** โครงการไม่ได้สร้างพระธรรมขึ้นใหม่ ไม่ได้เพิ่มหลักธรรม และไม่ได้ตั้งตนเป็นแหล่งความรู้ใหม่ บทบาทของโครงการคือการเป็นสะพาน — เชื่อมผู้ศึกษาเข้ากับองค์ความรู้ที่มีอยู่แล้ว เชื่อมหลักธรรมหนึ่งเข้ากับอีกหลักธรรมหนึ่ง และเชื่อมคำถามของผู้คนเข้ากับคำตอบที่ปรากฏอยู่ในแหล่งข้อมูลต้นทาง

## 3. เป้าหมายของโครงการ

โครงการ PKA มีเป้าหมายหลักดังนี้

1. **ทำให้พระธรรมเข้าถึงได้ง่าย** — ลดอุปสรรคในการเข้าถึงและทำความเข้าใจหลักธรรมที่ลึกซึ้ง
2. **รักษาความถูกต้องของพระธรรม** — ทุกเนื้อหาต้องสืบย้อนกลับไปยังแหล่งข้อมูลต้นทางได้เสมอ
3. **สร้างการเชื่อมโยงความรู้** — แสดงความสัมพันธ์ระหว่างหลักธรรม หมวดหมู่ และคัมภีร์
4. **สนับสนุนการศึกษาเชิงลึก** — เป็นเครื่องมือช่วยผู้ศึกษาทุกระดับ ตั้งแต่ผู้เริ่มต้นจนถึงผู้ศึกษาขั้นสูง
5. **คงไว้ซึ่งความน่าเชื่อถือ** — เป็นแหล่งศึกษาที่ผู้ใช้วางใจได้ว่าจะไม่บิดเบือนหลักธรรม

## 4. วิสัยทัศน์

> เป็นสถาปัตยกรรมความรู้พระธรรมที่ **ถูกต้อง ตรวจสอบได้ และเชื่อมโยงกัน** ซึ่งทำให้ผู้คนทุกระดับสามารถเข้าถึงและเข้าใจปฏิสัมภิทามรรคและพระไตรปิฎกได้อย่างลึกซึ้ง โดยไม่สูญเสียความบริสุทธิ์ของพระธรรม

โครงการมุ่งเป็นสะพานเชื่อมระหว่างความลึกซึ้งของพระธรรมดั้งเดิมกับผู้ศึกษาในยุคปัจจุบัน โดยรักษาสมดุลระหว่าง *ความเข้าถึงง่าย* กับ *ความถูกต้องสมบูรณ์*

## 5. พันธกิจ

เพื่อบรรลุวิสัยทัศน์ โครงการมีพันธกิจดังนี้

- รวบรวมและจัดระเบียบเนื้อหาพระธรรมให้เป็นโครงสร้างที่ค้นหาและเชื่อมโยงได้
- รักษาสายการอ้างอิง (citation) ของทุกเนื้อหาให้สืบย้อนไปยังพระไตรปิฎกได้เสมอ
- จัดให้มีระบบค้นหาที่เป็นหัวใจของการเข้าถึงความรู้
- จัดให้มีผู้ช่วยอัจฉริยะที่ช่วยค้นหา อธิบาย และเชื่อมโยง โดยไม่สร้างหลักธรรมขึ้นใหม่
- ออกแบบประสบการณ์การใช้งานที่เคารพต่อเนื้อหาพระธรรมและเหมาะกับการศึกษาอย่างมีสมาธิ
- พัฒนาอย่างต่อเนื่องและรับผิดชอบ โดยยึดหลักการในเอกสารฉบับนี้เป็นเข็มทิศ

## 6. คุณค่าหลักของโครงการ (Core Values)

คุณค่าหกประการนี้เป็นเกณฑ์ที่ใช้วัดทุกการตัดสินใจของโครงการ เมื่อมีข้อสงสัยว่าควรเดินทางใด ให้ย้อนกลับมาที่คุณค่าเหล่านี้

### 6.1 ความถูกต้อง (Accuracy)

ความถูกต้องคือรากฐานที่ทุกอย่างของโครงการตั้งอยู่ เนื้อหาพระธรรมที่คลาดเคลื่อนแม้เพียงเล็กน้อยอาจนำพาผู้ศึกษาไปสู่ความเข้าใจผิดที่ขยายตัวต่อไปได้ โครงการจึงถือว่าความถูกต้องของพระธรรมเป็นเรื่องที่ประนีประนอมไม่ได้

ความถูกต้องในที่นี้ไม่ได้หมายถึงเพียงการสะกดคำให้ตรงเท่านั้น แต่หมายถึงการรักษาความหมาย บริบท และเจตนารมณ์ของเนื้อหาต้นทางอย่างครบถ้วน เนื้อหาที่ถูกตัดทอนจนเปลี่ยนความหมาย ถือว่าไม่ถูกต้องแม้ทุกตัวอักษรจะตรงกับต้นฉบับ

เมื่อใดที่ความถูกต้องขัดแย้งกับคุณค่าอื่น ความถูกต้องเป็นฝ่ายชนะเสมอ นี่คือหลักที่ไม่มีข้อยกเว้น

### 6.2 ความโปร่งใส (Transparency)

ผู้ใช้ต้องมองเห็นได้เสมอว่าเนื้อหาที่ตนกำลังอ่านมาจากไหน อยู่ในระดับชั้นใดของความน่าเชื่อถือ และสิ่งใดเป็นพุทธพจน์ สิ่งใดเป็นคำอธิบาย โครงการไม่ปิดบังที่มาของข้อมูล และไม่ทำให้ผู้ใช้เข้าใจผิดเกี่ยวกับสถานะของเนื้อหา

ความโปร่งใสยังหมายถึงการเปิดเผยข้อจำกัดของตนเอง เมื่อโครงการไม่มีข้อมูล ไม่แน่ใจ หรือมีเนื้อหาที่ยังตรวจสอบไม่ได้ ต้องบอกตามจริง ไม่กลบเกลื่อน

ความโปร่งใสคือสิ่งที่ทำให้ผู้ใช้สามารถตัดสินใจได้ด้วยตนเองว่าจะวางใจเนื้อหาใดมากน้อยเพียงใด โครงการไม่ตัดสินใจแทนผู้ใช้ แต่ให้ข้อมูลที่เพียงพอเพื่อให้ผู้ใช้ตัดสินใจเองได้

### 6.3 การตรวจสอบได้ (Verifiability)

ทุกเนื้อหาในโครงการต้องสามารถสืบย้อนกลับไปยังแหล่งข้อมูลต้นทางได้จริง ไม่ใช่เพียงอ้างว่ามีที่มา แต่ผู้ใช้ต้องตามรอยนั้นไปจนถึงต้นทางได้ด้วยตนเอง

การตรวจสอบได้เป็นสิ่งที่แยกองค์ความรู้ที่น่าเชื่อถือออกจากคำกล่าวอ้างลอย ๆ เนื้อหาที่ตรวจสอบไม่ได้ ไม่ว่าจะฟังดูน่าเชื่อถือเพียงใด ก็ยังไม่อยู่ในสถานะที่พร้อมให้ผู้ใช้วางใจอย่างเต็มที่

โครงการถือว่าการตรวจสอบได้เป็นหน้าที่ของระบบ ไม่ใช่ภาระของผู้ใช้ ระบบต้องออกแบบให้การตามรอยกลับไปยังต้นทางเป็นเรื่องง่ายและเป็นธรรมชาติ

### 6.4 ความเรียบง่าย (Simplicity)

ความเรียบง่ายเป็นคุณค่าที่ปกป้องทั้งผู้ใช้และโครงการเอง สำหรับผู้ใช้ ความเรียบง่ายช่วยให้จดจ่อกับเนื้อหาพระธรรมโดยไม่ถูกรบกวน สำหรับโครงการ ความเรียบง่ายทำให้ระบบยั่งยืนและดูแลรักษาได้ในระยะยาว

ความเรียบง่ายไม่ใช่ความมักง่าย แต่เป็นผลของการคัดกรองอย่างพิถีพิถันว่าสิ่งใดจำเป็นจริง ๆ สิ่งที่ไม่ได้รับใช้การศึกษาพระธรรมโดยตรง ควรถูกตัดออก ไม่ใช่สะสมไว้

ในทุกทางเลือก หากผลลัพธ์เท่ากัน โครงการเลือกหนทางที่เรียบง่ายกว่าเสมอ เพราะความซับซ้อนที่ไม่จำเป็นคือหนี้ที่ต้องชดใช้ในอนาคต

### 6.5 ความยั่งยืน (Sustainability)

โครงการนี้มุ่งหมายให้เป็นเอกสารและระบบที่ดำรงอยู่ได้ยาวนานนับสิบปี การตัดสินใจทุกอย่างจึงต้องคำนึงถึงความยั่งยืน ไม่ใช่เพียงความได้เปรียบในระยะสั้น

ความยั่งยืนหมายถึงการสร้างสิ่งที่ผู้อื่นสามารถสานต่อได้ เข้าใจได้ และดูแลรักษาได้แม้ผู้ริเริ่มไม่อยู่แล้ว โครงสร้างที่ชัดเจน หลักการที่บันทึกไว้ และเหตุผลที่อธิบายได้ คือสิ่งที่ทำให้โครงการอยู่รอดข้ามกาลเวลา

โครงการให้คุณค่ากับสิ่งที่ทนทานมากกว่าสิ่งที่ฉาบฉวย และเลือกรากฐานที่มั่นคงมากกว่าความหวือหวาที่ผ่านไปเร็ว

### 6.6 การแบ่งปันองค์ความรู้ (Knowledge Sharing)

โครงการเกิดขึ้นเพื่อให้พระธรรมเข้าถึงได้กว้างขวางขึ้น มิใช่เพื่อกักเก็บไว้ องค์ความรู้ที่รวบรวมและจัดระเบียบไว้มีคุณค่าก็ต่อเมื่อถูกส่งต่อไปยังผู้ที่ต้องการศึกษา

การแบ่งปันต้องเป็นไปอย่างรับผิดชอบ — แบ่งปันเนื้อหาที่ถูกต้อง พร้อมที่มาที่ตรวจสอบได้ และในรูปแบบที่ผู้รับเข้าใจได้ การแบ่งปันที่ขาดความรับผิดชอบอาจกระจายความคลาดเคลื่อนได้เร็วเท่ากับกระจายความรู้

โครงการมองการแบ่งปันเป็นทั้งเป้าหมายและหน้าที่ และถือว่าการทำให้ผู้คนเข้าถึงพระธรรมที่ถูกต้องได้ง่ายขึ้น คือคุณค่าสูงสุดที่โครงการมุ่งมอบให้

---

## 7. ขอบเขต

### 7.1 อยู่ในขอบเขต

- เนื้อหาที่มีรากฐานจากปฏิสัมภิทามรรคและพระไตรปิฎก
- การจัดโครงสร้าง การเชื่อมโยง และการอ้างอิงเนื้อหาพระธรรม
- ระบบค้นหาและการนำทางเพื่อเข้าถึงเนื้อหา
- การช่วยอธิบายและเชื่อมโยงเนื้อหาที่มีอยู่จริง
- ประสบการณ์การอ่านและการศึกษาบนอุปกรณ์เคลื่อนที่เป็นหลัก

### 7.2 อยู่นอกขอบเขต

- การตีความที่ขัดแย้งกับพระไตรปิฎก
- การสร้างหลักธรรมหรือคำสอนขึ้นใหม่
- เนื้อหาที่ไม่สามารถสืบย้อนไปยังแหล่งข้อมูลต้นทางได้
- การวินิจฉัยหรือชี้ขาดข้อโต้แย้งทางหลักธรรมแทนผู้รู้

---

## 8. หลักการของโครงการ

หลักการแม่บทที่ครอบคลุมทั้งโครงการ และเป็นที่มาของหลักการเฉพาะด้านในหัวข้อถัด ๆ ไป

1. **ความถูกต้องมาก่อนความสะดวก** — เมื่อใดที่ความถูกต้องของพระธรรมขัดกับความสะดวกในการใช้งาน ให้ความถูกต้องเป็นฝ่ายชนะเสมอ
2. **ตรวจสอบย้อนกลับได้** — ข้อมูลทุกอย่างต้องสืบย้อนไปยังแหล่งข้อมูลต้นทางได้
3. **แหล่งข้อมูลสูงสุดเดียว** — พระไตรปิฎกเป็นแหล่งข้อมูลสูงสุด และข้อมูลแต่ละชิ้นต้องมี Source of Truth เพียงแห่งเดียว
4. **แยกข้อมูลออกจากการแสดงผล** — เนื้อหากับวิธีนำเสนอต้องไม่ปะปนกัน
5. **การค้นหาเป็นหัวใจ** — ระบบค้นหาคือทางเข้าหลักสู่ความรู้
6. **AI เป็นผู้ช่วย ไม่ใช่ผู้สร้างหลักธรรม**
7. **ภาษาไทยเป็นภาษาหลัก**
8. **Mobile First** — ออกแบบเพื่ออุปกรณ์เคลื่อนที่เป็นอันดับแรก
9. **ทุกฟีเจอร์ต้องรับใช้การศึกษาพระธรรม** — หากฟีเจอร์ใดไม่ช่วยให้ศึกษาพระธรรมได้ง่ายขึ้น ฟีเจอร์นั้นไม่ควรมี

---

## 9. หลักการด้านข้อมูล

1. ข้อมูลทุกชิ้นต้อง **ตรวจสอบย้อนกลับได้** ไปยังแหล่งข้อมูลต้นทาง
2. ข้อมูลแต่ละรายการมี **Source of Truth เพียงแห่งเดียว** ห้ามมีสำเนาที่ถือเป็นต้นฉบับซ้อนกัน
3. **แยกข้อมูลออกจากการแสดงผล** — เนื้อหาต้องเป็นอิสระจากรูปแบบการนำเสนอ
4. ข้อมูลต้องมีโครงสร้างที่ชัดเจน สม่ำเสมอ และคงเส้นคงวา
5. การเปลี่ยนแปลงข้อมูลต้องมีร่องรอยและเหตุผลกำกับ
6. ข้อมูลที่สืบย้อนไม่ได้ ถือว่ายังไม่พร้อมเผยแพร่

---

## 10. ลำดับชั้นของแหล่งข้อมูล (Data Authority)

ข้อมูลในโครงการไม่ได้มีน้ำหนักความน่าเชื่อถือเท่ากันทั้งหมด แต่มีลำดับชั้นที่ชัดเจน เนื้อหาในระดับสูงกว่าย่อมมีอำนาจเหนือเนื้อหาในระดับต่ำกว่าเสมอ เมื่อเกิดความขัดแย้ง ให้ยึดตามระดับที่สูงกว่า

```
        ┌─────────────────────────────┐
        │  ระดับ A : พระไตรปิฎก        │  ← แหล่งข้อมูลสูงสุด / Source of Truth
        └─────────────────────────────┘
                       │
                       ▼
        ┌─────────────────────────────┐
        │  ระดับ B : อรรถกถา           │
        └─────────────────────────────┘
                       │
                       ▼
        ┌─────────────────────────────┐
        │  ระดับ C : ฎีกา              │
        └─────────────────────────────┘
                       │
                       ▼
        ┌─────────────────────────────┐
        │  ระดับ D : คำอธิบายของโครงการ │
        └─────────────────────────────┘
                       │
                       ▼
        ┌─────────────────────────────┐
        │  ระดับ E : AI Summary        │  ← ระดับต่ำสุด / ห้ามเป็น Source of Truth
        └─────────────────────────────┘
```

- **ระดับ A — พระไตรปิฎก** เป็นแหล่งข้อมูลสูงสุด และเป็น Source of Truth เพียงแหล่งเดียวของหลักธรรม เนื้อหาในระดับนี้มีอำนาจเหนือทุกระดับ
- **ระดับ B — อรรถกถา** เป็นคำอธิบายขยายความพระไตรปิฎก มีน้ำหนักรองลงมา และต้องไม่ถูกยกขึ้นมาแทนที่พระไตรปิฎก
- **ระดับ C — ฎีกา** เป็นคำอธิบายขยายความอรรถกถาอีกชั้นหนึ่ง อยู่ในลำดับถัดลงมา
- **ระดับ D — คำอธิบายของโครงการ** เป็นเนื้อหาที่โครงการเรียบเรียงเพื่อช่วยการเข้าใจ ต้องระบุชัดว่าเป็นคำอธิบายของโครงการ และต้องไม่ถูกเข้าใจผิดว่าเป็นพุทธพจน์
- **ระดับ E — AI Summary** เป็นการสรุปและเชื่อมโยงโดยผู้ช่วยอัจฉริยะ อยู่ในระดับต่ำสุดของลำดับชั้น

> **AI ไม่มีวันอยู่ระดับเดียวกับพระไตรปิฎก**
> ไม่ว่าผู้ช่วยอัจฉริยะจะพัฒนาไปไกลเพียงใด ผลผลิตของมันจะอยู่ที่ระดับ E เสมอ และจะไม่มีวันเลื่อนขึ้นมาเทียบเท่าแหล่งข้อมูลในระดับ A
>
> **AI ไม่มีสิทธิ์เป็น Source of Truth**
> ผู้ช่วยอัจฉริยะเป็นผู้สรุปและเชื่อมโยงเนื้อหาที่มีอยู่จริงเท่านั้น ไม่ใช่ต้นทางของความจริง เนื้อหาที่ผู้ช่วยสร้างขึ้นต้องอ้างอิงกลับไปยังระดับที่สูงกว่าได้เสมอ

## 11. Data Lineage — สายธารของข้อมูล

ข้อมูลทุกชิ้นในโครงการต้องสามารถตอบคำถามสี่ข้อนี้ได้เสมอ

- **มาจากไหน** — แหล่งข้อมูลต้นทางของข้อมูลชิ้นนี้คืออะไร
- **ผ่านอะไรบ้าง** — ข้อมูลนี้ผ่านการประมวลผล จัดโครงสร้าง หรือเรียบเรียงอย่างไรบ้างกว่าจะมาถึงผู้ใช้
- **ถูกใช้ที่ใด** — ข้อมูลนี้ปรากฏหรือถูกนำไปใช้ในส่วนใดของระบบบ้าง
- **ถูกอ้างอิงที่ใด** — มีเนื้อหาอื่นใดที่อ้างอิงข้อมูลชิ้นนี้

สายธารของข้อมูลในโครงการไหลตามลำดับนี้

```
   พระไตรปิฎก
       │   (แหล่งข้อมูลต้นทาง)
       ▼
   โครงสร้างข้อมูล
       │   (จัดระเบียบให้ค้นหาและเชื่อมโยงได้)
       ▼
   Search
       │   (ทางเข้าหลักสู่ความรู้)
       ▼
   Knowledge
       │   (การเชื่อมโยงเป็นองค์ความรู้)
       ▼
   AI
       │   (ช่วยค้นหา อธิบาย เชื่อมโยง)
       ▼
   Presentation
           (การนำเสนอต่อผู้ใช้)
```

แต่ละขั้นในสายธารนี้ต้องรักษาความเชื่อมโยงกลับไปยังขั้นก่อนหน้า จนถึงพระไตรปิฎกซึ่งเป็นต้นทาง การนำเสนอต้องสืบกลับไปถึง AI หรือ Knowledge ได้ องค์ความรู้ต้องสืบกลับไปถึงโครงสร้างข้อมูลได้ และโครงสร้างข้อมูลต้องสืบกลับไปถึงพระไตรปิฎกได้

> **หากสายนี้ขาดที่จุดใดจุดหนึ่ง ถือว่าข้อมูลชิ้นนั้นยังไม่สมบูรณ์**
> เนื้อหาที่ไม่สามารถสืบสายกลับไปถึงต้นทางได้ ไม่ว่าจะดูสมบูรณ์เพียงใดในการนำเสนอ ก็ยังไม่อยู่ในสถานะที่พร้อมให้ผู้ใช้วางใจ และต้องได้รับการแก้ไขให้สายเชื่อมต่อครบก่อนเสมอ

---

## 12. หลักการด้านพระธรรม

1. **พระไตรปิฎกเป็นแหล่งข้อมูลสูงสุด** เหนือคำอธิบาย ความเห็น หรือการตีความใด ๆ
2. ลำดับชั้นของความน่าเชื่อถือต้องชัดเจน: พระไตรปิฎก → อรรถกถาและคัมภีร์รอง → คำอธิบายประกอบ
3. **ห้ามบิดเบือนหรือดัดแปลงหลักธรรม** เพื่อความสะดวกในการนำเสนอ
4. เมื่อมีการตีความหลายแนว ต้องนำเสนออย่างตรงไปตรงมา ไม่ชี้ขาดแทนผู้รู้
5. ต้องแยกให้ชัดระหว่าง *พุทธพจน์* *คำอธิบายของคัมภีร์* และ *คำอธิบายของโครงการ*
6. เคารพต่อความศักดิ์สิทธิ์ของเนื้อหาในทุกการตัดสินใจ

---

## 13. หลักการด้าน AI

1. **AI ไม่มีสิทธิ์สร้างหลักธรรมใหม่** — เป็นข้อห้ามเด็ดขาด
2. AI ทำหน้าที่เป็น **ผู้ช่วยค้นหา อธิบาย และเชื่อมโยง** เนื้อหาที่มีอยู่จริงเท่านั้น
3. คำตอบของ AI ต้อง **อ้างอิงแหล่งข้อมูลต้นทาง** ที่ตรวจสอบย้อนกลับได้เสมอ
4. AI ต้องไม่กลบเกลื่อนเมื่อไม่มีข้อมูล — ต้องบอกตามจริงว่าไม่พบหรือไม่แน่ใจ
5. AI ต้องไม่ตีความหลักธรรมในฐานะผู้ชี้ขาด แต่ช่วยนำเสนอแนวการตีความที่มีอยู่
6. ผู้ใช้ต้องสามารถแยกแยะได้เสมอว่าสิ่งใดมาจากแหล่งข้อมูล และสิ่งใดเป็นคำอธิบายของผู้ช่วย

---

## 14. หลักการด้าน Search

1. **Search เป็นหัวใจของระบบ** — เป็นทางเข้าหลักสู่ความรู้ทั้งหมด
2. การค้นหาต้องรวดเร็ว แม่นยำ และเข้าใจภาษาไทยเป็นอย่างดี
3. ผลการค้นหาทุกรายการต้อง **เชื่อมโยงกลับไปยังแหล่งข้อมูลต้นทาง** ได้
4. การค้นหาต้องรองรับทั้งการค้นด้วยคำตรงและการค้นด้วยความหมาย
5. การค้นหาต้องช่วยให้ผู้ใช้ *ค้นพบ* ความเชื่อมโยง ไม่ใช่เพียงแสดงรายการผลลัพธ์
6. ผลการค้นหาต้องโปร่งใส ผู้ใช้เข้าใจได้ว่าเหตุใดผลลัพธ์จึงปรากฏ

---

## 15. หลักการด้าน UX

1. ออกแบบเพื่อ **การศึกษาอย่างมีสมาธิ** ไม่รบกวนสมาธิของผู้อ่าน
2. ความเรียบง่ายสำคัญกว่าความหวือหวา — ลดสิ่งรบกวนที่ไม่จำเป็น
3. การนำทางต้องชัดเจน ผู้ใช้ต้องรู้เสมอว่าตนอยู่ที่ใดและจะไปต่อทางใด
4. การออกแบบต้องเคารพต่อความศักดิ์สิทธิ์ของเนื้อหา
5. ประสบการณ์การอ่านต้องสบายตา เหมาะกับการอ่านเนื้อหายาวและลึก
6. ทุกองค์ประกอบบนหน้าจอต้องมีเหตุผลรองรับว่าช่วยการศึกษาอย่างไร

---

## 16. หลักการด้าน Mobile

1. **Mobile First** — ออกแบบสำหรับอุปกรณ์เคลื่อนที่เป็นอันดับแรก แล้วจึงขยายสู่จอใหญ่
2. ผู้ใช้ต้องศึกษาพระธรรมได้สะดวกทุกที่ทุกเวลาบนอุปกรณ์เคลื่อนที่
3. การอ่านและการค้นหาบนจอเล็กต้องลื่นไหลและไม่ลดทอนความสามารถ
4. รองรับสภาพการใช้งานจริง เช่น การเชื่อมต่อที่ไม่เสถียร
5. ประหยัดทรัพยากรของผู้ใช้ ทั้งพลังงานและข้อมูล

---

## 17. หลักการด้านการอ้างอิง

1. ทุกเนื้อหาต้องมี **การอ้างอิงที่ชัดเจน** ไปยังแหล่งข้อมูลต้นทาง
2. การอ้างอิงต้องแม่นยำถึงระดับที่ผู้ใช้ตรวจสอบได้จริง
3. รูปแบบการอ้างอิงต้องสม่ำเสมอตลอดทั้งโครงการ
4. ต้องแยกชัดระหว่างการอ้างอิงโดยตรง (พุทธพจน์) กับการอ้างอิงโดยอ้อม (คำอธิบาย)
5. หากเนื้อหาใดไม่มีการอ้างอิงที่ตรวจสอบได้ ต้องระบุสถานะนี้อย่างเปิดเผย
6. การอ้างอิงคือหลักประกันความน่าเชื่อถือของโครงการ

---

## 18. หลักการด้านความถูกต้อง

1. **ความถูกต้องของพระธรรมอยู่เหนือทุกสิ่ง** รวมถึงเหนือความสวยงามและความสะดวก
2. เมื่อไม่แน่ใจในความถูกต้อง ให้เลือกหนทางที่ระมัดระวังที่สุด
3. ความผิดพลาดด้านเนื้อหาพระธรรมถือเป็นเรื่องร้ายแรงที่สุดของโครงการ
4. ต้องมีกระบวนการทบทวนและแก้ไขเมื่อพบความคลาดเคลื่อน
5. ความถูกต้องต้องตรวจสอบได้ ไม่ใช่เพียงอ้างว่าถูกต้อง
6. ความโปร่งใสเกี่ยวกับข้อจำกัดและความไม่แน่นอนเป็นส่วนหนึ่งของความถูกต้อง

---

## 19. หลักการด้านการพัฒนา

1. การพัฒนาทุกอย่างต้องสอดคล้องกับหลักการในเอกสารฉบับนี้
2. **ทุกฟีเจอร์ต้องช่วยให้ศึกษาพระธรรมได้ง่ายขึ้น** มิฉะนั้นไม่ควรพัฒนา
3. พัฒนาอย่างค่อยเป็นค่อยไป โดยรักษาความถูกต้องในทุกขั้น
4. ความเรียบง่ายและความยั่งยืนสำคัญกว่าความซับซ้อนชั่วคราว
5. การตัดสินใจต้องมีเหตุผลและร่องรอยที่ทบทวนได้
6. ฟังเสียงผู้ศึกษาจริง และปรับปรุงอย่างรับผิดชอบ

---

## 20. หลักการตัดสินใจ (Decision Principles)

เมื่อโครงการเผชิญทางเลือกหลายทางที่ดูสมเหตุสมผลพอ ๆ กัน ให้ตัดสินใจตามลำดับความสำคัญนี้ โดยข้อที่อยู่สูงกว่ามีน้ำหนักเหนือข้อที่อยู่ต่ำกว่าเสมอ

```
   1. ความถูกต้องของพระธรรม
            ▼
   2. การตรวจสอบย้อนกลับ
            ▼
   3. ความเข้าใจของผู้ใช้
            ▼
   4. ความเรียบง่าย
            ▼
   5. ความสวยงาม
```

1. **ความถูกต้องของพระธรรม** — เป็นเกณฑ์สูงสุด หากทางเลือกใดทำให้พระธรรมคลาดเคลื่อน ให้ตัดทางเลือกนั้นทันที ไม่ว่าจะได้เปรียบในด้านอื่นเพียงใด
2. **การตรวจสอบย้อนกลับ** — เมื่อทุกทางเลือกถูกต้องเท่ากัน ให้เลือกทางที่ยังรักษาการสืบย้อนกลับไปยังแหล่งข้อมูลต้นทางได้ดีกว่า เพราะความถูกต้องที่พิสูจน์ไม่ได้ยังไม่ครบสมบูรณ์
3. **ความเข้าใจของผู้ใช้** — เมื่อความถูกต้องและการตรวจสอบได้เท่ากัน ให้เลือกทางที่ช่วยให้ผู้ใช้เข้าใจพระธรรมได้ดีกว่า เพราะเป้าหมายของโครงการคือทำให้การศึกษาเข้าถึงง่ายขึ้น
4. **ความเรียบง่าย** — เมื่อทางเลือกช่วยความเข้าใจได้เท่ากัน ให้เลือกทางที่เรียบง่ายกว่า เพราะความเรียบง่ายปกป้องสมาธิของผู้ใช้และความยั่งยืนของระบบ
5. **ความสวยงาม** — เป็นเกณฑ์ลำดับสุดท้าย เมื่อทุกข้อข้างต้นเท่ากันแล้ว จึงพิจารณาความสวยงาม ความสวยงามเป็นสิ่งพึงปรารถนา แต่ต้องไม่มาก่อนสิ่งใดในสี่ข้อแรก

---

## 21. สิ่งที่โครงการจะไม่ทำ

1. **จะไม่สร้างหลักธรรมหรือคำสอนขึ้นใหม่**
2. **จะไม่บิดเบือนหรือดัดแปลงพระธรรม** เพื่อความสะดวกใด ๆ
3. **จะไม่เผยแพร่เนื้อหาที่สืบย้อนไปยังแหล่งข้อมูลต้นทางไม่ได้**
4. จะไม่ปล่อยให้ AI ทำหน้าที่ชี้ขาดหรือตีความหลักธรรมแทนผู้รู้
5. จะไม่ให้ความสวยงามหรือเทคโนโลยีอยู่เหนือความถูกต้องของพระธรรม
6. จะไม่เพิ่มฟีเจอร์ที่ไม่ช่วยการศึกษาพระธรรม
7. จะไม่ปะปนเนื้อหาต้นฉบับกับคำอธิบายจนผู้ใช้แยกไม่ออก
8. **ห้ามแก้ไขข้อความพระไตรปิฎก** — เนื้อหาต้นทางต้องคงไว้ตามต้นฉบับเสมอ
9. **ห้ามลบ Citation** — การอ้างอิงเป็นหลักประกันความน่าเชื่อถือ ห้ามตัดออกไม่ว่ากรณีใด
10. **ห้ามตัดข้อความจนความหมายเปลี่ยน** — การย่อหรือตัดทอนที่ทำให้ความหมายคลาดเคลื่อนถือเป็นการบิดเบือน
11. **ห้ามสร้างความสัมพันธ์ของข้อมูลโดยไม่มีหลักฐาน** — การเชื่อมโยงทุกเส้นต้องมีหลักฐานรองรับ
12. **ห้ามให้ AI ตอบโดยไม่มีที่มา** — คำตอบที่ไม่มีแหล่งอ้างอิงไม่อยู่ในมาตรฐานของโครงการ
13. **ห้ามผสมข้อความต้นฉบับกับคำอธิบายจนผู้ใช้แยกไม่ออก** — เส้นแบ่งระหว่างพุทธพจน์กับคำอธิบายต้องชัดเจนเสมอ

## 22. Roadmap ระดับสูง

> Roadmap นี้เป็นระดับวิสัยทัศน์ แสดงทิศทาง ไม่ใช่กำหนดการเชิงเทคนิค โดยลำดับของระยะสะท้อนลำดับความสำคัญ — ต้องวางรากฐานให้มั่นคงก่อนจึงสร้างสิ่งที่อยู่เหนือขึ้นไป

```
   Phase 1 : Architecture
        │   (วางรากฐานและสถาปัตยกรรมความรู้)
        ▼
   Phase 2 : Standards
        │   (กำหนดมาตรฐานและหลักการร่วม)
        ▼
   Phase 3 : Knowledge Model
        │   (สร้างแบบจำลององค์ความรู้และการเชื่อมโยง)
        ▼
   Phase 4 : User Experience
        │   (ออกแบบประสบการณ์การศึกษา)
        ▼
   Phase 5 : Implementation
        │   (นำหลักการสู่การปฏิบัติ)
        ▼
   Phase 6 : Continuous Improvement
            (รักษาความถูกต้องและพัฒนาต่อเนื่อง)
```

| ระยะ | จุดมุ่งหมาย |
|---|---|
| **Phase 1 — Architecture** | วางรากฐานทางความคิดและสถาปัตยกรรมความรู้ของโครงการ ให้มีทิศทางที่ชัดเจนก่อนสร้างสิ่งอื่น |
| **Phase 2 — Standards** | กำหนดมาตรฐานและหลักการร่วมที่ทุกส่วนของโครงการต้องยึดถือ เพื่อความสม่ำเสมอและตรวจสอบได้ |
| **Phase 3 — Knowledge Model** | สร้างแบบจำลองขององค์ความรู้ — โครงสร้าง การอ้างอิง และความเชื่อมโยงระหว่างหลักธรรม |
| **Phase 4 — User Experience** | ออกแบบประสบการณ์การอ่านและการศึกษาที่เคารพเนื้อหาและเข้าถึงง่าย |
| **Phase 5 — Implementation** | นำหลักการและแบบจำลองสู่การปฏิบัติจริง โดยรักษาความถูกต้องในทุกขั้น |
| **Phase 6 — Continuous Improvement** | รักษาความถูกต้อง ปรับปรุงต่อเนื่อง และขยายเนื้อหาอย่างรับผิดชอบในระยะยาว |

---

## 23. หลักการด้านการเปลี่ยนแปลงเอกสาร

เอกสารฉบับนี้เป็นรากฐานของโครงการ การเปลี่ยนแปลงจึงต้องเป็นไปอย่างระมัดระวังและมีร่องรอย

1. **เอกสารนี้เป็นเอกสารแม่ (Master Document)** — เป็นเอกสารระดับสูงสุดที่เอกสารอื่นทุกฉบับต้องสอดคล้องด้วย
2. **การแก้ไขต้องไม่ขัดกับหลักการเดิม** — การปรับปรุงสามารถเพิ่มความสมบูรณ์ได้ แต่ต้องไม่ทำให้หลักการที่วางไว้อ่อนลงหรือเปลี่ยนเจตนารมณ์
3. **หากเปลี่ยนหลักการระดับ Vision ต้องบันทึกเหตุผล** — การเปลี่ยนแปลงในระดับวิสัยทัศน์เป็นเรื่องใหญ่ ต้องมีคำอธิบายที่ชัดเจนว่าเหตุใดจึงเปลี่ยน และเปลี่ยนไปในทิศทางใด
4. **ต้องมี Version History** — ทุกการเปลี่ยนแปลงต้องบันทึกไว้ในประวัติการเปลี่ยนแปลงท้ายเอกสาร เพื่อให้สืบย้อนได้ว่าเอกสารวิวัฒน์มาอย่างไร
5. **เอกสารลูกทุกฉบับต้องอ้างอิง PKA-STD-001** — เอกสารระดับล่างทั้งหมดของโครงการต้องอ้างอิงเอกสารฉบับนี้เป็นต้นทางของหลักการ

## 24. นิยามคำศัพท์

| คำศัพท์ | ความหมาย |
|---|---|
| **ปฏิสัมภิทามรรค** | คัมภีร์ในพระไตรปิฎกที่เป็นรากฐานของโครงการ มีโครงสร้างหลักธรรมที่ละเอียดและเชื่อมโยงกัน |
| **พระไตรปิฎก** | คัมภีร์หลักทางพระพุทธศาสนา ถือเป็นแหล่งข้อมูลสูงสุดของโครงการ |
| **อรรถกถา** | คัมภีร์ที่อธิบายขยายความพระไตรปิฎก อยู่ในลำดับชั้นความน่าเชื่อถือรองจากพระไตรปิฎก |
| **ฎีกา** | คัมภีร์ที่อธิบายขยายความอรรถกถาอีกชั้นหนึ่ง อยู่ในลำดับถัดลงมา |
| **Source of Truth (แหล่งข้อมูลต้นทางเดียว)** | จุดอ้างอิงเดียวที่ถือเป็นต้นฉบับของข้อมูลแต่ละชิ้น |
| **ตรวจสอบย้อนกลับได้ (Traceability)** | คุณสมบัติที่ทำให้สามารถสืบย้อนเนื้อหากลับไปยังแหล่งข้อมูลต้นทางได้ |
| **Data Authority (ลำดับชั้นของแหล่งข้อมูล)** | การจัดลำดับความน่าเชื่อถือของข้อมูลตั้งแต่ระดับ A ถึง E โดยระดับสูงกว่ามีอำนาจเหนือระดับต่ำกว่า |
| **Data Lineage (สายธารของข้อมูล)** | เส้นทางของข้อมูลตั้งแต่แหล่งต้นทางจนถึงการนำเสนอ ที่ต้องสืบย้อนได้ครบทุกขั้น |
| **การอ้างอิง (Citation)** | การระบุแหล่งที่มาของเนื้อหาอย่างชัดเจนและตรวจสอบได้ |
| **ผู้ช่วยอัจฉริยะ (AI)** | ผู้ช่วยที่ทำหน้าที่ค้นหา อธิบาย และเชื่อมโยงเนื้อหาที่มีอยู่จริง โดยไม่สร้างหลักธรรมใหม่ และอยู่ในระดับต่ำสุดของลำดับชั้นแหล่งข้อมูล |
| **การค้นหา (Search)** | ระบบที่เป็นทางเข้าหลักและหัวใจของการเข้าถึงความรู้ในโครงการ |
| **Mobile First** | แนวทางการออกแบบที่ยึดอุปกรณ์เคลื่อนที่เป็นอันดับแรก |
| **การแยกข้อมูลออกจากการแสดงผล** | หลักการที่กำหนดให้เนื้อหาเป็นอิสระจากรูปแบบการนำเสนอ |
