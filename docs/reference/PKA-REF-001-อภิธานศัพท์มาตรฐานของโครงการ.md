## 1. วัตถุประสงค์

เอกสารฉบับนี้กำหนด **มาตรฐานของคำศัพท์ทั้งหมดที่ใช้ในโครงการ** เพื่อให้ทุกส่วน — การค้นหา (Search) ผู้ช่วยอัจฉริยะ (AI) การแสดงผลเชิงภาพ (Visualization) และองค์ความรู้ (Knowledge) — **ใช้คำชุดเดียวกัน** อย่างสม่ำเสมอ

อภิธานศัพท์นี้เป็น **ภาษากลางของโครงการ** ที่ทำให้การเรียกชื่อสิ่งต่าง ๆ เป็นมาตรฐานเดียว ลดความสับสนจากการใช้คำต่างกันเรียกสิ่งเดียวกัน

> เอกสารฉบับนี้กำหนด *มาตรฐานของคำ* ไม่ใช่ *ความหมายเชิงพระธรรมของคำ* — การอธิบายเนื้อหาและหลักธรรมเป็นเรื่องของชั้นองค์ความรู้ (Knowledge) ในอนาคต

---

## 2. ขอบเขต

### 2.1 อยู่ในขอบเขต

- หลักการและประเภทของคำศัพท์ในระบบ
- มาตรฐานของชื่อ (ชื่อมาตรฐาน ชื่อบาลี ชื่อภาษาอังกฤษ ชื่อเรียกอื่น)
- การสะกดที่ยอมรับ คำต้องห้าม และการกำหนดรหัสคำศัพท์
- วงจรชีวิต คุณภาพ และการอนุมัติคำศัพท์

### 2.2 อยู่นอกขอบเขต

- คำนิยามเชิงพระธรรมหรือพระอภิธรรมของคำใด ๆ
- การอธิบายเนื้อหา หลักธรรม หรือความหมายขององค์ธรรม
- รายการคำศัพท์จริง (instance) — เอกสารนี้กำหนด *มาตรฐาน* ไม่ใช่ *รายการคำ*
- การออกแบบฐานข้อมูล รูปแบบไฟล์ หรือเทคโนโลยีใด ๆ

---

## 3. หลักการ

1. **หนึ่งความหมาย หนึ่งชื่อมาตรฐาน** — แต่ละคำศัพท์มีชื่อมาตรฐานภาษาไทยเพียงหนึ่งชื่อ
2. **ภาษาไทยเป็นภาษาหลัก** — ชื่อมาตรฐานเป็นภาษาไทยเสมอ ตามเอกสารแม่
3. **ใช้คำชุดเดียวกันทั้งโครงการ** — Search, AI, Visualization และ Knowledge อ้างอิงคำชุดเดียว
4. **ตรวจสอบย้อนกลับได้** — คำศัพท์ที่อ้างถึงแหล่งที่มาต้องสืบย้อนได้
5. **ไม่ตัดสินความหมายเชิงพระธรรม** — อภิธานศัพท์กำหนดมาตรฐานของคำ ไม่ใช่ผู้ชี้ขาดความหมาย
6. **คงเส้นคงวาและขยายได้** — มาตรฐานต้องสม่ำเสมอ และเปิดให้เพิ่มคำใหม่อย่างมีระเบียบ

---

## 4. ประเภทของคำศัพท์

จำแนกประเภทเชิงมาตรฐาน (ยังไม่ระบุคำจริง)

| ประเภท | คำอธิบาย |
|---|---|
| **คำศัพท์เนื้อหา** (Content Term) | คำที่ใช้เรียกสิ่งที่ปรากฏในเนื้อหาคัมภีร์ (กำหนดเฉพาะมาตรฐานของชื่อ ไม่อธิบายความหมาย) |
| **คำศัพท์โครงสร้าง** (Structural Term) | คำที่ใช้เรียกหมวดหมู่ ลำดับชั้น หรือโครงสร้าง |
| **คำศัพท์ระบบ** (System Term) | คำที่ใช้ภายในโครงการ เช่น ชื่อชั้นเอกสารหรือสถานะ |
| **คำศัพท์ชื่อเฉพาะ** (Proper-name Term) | คำที่ใช้เรียกชื่อเฉพาะ เช่น ชื่อคัมภีร์ |

ทุกประเภทอยู่ภายใต้มาตรฐานการตั้งชื่อเดียวกันในเอกสารฉบับนี้

---

## 5. ชื่อมาตรฐาน

1. คำศัพท์แต่ละคำมี **ชื่อมาตรฐานภาษาไทยเพียงหนึ่งชื่อ** (Canonical Name)
2. ชื่อมาตรฐานต้อง **ชัดเจน ไม่กำกวม** และใช้อ้างอิงเป็นทางการทั้งโครงการ
3. ชื่ออื่น ๆ (บาลี อังกฤษ ชื่อเรียกอื่น) เป็น **ชื่อรอง** ไม่ใช่ชื่อมาตรฐาน
4. การเปลี่ยนชื่อมาตรฐานต้องมีร่องรอยและเหตุผลกำกับ
5. ชื่อมาตรฐานสอดคล้องกับหลักการตั้งชื่อใน [PKA-DOM-002 §10](../domain/PKA-DOM-002-มาตรฐานโครงสร้างองค์ความรู้.md)

---

## 6. ชื่อบาลี

1. คำศัพท์อาจมี **ชื่อบาลี** เป็นชื่อรอง เพื่อรองรับการศึกษาที่อิงภาษาต้นทาง
2. ชื่อบาลีต้องระบุได้ว่าเป็นภาษาบาลี และแยกจากชื่อมาตรฐานภาษาไทยอย่างชัดเจน
3. ชื่อบาลีเป็นข้อมูลของคำศัพท์ ไม่ใช่คำอธิบายความหมาย
4. การสะกดชื่อบาลีให้เป็นไปตาม §9 (การสะกดที่ยอมรับ)

---

## 7. ชื่อภาษาอังกฤษ

1. คำศัพท์อาจมี **ชื่อภาษาอังกฤษ** เป็นชื่อรอง เพื่อช่วยผู้ศึกษาที่ใช้ภาษาอังกฤษ
2. ชื่อภาษาอังกฤษต้องไม่แทนที่ชื่อมาตรฐานภาษาไทย
3. ชื่อภาษาอังกฤษเป็นข้อมูลของคำศัพท์ ไม่ใช่คำนิยาม
4. หากมีหลายตัวเลือก ให้กำหนดตัวเลือกที่เป็นมาตรฐานหนึ่งเดียว และจัดที่เหลือเป็นชื่อเรียกอื่น

---

## 8. ชื่อเรียกอื่น

1. **ชื่อเรียกอื่น (Alias)** ทุกชื่อต้อง **เชื่อมโยงกลับไปยังชื่อมาตรฐาน** เสมอ
2. ชื่อเรียกอื่นใช้เพื่อ **ช่วยการค้นหา** ไม่ใช่เพื่อสร้างคำศัพท์ซ้ำ
3. ชื่อเรียกอื่นแต่ละชื่อควรระบุที่มาได้
4. ชื่อเรียกอื่นต้องไม่ทำให้สับสนกับชื่อมาตรฐานของคำศัพท์อื่น

---

## 9. การสะกดที่ยอมรับ

1. แต่ละคำศัพท์ต้องกำหนด **การสะกดมาตรฐาน** หนึ่งแบบ
2. การสะกดอื่นที่พบบ่อยอาจถูกบันทึกเป็น **การสะกดที่ยอมรับ** เพื่อรองรับการค้นหา โดยชี้กลับไปยังการสะกดมาตรฐาน
3. การสะกดที่ยอมรับไม่ใช่ชื่อมาตรฐานใหม่ แต่เป็นรูปแปรของการเขียน
4. การกำหนดการสะกดต้องคงเส้นคงวาทั้งโครงการ

---

## 10. คำต้องห้าม

1. **คำต้องห้าม (Forbidden Term)** คือคำที่ **ไม่อนุญาตให้ใช้** ในระบบ เช่น คำกำกวม คำที่ทำให้เข้าใจผิด หรือคำที่เลิกใช้แล้ว
2. คำต้องห้ามแต่ละคำควรระบุ **คำมาตรฐานที่ใช้แทน** ได้
3. ระบบต้องไม่นำคำต้องห้ามมาใช้เป็นชื่อมาตรฐานหรือชื่อรอง
4. การกำหนดคำต้องห้ามต้องมีเหตุผลและร่องรอยกำกับ

---

## 11. การกำหนดรหัสคำศัพท์

1. คำศัพท์แต่ละคำมี **รหัสประจำตัวเฉพาะ ไม่ซ้ำกัน** ทั้งระบบ
2. รหัสต้อง **คงที่** ตลอดอายุของคำศัพท์ และ **ห้ามนำกลับมาใช้ซ้ำ**
3. รหัสควร **เป็นกลางต่อความหมาย** ไม่ผูกกับการสะกดหรือภาษาใดภาษาหนึ่ง
4. หลักการกำหนดรหัสสอดคล้องกับ [PKA-STD-002 §11](../standards/PKA-STD-002-มาตรฐานข้อมูล.md) — เอกสารนี้กำหนดเพียงหลักการ

## 12. วงจรชีวิตของคำศัพท์

```
   ร่าง → กำลังตรวจสอบ → อนุมัติ → เผยแพร่ → ปรับปรุง → เลิกใช้
```

| สถานะ | ความหมาย |
|---|---|
| **ร่าง (Draft)** | เสนอคำศัพท์ขั้นต้น ยังไม่ตรวจสอบ |
| **กำลังตรวจสอบ (In Review)** | อยู่ระหว่างทบทวนชื่อ การสะกด และความสม่ำเสมอ |
| **อนุมัติ (Approved)** | ผ่านการตรวจสอบและได้รับอนุมัติ แต่ยังไม่เปิดใช้ทั่วโครงการ |
| **เผยแพร่ (Published)** | เปิดให้ทุกส่วนของโครงการใช้เป็นคำมาตรฐาน |
| **ปรับปรุง (Revised)** | ถูกแก้ไข เกิดเวอร์ชันใหม่ พร้อมเหตุผลและผู้อนุมัติ |
| **เลิกใช้ (Retired)** | ไม่ใช้งานแล้ว แต่คงไว้เพื่อร่องรอย ไม่ลบทิ้ง |

วงจรชีวิตนี้สอดคล้องกับ [PKA-STD-002 §7](../standards/PKA-STD-002-มาตรฐานข้อมูล.md)

## 13. คุณภาพของคำศัพท์

ก่อนเลื่อนคำศัพท์สู่ "อนุมัติ" และ "เผยแพร่" ให้ตรวจตาม **Checklist คุณภาพคำศัพท์**

- ☑ **มีชื่อมาตรฐาน** — มีชื่อมาตรฐานภาษาไทยที่ชัดเจน
- ☑ **มีรหัส** — มีรหัสประจำตัวเฉพาะที่ไม่ซ้ำ
- ☑ **ไม่กำกวม** — ไม่ทับซ้อนหรือสับสนกับคำศัพท์อื่น
- ☑ **การสะกดมาตรฐานชัดเจน** — กำหนดการสะกดมาตรฐานแล้ว
- ☑ **ชื่อรองเชื่อมกลับชื่อมาตรฐาน** — ชื่อบาลี/อังกฤษ/ชื่อเรียกอื่นชี้กลับชื่อมาตรฐาน
- ☑ **ไม่ใช้คำต้องห้าม** — ไม่นำคำต้องห้ามมาใช้
- ☑ **ผ่านการอนุมัติ** — มีผู้อนุมัติและร่องรอย

---

## 14. การอนุมัติคำศัพท์

1. คำศัพท์จะ **เผยแพร่** ได้ก็ต่อเมื่อผ่านการตรวจสอบคุณภาพและได้รับอนุมัติ
2. การอนุมัติต้องมี **ผู้อนุมัติที่ระบุได้** และร่องรอยเวลา
3. **ผู้ช่วยอัจฉริยะไม่มีอำนาจอนุมัติ** คำศัพท์ ตามเอกสารแม่
4. หลักการอนุมัติสอดคล้องกับ [PKA-STD-002 §18](../standards/PKA-STD-002-มาตรฐานข้อมูล.md)

---

## 15. สิ่งที่ยังไม่กำหนด

- รายการคำศัพท์จริง (instance) ทั้งหมด
- คำนิยามเชิงพระธรรมหรือความหมายขององค์ธรรมใด ๆ
- รูปแบบรหัสคำศัพท์เชิงเทคนิคและกฎการสร้างรหัสจริง
- การออกแบบฐานข้อมูล รูปแบบไฟล์ หรือเทคโนโลยี
- กระบวนการตรวจสอบและอนุมัติเชิงปฏิบัติโดยละเอียด

| **ระยะที่ 1** | วางมาตรฐานของคำศัพท์ (เอกสารฉบับนี้) |
| **ระยะที่ 2** | ประสานอภิธานศัพท์กับชั้นเชิงความหมาย ([PKA-ARC-004](../architecture/PKA-ARC-004-สถาปัตยกรรมเชิงความหมาย.md)) |
| **ระยะที่ 3** | เริ่มลงทะเบียนคำศัพท์จริงตามมาตรฐานนี้ (ในชั้นที่เกี่ยวข้อง) |
| **ระยะที่ 4** | ขยายอภิธานศัพท์ให้รองรับพระไตรปิฎก อรรถกถา ฎีกา และคัมภีร์อื่น |

อภิธานศัพท์นี้ออกแบบให้เป็น **รากฐานของ Knowledge, Search และ AI ในอนาคต** โดยทุกส่วนใช้คำชุดเดียวกัน
