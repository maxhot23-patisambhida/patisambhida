## 1. วัตถุประสงค์

เอกสารฉบับนี้อธิบายว่า **ตระกูลเอกสารทั้งหมดของโครงการเชื่อมโยงกันอย่างไร** ได้แก่ STD, ARC, DOM, REF, KNW และ ADR เพื่อให้ทุกฝ่ายเข้าใจว่าเอกสารแต่ละตระกูลทำหน้าที่อะไร อยู่ในลำดับชั้นใด และพึ่งพากันอย่างไร

## 2. ขอบเขต

**อยู่ในขอบเขต:** ความเชื่อมโยงเชิงโครงสร้างระหว่างตระกูลเอกสาร และกฎของความเชื่อมโยง

**อยู่นอกขอบเขต:** เนื้อหาภายในเอกสารแต่ละฉบับ, วิธีพัฒนา, และเทคโนโลยีใด ๆ

## 3. ตระกูลเอกสารในโครงการ

| ตระกูล | ชื่อเต็ม | หน้าที่ | ระดับ |
|---|---|---|---|
| **STD** | Standards (มาตรฐาน) | กำหนดวิสัยทัศน์ (STD-001) และกฎกลาง (STD-002+) | 0–1 |
| **ARC** | Architecture (สถาปัตยกรรม) | อธิบายภาพรวมและการเชื่อมโยง | 2 |
| **DOM** | Domain (แบบจำลองโดเมน) | กำหนดแบบจำลององค์ความรู้ | 3 |
| **REF** | Reference (แหล่งอ้างอิง) | อธิบายคัมภีร์และแหล่งอ้างอิงต้นทาง | 4 |
| **KNW** | Knowledge (องค์ความรู้) | เก็บองค์ความรู้จริง | 5 |
| **ADR** | Decision Records (บันทึกการตัดสินใจ) | บันทึกเหตุผลของการตัดสินใจ | 6 |

## 4. แผนภาพความเชื่อมโยงของตระกูลเอกสาร

                └──────────────┬──────────────┘
                               │

                ┌─────────────────────────────┐

     │  ARC :       │ │  DOM :       │ │  REF :       │
     │ สถาปัตยกรรม   │ │ แบบจำลองโดเมน │ │ แหล่งอ้างอิง   │
     └──────┬───────┘ └──────┬───────┘ └──────┬───────┘
            │                │                │
            └────────────────┼────────────────┘
                             ▼
                  ┌──────────────────────┐
                  │  KNW : องค์ความรู้จริง │
                  └──────────┬───────────┘
                             │
                             ▼

> ทิศของลูกศรคือทิศของการ **อยู่ภายใต้และอ้างอิง** — ตระกูลปลายลูกศรอยู่ภายใต้ตระกูลต้นลูกศร

## 5. อธิบายความเชื่อมโยงแต่ละตระกูล

- **STD → ทุกตระกูล** — วิสัยทัศน์และมาตรฐานเป็นต้นทางของหลักการ ทุกตระกูลต้องอ้างอิงและไม่ขัดแย้ง
- **ARC อยู่ภายใต้ STD** — สถาปัตยกรรมอธิบายภาพรวมตามกรอบของมาตรฐาน โดยไม่กำหนดกฎเอง
- **DOM อยู่ภายใต้ STD และสอดคล้องกับ ARC** — แบบจำลองโดเมนกำหนดโครงสร้างองค์ความรู้ตามมาตรฐานและภายในภาพรวมของสถาปัตยกรรม
- **REF อยู่ภายใต้ STD** — แหล่งอ้างอิงอธิบายคัมภีร์ต้นทางตามลำดับชั้นแหล่งข้อมูล
- **KNW อยู่ภายใต้ DOM และ REF** — องค์ความรู้จริงถูกจัดโครงสร้างตามแบบจำลองโดเมน และอ้างอิงแหล่งอ้างอิง
- **ADR ขวางทุกชั้น** — บันทึกการตัดสินใจอธิบายเหตุผลของการเลือกแนวทางในตระกูลใดก็ได้

## 6. กฎของความเชื่อมโยง

1. **เอกสารทุกฉบับอ้างอิง STD-001** เป็นเอกสารแม่
2. **ระดับล่างต้องไม่ขัดกับระดับบน** — เมื่อขัดแย้ง ให้ยึดระดับที่สูงกว่า
3. **แต่ละตระกูลคงหน้าที่ของตน** — STD ไม่อธิบายภาพรวม, ARC ไม่กำหนดกฎ, DOM ไม่เก็บเนื้อหาจริง
4. **มาตรฐานและแบบจำลองโดเมนต้องไม่ถูกย้ายเข้า ARC**
5. **การเปลี่ยนแปลงระดับบนต้องทบทวนผลกระทบต่อระดับล่าง** ทุกครั้ง

## 7. สิ่งที่ยังไม่กำหนด

- โครงสร้างภายในของตระกูล KNW และ REF ในเชิงปฏิบัติ
- รหัสและการตั้งชื่อของเอกสารในตระกูลที่ยังไม่เริ่ม
- วิธีพัฒนาและเทคโนโลยี

## 8. แผนพัฒนา

ตระกูล REF และ KNW จะได้รับการจัดโครงสร้างเมื่อเข้าสู่เฟสที่เกี่ยวข้อง โดยไม่ต้องเปลี่ยนสถาปัตยกรรมเอกสารหลัก
