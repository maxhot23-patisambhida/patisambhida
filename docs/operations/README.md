# operations/ — เอกสารปฏิบัติการ (PKA-OPS)

โฟลเดอร์นี้บรรจุเอกสารตระกูล **ปฏิบัติการ (`PKA-OPS-*`)** ซึ่งเป็น **ชั้นปฏิบัติการ (Operational Layer)** ของโครงการ — กำหนดว่า **องค์กรของโครงการจัดระเบียบและดำเนินงานอย่างไร** เพื่อให้หลักการในเอกสารระดับสูงกว่าถูกนำไปปฏิบัติได้จริงโดยมีเจ้าภาพและสายความรับผิดชอบที่ชัดเจน

## หน้าที่ของตระกูล OPS

- ตระกูล OPS อธิบาย **"องค์กรและการดำเนินงาน"** — โครงสร้างองค์กร สำนักงาน หน้าที่ การประสานงาน สิทธิ์การอนุมัติ และความรับผิดชอบ
- ตระกูล OPS เป็น **กฎบัตรและหลักการระดับองค์กร** **ไม่ใช่ Workflow ที่ปฏิบัติได้ทันที ไม่ใช่ชุดคำสั่ง และไม่กล่าวถึงเทคโนโลยีใด ๆ**

## ความสัมพันธ์กับ STD / ARC / DOM / REF

- **เป็นชั้นปฏิบัติการที่อยู่คู่กับ** มาตรฐาน (STD) สถาปัตยกรรม (ARC) แบบจำลองโดเมน (DOM) และแหล่งอ้างอิง (REF) โดย **ไม่แทนที่เอกสารใด**
- **อยู่ใต้ STD** — องค์กรดำเนินงานภายใต้กฎที่ตระกูล STD กำหนด และต้องไม่ขัดกับเอกสารแม่
- **เคารพ ARC และ DOM** — การดำเนินงานต้องสอดคล้องกับภาพรวมของระบบ (ARC) และแบบจำลองขององค์ความรู้ (DOM)
- ทุกฉบับต้องอ้างอิง [PKA-STD-001 (เอกสารแม่)](../standards/PKA-STD-001-ธรรมนูญโครงการ.md)
