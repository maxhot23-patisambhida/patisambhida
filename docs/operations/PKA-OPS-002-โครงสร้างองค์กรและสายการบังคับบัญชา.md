## 1. วัตถุประสงค์ (Purpose)

เอกสารฉบับนี้กำหนด **โครงสร้างองค์กรอย่างเป็นทางการ** และ **สายการบังคับบัญชา** ของสำนักงานองค์ความรู้ดิจิทัล (Digital Knowledge Office) ที่จัดตั้งขึ้นใน [PKA-OPS-001](PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md) โดยลงรายละเอียดว่าระดับต่าง ๆ ขององค์กร — ผู้บริหารงานองค์กร สำนักงาน ตัวแทนงาน และหน่วยปฏิบัติ — สัมพันธ์กันอย่างไร ใครสั่งใคร ใครรายงานต่อใคร และองค์กรเติบโตได้อย่างไรโดยไม่เสียรูป

เอกสารนี้มีไว้เพื่อ

1. แสดง **แผนผังองค์กร** ของสำนักงานองค์ความรู้ดิจิทัลให้เห็นเป็นภาพเดียวกันทั้งโครงการ
2. กำหนด **สายการบังคับบัญชา (Chain of Command)** ที่ชัดเจน — คำสั่งไหลลง การรายงานไหลขึ้น และการอนุมัติเป็นของมนุษย์
3. กำหนด **ความสัมพันธ์เชิงโครงสร้าง** ระหว่างสำนักงาน ตัวแทนงาน และหน่วยปฏิบัติ
4. วางหลักว่า **โครงสร้างองค์กรเป็นสิ่งถาวร** ในขณะที่ตัวแทนงานและหน่วยปฏิบัติเป็นกำลังพลที่ทดแทนได้
5. กำหนด **หลักการเติบโต** ที่ให้องค์กรเพิ่มสำนักงาน ตัวแทนงาน และหน่วยปฏิบัติได้ โดยไม่ต้องเปลี่ยนโครงสร้าง

> เมื่ออ่านเอกสารฉบับนี้จบ ผู้อ่านควรเข้าใจ **โครงสร้างองค์กร สายการบังคับบัญชา และความสัมพันธ์ของสำนักงาน ตัวแทนงาน และหน่วยปฏิบัติ** ได้ครบถ้วน โดยไม่จำเป็นต้องรู้กระแสงาน (Workflow) หรือเรื่องเทคนิคใด ๆ

## 2. ขอบเขต (Scope)

**อยู่ในขอบเขต**

- โครงสร้างองค์กรเชิงลำดับชั้น และระดับการบังคับบัญชา
- บทบาทเชิงโครงสร้างของผู้บริหารงานองค์กร สำนักงาน ตัวแทนงาน และหน่วยปฏิบัติ
- ความสัมพันธ์ระหว่างระดับ — การมอบหมาย การรายงาน ช่วงการควบคุม และการประสานงาน
- หลักการจัดตั้งสำนักงานใหม่ และการเพิ่มตัวแทนงาน/หน่วยปฏิบัติ
- หลักการ "องค์กรถาวร กำลังพลทดแทนได้"

**อยู่นอกขอบเขต**

- หน้าที่โดยละเอียดของแต่ละสำนักงาน (จะกำหนดใน **PKA-OPS-004 — Office Catalog**)
- การออกแบบตัวแทนงานแต่ละตัว ความสามารถ หรือชุดคำสั่ง
- กระแสงานเชิงปฏิบัติและขั้นตอนการทำงาน
- เครื่องมือ เทคโนโลยี รูปแบบข้อมูล หรือรายละเอียดเชิงปฏิบัติเชิงเทคนิคใด ๆ

> เอกสารฉบับนี้ตอบเฉพาะคำถามว่า **"องค์กรมีรูปร่างอย่างไร และใครขึ้นกับใคร"** ส่วนคำถามว่า **"แต่ละสำนักงานทำอะไรโดยละเอียด"** เป็นหน้าที่ของ PKA-OPS-004 (Office Catalog) ต่อไป

## 3. หลักการออกแบบองค์กร (Organization Design Principles)

โครงสร้างองค์กรนี้ออกแบบบนหลักการพื้นฐานอย่างน้อยสิบประการ ซึ่งใช้เป็นเกณฑ์ตัดสินทุกการจัดและการเปลี่ยนแปลงองค์กร

1. **แยกหน้าที่ให้ชัด (Separation of Duties)** — แต่ละระดับและแต่ละสำนักงานมีหน้าที่ของตนที่ไม่ทับซ้อนและไม่มีช่องว่าง
2. **ขยายได้ (Scalable)** — องค์กรต้องเพิ่มสำนักงาน ตัวแทนงาน และหน่วยปฏิบัติได้ โดยไม่ต้องรื้อโครงสร้างเดิม
3. **ไม่ผูกกับผู้ช่วยอัจฉริยะ (AI-Agnostic)** — โครงสร้างองค์กรต้องไม่ขึ้นกับผู้ช่วยอัจฉริยะรุ่นใดรุ่นหนึ่ง เปลี่ยนผู้ช่วยอัจฉริยะได้โดยองค์กรไม่กระทบ
4. **ไม่ผูกกับบุคคล (Person-Independent)** — โครงสร้างต้องไม่ขึ้นกับบุคคลใดบุคคลหนึ่ง ตำแหน่งคงอยู่แม้ผู้ปฏิบัติเปลี่ยน
5. **ตรวจสอบได้ (Auditable)** — ทุกสายการบังคับบัญชาและการมอบหมายต้องสืบย้อนและตรวจสอบได้
6. **กำกับดูแลได้ (Governable)** — ทุกระดับอยู่ภายใต้การกำกับที่ชัดเจน ไม่มีส่วนใดอยู่นอกการควบคุม
7. **เปลี่ยนหน่วยปฏิบัติได้ (Replaceable Workers)** — หน่วยปฏิบัติเปลี่ยนได้ตลอดเวลาโดยไม่กระทบโครงสร้าง
8. **เปลี่ยนตัวแทนงานได้ (Replaceable Agents)** — ตัวแทนงานเพิ่ม ลด หรือเปลี่ยนได้ภายใต้สำนักงานเดิม
9. **เปลี่ยนผู้ช่วยอัจฉริยะได้ (Replaceable AI)** — ผู้ช่วยอัจฉริยะเปลี่ยนรุ่นได้ โดยหน้าที่ขององค์กรไม่เปลี่ยน
10. **ไม่เปลี่ยนโครงสร้าง (Stable Structure)** — แม้ทุกองค์ประกอบข้างต้นเปลี่ยนได้ แต่ **โครงสร้างองค์กรต้องคงเดิม** เป็นหลักยึดที่มั่นคงที่สุด
11. **มนุษย์รับผิดชอบสูงสุด (Human Accountability)** — อำนาจอนุมัติองค์ความรู้และการตัดสินที่กระทบความถูกต้องของพระธรรมเป็นของมนุษย์เสมอ
12. **ประสานผ่านเหตุการณ์ (Event-Coordinated)** — สำนักงานไม่เรียกกันโดยตรง แต่ประสานผ่านเหตุการณ์และผู้บริหารงานองค์กร

## 4. ปรัชญาขององค์กร (Organizational Philosophy)

องค์กรนี้ตั้งอยู่บนความเชื่อว่า **ความมั่นคงของโครงสร้างคือหลักประกันความถูกต้องในระยะยาว** เมื่อโครงสร้างองค์กรชัดเจนและไม่สั่นคลอน ทุกงานย่อมมีเจ้าภาพ ทุกการตัดสินใจย่อมมีผู้รับผิดชอบ และความถูกต้องของพระธรรมย่อมไม่หล่นหายระหว่างการเปลี่ยนผ่านของผู้ปฏิบัติหรือเครื่องมือ

ปรัชญาหลักสามประการ

- **โครงสร้างยืนยง ผู้ปฏิบัติหมุนเวียน** — องค์กรออกแบบให้ "ตำแหน่ง" และ "หน้าที่" คงอยู่ ส่วน "ผู้ทำงาน" ไม่ว่าจะเป็นมนุษย์หรือผู้ช่วยอัจฉริยะ เป็นสิ่งที่หมุนเวียนและทดแทนได้ องค์กรจึงไม่ล่มสลายเมื่อผู้ปฏิบัติเปลี่ยน
- **องค์กรนำเครื่องมือ** — โครงสร้างองค์กรมาก่อนและเป็นอิสระจากเครื่องมือใด ๆ เครื่องมือรับใช้หน้าที่ขององค์กร ไม่ใช่กำหนดรูปขององค์กร
- **มนุษย์เป็นศูนย์กลางของความรับผิดชอบ** — ไม่ว่าองค์กรจะมีผู้ช่วยอัจฉริยะมากเพียงใด ความรับผิดชอบสูงสุดต่อความถูกต้องของพระธรรมยังเป็นของมนุษย์เสมอ

## 5. โครงสร้างองค์กร (Organizational Structure)

โครงสร้างองค์กรของสำนักงานองค์ความรู้ดิจิทัลมีห้าระดับ เรียงจากบนสุดลงล่าง

```
            ┌─────────────────────────────────────┐
            │        Executive Director           │
            │      (ผู้อำนวยการสูงสุด — มนุษย์)      │
            └─────────────────────────────────────┘
                              │   กำหนดเจตจำนงและอนุมัติสูงสุด
                              ▼
            ┌─────────────────────────────────────┐
            │     Chief Knowledge Orchestrator    │
            │        (ผู้บริหารงานองค์กร)           │
            └─────────────────────────────────────┘
                              │   กำกับและประสานงานทั้งองค์กร
                              ▼
            ┌─────────────────────────────────────┐
            │              Office                 │
            │            (สำนักงาน)                │
            └─────────────────────────────────────┘
                              │   รับผิดชอบงานหนึ่งด้าน
                              ▼
            ┌─────────────────────────────────────┐
            │              Agent                  │
            │            (ตัวแทนงาน)               │
            └─────────────────────────────────────┘
                              │   รับผิดชอบภารกิจหนึ่ง
                              ▼
            ┌─────────────────────────────────────┐
            │              Worker                 │
            │           (หน่วยปฏิบัติ)             │
            └─────────────────────────────────────┘
                                  ลงมือทำงานหนึ่งหน่วย
```

**คำอธิบายแต่ละระดับ**

| ระดับ | องค์ประกอบ | ผู้ปฏิบัติ | หน้าที่เชิงโครงสร้าง |
|---|---|---|---|
| 1 | **ผู้อำนวยการสูงสุด** (Executive Director) | มนุษย์ | กำหนดเจตจำนงสูงสุดขององค์กร และเป็นผู้ถืออำนาจอนุมัติองค์ความรู้สูงสุด |
| 2 | **ผู้บริหารงานองค์กร** (Chief Knowledge Orchestrator) | มนุษย์หรือบทบาทประสานงาน | กำกับ ประสาน และติดตามงานของสำนักงานทั้งหมด |
| 3 | **สำนักงาน** (Office) | โครงสร้างถาวร | รับผิดชอบงานหนึ่งด้านขององค์กรอย่างเต็มตัว |
| 4 | **ตัวแทนงาน** (Agent) | กำลังพลทดแทนได้ | รับผิดชอบภารกิจเฉพาะหนึ่งภายในสำนักงาน |
| 5 | **หน่วยปฏิบัติ** (Worker) | กำลังพลทดแทนได้ | ลงมือทำงานหนึ่งหน่วยที่เล็กที่สุด |

> **ผู้อำนวยการสูงสุด (Executive Director)** เป็นมนุษย์ที่อยู่เหนือสุดของสายการบังคับบัญชา และเป็นผู้รับผิดชอบสูงสุดต่อองค์กร ส่วนผู้บริหารงานองค์กรเป็นผู้ดูแลการดำเนินงานประจำวันแทน แต่ **ไม่มีอำนาจอนุมัติองค์ความรู้แทนมนุษย์** ตามหลักการในเอกสารแม่และ PKA-OPS-001

## 6. ระดับการบังคับบัญชา (Command Levels)

แต่ละระดับขององค์กรรับผิดชอบต่อระดับที่อยู่เหนือตน และกำกับระดับที่อยู่ใต้ตน

1. **ผู้อำนวยการสูงสุด** กำหนดเจตจำนงและถืออำนาจอนุมัติสูงสุด ไม่ลงไปจัดการงานประจำวัน
2. **ผู้บริหารงานองค์กร** รับเจตจำนงจากผู้อำนวยการสูงสุด แล้วแปรเป็นการกำกับสำนักงาน — จัดลำดับงาน ติดตาม ประสาน แต่ไม่สร้างหรืออนุมัติองค์ความรู้
3. **สำนักงาน** รับการกำกับจากผู้บริหารงานองค์กรผ่านเหตุการณ์ และกำกับตัวแทนงานภายในของตน
4. **ตัวแทนงาน** รับภารกิจจากสำนักงานของตน และมอบหมายงานย่อยให้หน่วยปฏิบัติ
5. **หน่วยปฏิบัติ** รับงานจากตัวแทนงาน ลงมือทำ แล้วส่งผลกลับขึ้นไปตามสาย

**กฎของระดับการบังคับบัญชา**

- ไม่มีระดับใดข้ามสายไปสั่งงานระดับที่ไม่ได้อยู่ใต้ตนโดยตรง
- งานข้ามสำนักงานต้องผ่านเหตุการณ์และผู้บริหารงานองค์กรเสมอ
- การอนุมัติองค์ความรู้ไม่อยู่ในสายการบังคับบัญชาของผู้ช่วยอัจฉริยะ แต่สงวนไว้สำหรับมนุษย์

## 7. สายการบังคับบัญชา (Chain of Command)

สายการบังคับบัญชาขององค์กรนี้ยึดหลักสามทิศทางที่ชัดเจนและแยกจากกัน

```
        คำสั่ง (Command)          การรายงาน (Reporting)
              │                          ▲
              ▼                          │
   Executive Director ───────────────────┘
   Chief Knowledge Orchestrator
   Office
   Agent
   Worker

        การอนุมัติองค์ความรู้ (Approval) ──► มนุษย์เท่านั้น
```

1. **คำสั่งไหลลง (Command flows down)** — เจตจำนงและการมอบหมายไหลจากบนลงล่าง: ผู้อำนวยการสูงสุด → ผู้บริหารงานองค์กร → สำนักงาน → ตัวแทนงาน → หน่วยปฏิบัติ
2. **การรายงานไหลขึ้น (Reporting flows up)** — สถานะ ผลงาน และปัญหาไหลจากล่างขึ้นบนตามสายเดียวกันในทิศย้อนกลับ
3. **การอนุมัติเป็นของมนุษย์ (Approval is human-only)** — การอนุมัติองค์ความรู้เพื่อเผยแพร่ไม่ไหลตามสายการบังคับบัญชาของผู้ช่วยอัจฉริยะ แต่เป็นอำนาจของมนุษย์เสมอ สอดคล้องกับหลักการ Human in the Loop ใน [PKA-OPS-001](PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md)

> **หลักสำคัญ:** คำสั่งและการรายงานเป็นเรื่องของการดำเนินงาน จึงไหลตามสายได้ทั้งสองทิศ แต่ **การอนุมัติองค์ความรู้เป็นเรื่องของความถูกต้องของพระธรรม** จึงไม่มีวันมอบให้องค์ประกอบที่ไม่ใช่มนุษย์

## 8. Chief Knowledge Orchestrator — ผู้บริหารงานองค์กร

**ผู้บริหารงานองค์กร (Chief Knowledge Orchestrator)** อยู่ระดับที่สองของสายการบังคับบัญชา ใต้ผู้อำนวยการสูงสุด และเหนือสำนักงานทั้งหมด ทำหน้าที่เป็นศูนย์กลางการประสานงานขององค์กร

- **กำกับสำนักงานทั้งหมด** — ดูแลให้ทุกสำนักงานทำงานสอดประสานกัน แต่ไม่ลงไปทำงานขององค์ความรู้เอง
- **เป็นศูนย์กลางการประสานงาน** — การประสานข้ามสำนักงานทั้งหมดผ่านผู้บริหารงานองค์กรและเหตุการณ์ ไม่ใช่การที่สำนักงานเรียกกันเอง
- **ไม่มีอำนาจอนุมัติองค์ความรู้** — บทบาทคือบริหารการไหลของงาน ไม่ใช่ตัดสินเนื้อหาพระธรรม

> รายละเอียดบทบาทของผู้บริหารงานองค์กรกำหนดไว้แล้วใน [PKA-OPS-001 หัวข้อ 11](PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md#11-chief-knowledge-orchestrator--ผู้บริหารงานองค์กร) เอกสารฉบับนี้ระบุเพียง **ตำแหน่งเชิงโครงสร้าง** ของผู้บริหารงานองค์กรในสายการบังคับบัญชา

## 9. สำนักงาน (Office)

**สำนักงาน (Office)** เป็นระดับที่สามของโครงสร้าง และเป็น **หน่วยโครงสร้างถาวร** ขององค์กร

- สำนักงานหนึ่งรับผิดชอบ **งานหนึ่งด้าน** ขององค์กรอย่างเต็มตัว
- ทุกสำนักงานมี **ภารกิจ (Mission) ขอบเขต (Scope) ปัจจัยนำเข้า (Input) และผลผลิต (Output)** ที่ชัดเจน — แต่ **รายละเอียดของทั้งสี่ด้านนี้จะกำหนดใน PKA-OPS-004 (Office Catalog)** เอกสารฉบับนี้ระบุเพียงว่าสำนักงานเป็นหน่วยโครงสร้างที่ต้องมีทั้งสี่ด้านนี้
- สำนักงานกำกับตัวแทนงานหลายตัวที่อยู่ภายในตน
- สำนักงานเป็นโครงสร้างถาวร — ไม่ถูกยุบหรือเปลี่ยนรูปง่าย ๆ ตามการเปลี่ยนของผู้ปฏิบัติหรือเครื่องมือ

> รายชื่อสำนักงานหลักทั้งหมดขององค์กรกำหนดไว้แล้วใน [PKA-OPS-001 หัวข้อ 9–10](PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md#9-สำนักงานทั้งหมด-the-offices)

## 10. ตัวแทนงาน (Agent)

**ตัวแทนงาน (Agent)** เป็นระดับที่สี่ และเป็น **กำลังพลที่ทดแทนได้** ภายในสำนักงาน

- ตัวแทนงานหนึ่งตัวรับผิดชอบ **ภารกิจเฉพาะหนึ่ง** ภายในสำนักงาน
- **ตัวแทนงานทุกตัวสังกัดสำนักงานเดียวเท่านั้น** — ไม่มีตัวแทนงานใดอยู่สองสำนักงานพร้อมกัน และไม่ข้ามไปทำงานของสำนักงานอื่น
- ตัวแทนงานรับภารกิจจากสำนักงาน และมอบหมายงานย่อยให้หน่วยปฏิบัติ
- ตัวแทนงานเพิ่ม ลด หรือเปลี่ยนได้ภายใต้สำนักงานเดิม โดยโครงสร้างของสำนักงานไม่เปลี่ยน

## 11. หน่วยปฏิบัติ (Worker)

**หน่วยปฏิบัติ (Worker)** เป็นระดับล่างสุด และเป็น **หน่วยที่เล็กที่สุด** ขององค์กร

- **หน่วยปฏิบัติทุกหน่วยมีหน้าที่เดียว (Single Responsibility)** — ลงมือทำงานหนึ่งหน่วยที่ชัดเจน ไม่รับภาระหลายอย่างปนกัน
- หน่วยปฏิบัติรับงานจากตัวแทนงานที่มอบหมาย และส่งผลกลับไปยังตัวแทนงานนั้น
- หน่วยปฏิบัติเปลี่ยนได้ตลอดเวลา โดยไม่กระทบโครงสร้างของตัวแทนงานหรือสำนักงาน

**แผนผังภายในสำนักงานหนึ่ง** — สำนักงานหนึ่งมีตัวแทนงานหลายตัว และตัวแทนงานแต่ละตัวมีหน่วยปฏิบัติหลายหน่วย

```
                    ┌─────────────────┐
                    │     Office      │
                    │   (สำนักงาน)     │
                    └─────────────────┘
                            │
            ┌───────────────┼───────────────┐
            ▼               ▼               ▼
       ┌─────────┐     ┌─────────┐     ┌─────────┐
       │  Agent  │     │  Agent  │     │  Agent  │
       │ (ตัวแทน) │     │ (ตัวแทน) │     │ (ตัวแทน) │
       └─────────┘     └─────────┘     └─────────┘
            │
      ┌─────┼─────┐
      ▼     ▼     ▼
  ┌──────┐┌──────┐┌──────┐
  │Worker││Worker││Worker│
  └──────┘└──────┘└──────┘
   (แต่ละ Agent มีหน่วยปฏิบัติได้หลายหน่วย)
```

> หลักการ **"หนึ่งสำนักงาน มีหลายตัวแทนงาน — หนึ่งตัวแทนงาน มีหลายหน่วยปฏิบัติ"** คือรูปแบบการขยายตัวเชิงลึกขององค์กร ที่เพิ่มกำลังพลได้โดยไม่เปลี่ยนรูปของโครงสร้าง

## 12. ความสัมพันธ์ระหว่างระดับ (Relationships Between Levels)

ความสัมพันธ์ระหว่างห้าระดับยึดหลัก **"ความรับผิดชอบไหลลง คุณภาพไหลขึ้น"**

| จากระดับ | สู่ระดับ | ความสัมพันธ์ |
|---|---|---|
| ผู้อำนวยการสูงสุด | ผู้บริหารงานองค์กร | มอบเจตจำนง — รับรายงานภาพรวม |
| ผู้บริหารงานองค์กร | สำนักงาน | กำกับและประสาน — รับสถานะงาน |
| สำนักงาน | ตัวแทนงาน | มอบภารกิจ — รับผลภารกิจ |
| ตัวแทนงาน | หน่วยปฏิบัติ | มอบงานย่อย — รับผลงานหน่วย |

- ความสัมพันธ์เป็นแบบ **ลำดับชั้น (Hierarchical)** ทุกระดับมีระดับเหนือเพียงหนึ่งเดียวที่ตนรายงานต่อ
- ความรับผิดชอบต่อผลผลิตเป็นของระดับที่มอบหมาย แม้งานจริงจะลงมือโดยระดับล่างกว่า
- ไม่มีความสัมพันธ์ในแนวข้ามที่เป็นการสั่งงานโดยตรง — การประสานในแนวข้ามผ่านเหตุการณ์เท่านั้น

## 13. ช่วงการควบคุม (Span of Control)

**ช่วงการควบคุม (Span of Control)** คือจำนวนหน่วยที่อยู่ใต้การกำกับโดยตรงของระดับหนึ่ง

- **ผู้บริหารงานองค์กร** กำกับสำนักงานทั้งหมดขององค์กร
- **สำนักงานหนึ่ง** กำกับตัวแทนงานหลายตัวที่สังกัดตน
- **ตัวแทนงานหนึ่ง** กำกับหน่วยปฏิบัติหลายหน่วยที่ตนมอบหมาย

หลักการของช่วงการควบคุม

1. **ช่วงการควบคุมต้องชัดเจน** — แต่ละระดับรู้แน่ชัดว่าตนกำกับหน่วยใดบ้าง
2. **ไม่กำกับข้ามสาย** — ระดับหนึ่งกำกับเฉพาะหน่วยที่อยู่ใต้ตนโดยตรง ไม่ก้าวก่ายสายอื่น
3. **ขยายช่วงการควบคุมได้** — เมื่อเพิ่มตัวแทนงานหรือหน่วยปฏิบัติ ช่วงการควบคุมขยายตามได้ โดยโครงสร้างไม่เปลี่ยน

## 14. การมอบหมายงาน (Delegation)

**การมอบหมายงาน (Delegation)** คือการที่ระดับหนึ่งมอบงานให้ระดับที่อยู่ใต้ตนลงมือทำ

1. **มอบงาน ไม่ใช่มอบความรับผิดชอบทั้งหมด** — ระดับที่มอบหมายยังรับผิดชอบต่อผลลัพธ์สุดท้าย แม้งานจะลงมือโดยระดับล่าง
2. **มอบหมายตามสายเท่านั้น** — สำนักงานมอบให้ตัวแทนงานในสังกัด ตัวแทนงานมอบให้หน่วยปฏิบัติของตน ไม่มอบข้ามสำนักงาน
3. **มอบหมายพร้อมขอบเขตที่ชัด** — งานที่มอบต้องมีขอบเขตและความคาดหวังที่ชัดเจน เพื่อให้ผู้รับงานทำได้ตรงและตรวจสอบได้
4. **การมอบหมายไม่รวมอำนาจอนุมัติองค์ความรู้** — อำนาจนี้มอบหมายให้องค์ประกอบที่ไม่ใช่มนุษย์ไม่ได้

## 15. สายการรายงาน (Reporting Line)

**สายการรายงาน (Reporting Line)** คือเส้นทางที่สถานะและผลงานไหลกลับขึ้นไปตามลำดับชั้น

- **หนึ่งหน่วย หนึ่งสายรายงาน** — แต่ละหน่วยรายงานต่อระดับเหนือเพียงหนึ่งเดียว ไม่มีหน่วยใดรายงานสองสายพร้อมกัน
- **รายงานไหลขึ้นตามสายบังคับบัญชา** — หน่วยปฏิบัติ → ตัวแทนงาน → สำนักงาน → ผู้บริหารงานองค์กร → ผู้อำนวยการสูงสุด
- **รายงานครอบคลุมทั้งผลและปัญหา** — ไม่ใช่เพียงรายงานความสำเร็จ แต่รวมถึงปัญหาและความไม่แน่นอน เพื่อให้องค์กรเห็นสถานะตามจริง
- **สายการรายงานเป็นฐานของการตรวจสอบ** — เพราะทุกผลงานมีสายรายงานที่ชัด องค์กรจึงสืบย้อนได้ว่างานหนึ่งผ่านใครมาบ้าง

## 16. การประสานงานและหลักการสื่อสาร (Coordination & Communication Principles)

การประสานงานในแนวข้าม (ระหว่างสำนักงาน) ต่างจากการบังคับบัญชาในแนวดิ่ง และยึดหลักการสื่อสารที่เคร่งครัด

1. **สำนักงานไม่สื่อสารกันโดยตรง** — ไม่มีสำนักงานใดเรียกสำนักงานอื่นโดยตรง
2. **ทุกการประสานผ่านเหตุการณ์และผู้บริหารงานองค์กร** — การส่งต่องานและการประสานข้ามสำนักงานเกิดผ่านเหตุการณ์ (Event) และการกำกับของผู้บริหารงานองค์กร
3. **ห้ามสำนักงานหนึ่งเรียกหน่วยปฏิบัติของอีกสำนักงานโดยตรง** — เป็นข้อห้ามเด็ดขาด เพื่อรักษาเส้นแบ่งความรับผิดชอบและความเป็นระเบียบของสายการบังคับบัญชา
4. **การประสานต้องโปร่งใสและมีร่องรอย** — ทุกการประสานข้ามสำนักงานต้องเปิดเผยที่มาและตรวจสอบย้อนได้

```
   ❌ Office A ──────► Worker ของ Office B      (ห้ามเด็ดขาด)

   ✅ Office A ──► Event ──► Chief Knowledge Orchestrator ──► Office B
```

> เหตุที่สำนักงานไม่สื่อสารกันโดยตรง คือเพื่อให้องค์กร **ยืดหยุ่นและขยายได้** — สำนักงานหนึ่งไม่ต้องรู้ภายในของอีกสำนักงาน รู้เพียงเหตุการณ์ที่ตนต้องรับ สอดคล้องกับหลักการ Event-Driven ใน [PKA-OPS-001 หัวข้อ 16](PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md#16-event-คืออะไร-event-driven-principle)

## 17. การจัดตั้งสำนักงานใหม่ (Establishing a New Office)

องค์กรจัดตั้งสำนักงานใหม่ได้เมื่อมีงานด้านใหม่ที่ต้องมีเจ้าภาพ โดยยึดหลัก

1. **มีหน้าที่ชัดที่ยังไม่มีเจ้าภาพ** — สำนักงานใหม่ต้องรับผิดชอบงานด้านที่ยังไม่มีสำนักงานใดดูแล ไม่ซ้ำซ้อนกับสำนักงานเดิม
2. **กำหนดภารกิจ ขอบเขต ปัจจัยนำเข้า ผลผลิต** — สำนักงานใหม่ต้องนิยามทั้งสี่ด้านให้ครบก่อนเริ่มงาน (รายละเอียดตามแนวของ PKA-OPS-004)
3. **เข้าสู่การประสานผ่านเหตุการณ์** — สำนักงานใหม่รับฟังและประกาศเหตุการณ์ที่เกี่ยวข้อง โดยไม่ผูกตรงกับสำนักงานเดิม
4. **ไม่กระทบโครงสร้างเดิม** — การเพิ่มสำนักงานใหม่ต้องไม่บังคับให้ต้องรื้อสำนักงานที่มีอยู่

## 18. การเพิ่มตัวแทนงาน (Adding an Agent)

ภายในสำนักงานหนึ่ง องค์กรเพิ่มตัวแทนงานได้เมื่อมีภารกิจใหม่ที่ต้องมีผู้รับผิดชอบ โดยยึดหลัก

1. **สังกัดสำนักงานเดียว** — ตัวแทนงานใหม่สังกัดสำนักงานหนึ่งเท่านั้น
2. **รับผิดชอบภารกิจเฉพาะ** — ตัวแทนงานใหม่ต้องมีภารกิจที่ชัดและไม่ทับซ้อนกับตัวแทนงานเดิมในสำนักงานเดียวกัน
3. **ไม่เปลี่ยนโครงสร้างสำนักงาน** — การเพิ่มตัวแทนงานเป็นการขยายช่วงการควบคุมของสำนักงาน โดยรูปของสำนักงานไม่เปลี่ยน

## 19. การเพิ่มหน่วยปฏิบัติ (Adding a Worker)

ภายในตัวแทนงานหนึ่ง องค์กรเพิ่มหน่วยปฏิบัติได้เมื่อปริมาณงานต้องการ โดยยึดหลัก

1. **หนึ่งหน้าที่ต่อหน่วย** — หน่วยปฏิบัติใหม่ต้องมีหน้าที่เดียวที่ชัดเจน ตามหลัก Single Responsibility
2. **อยู่ใต้ตัวแทนงานเดียว** — หน่วยปฏิบัติใหม่รับงานจากและรายงานต่อตัวแทนงานเดียว
3. **เพิ่ม ลด เปลี่ยนได้อย่างอิสระ** — หน่วยปฏิบัติเป็นกำลังพลที่ทดแทนได้มากที่สุด เพิ่มหรือเปลี่ยนได้โดยไม่กระทบโครงสร้างเหนือขึ้นไป

## 20. การเติบโตขององค์กร (Organization Growth)

องค์กรนี้ออกแบบให้เติบโตได้สามทิศทาง โดย **ไม่เปลี่ยนโครงสร้างองค์กร**

```
   เติบโตแนวกว้าง :  เพิ่ม Office       (งานด้านใหม่)
   เติบโตแนวลึก   :  เพิ่ม Agent        (ภารกิจใหม่ในสำนักงานเดิม)
   เติบโตแนวปฏิบัติ:  เพิ่ม Worker       (กำลังลงมือทำมากขึ้น)
```

- **เพิ่มสำนักงาน (Office)** — ขยายขอบเขตงานขององค์กรไปยังด้านใหม่
- **เพิ่มตัวแทนงาน (Agent)** — ขยายความสามารถภายในสำนักงานเดิม
- **เพิ่มหน่วยปฏิบัติ (Worker)** — ขยายกำลังการลงมือทำงาน

> หลักสำคัญของการเติบโต: **องค์กรเพิ่มองค์ประกอบได้ทุกระดับ แต่รูปของโครงสร้าง (ห้าระดับและสายการบังคับบัญชา) ต้องคงเดิมเสมอ** การเติบโตคือการ "เติม" ไม่ใช่การ "รื้อ"

## 21. องค์กรถาวร กำลังพลทดแทนได้ (Stable Organization / Replaceable Workforce)

หลักการสำคัญที่สุดของเอกสารฉบับนี้คือการแยก **โครงสร้างที่ถาวร** ออกจาก **กำลังพลที่ทดแทนได้** อย่างเด็ดขาด

| องค์ประกอบ | สถานะ | เปลี่ยนได้หรือไม่ |
|---|---|---|
| **โครงสร้างองค์กร** (ห้าระดับ + สายการบังคับบัญชา) | ถาวร | **ไม่เปลี่ยน** |
| **สำนักงาน** (Office) | ถาวร | คงอยู่ ไม่เปลี่ยนรูปง่าย ๆ |
| **ตัวแทนงาน** (Agent) | ทดแทนได้ | เพิ่ม ลด เปลี่ยนได้ |
| **หน่วยปฏิบัติ** (Worker) | ทดแทนได้ | เปลี่ยนได้ตลอดเวลา |
| **ผู้ช่วยอัจฉริยะ** (AI) | ทดแทนได้ | เปลี่ยนรุ่นได้ |

หลักการนี้กล่าวว่า

- **สำนักงานเป็นโครงสร้างถาวร** — เป็นหลักยึดขององค์กรที่คงอยู่ข้ามการเปลี่ยนแปลงของผู้ปฏิบัติ
- **ตัวแทนงานเพิ่ม ลด เปลี่ยนได้** — เป็นกำลังพลที่ปรับเปลี่ยนตามภารกิจ
- **หน่วยปฏิบัติเปลี่ยนได้ตลอด** — เป็นกำลังพลที่ยืดหยุ่นที่สุด
- **ผู้ช่วยอัจฉริยะเปลี่ยนรุ่นได้** — เครื่องมือและผู้ช่วยพัฒนาไปได้โดยอิสระ
- **แต่องค์กรต้องไม่เปลี่ยน** — ไม่ว่ากำลังพลและเครื่องมือจะเปลี่ยนไปเพียงใด โครงสร้างองค์กรยังคงเดิม

> นี่คือเหตุผลที่องค์กรนี้ **ยั่งยืน** — เพราะความถูกต้องของพระธรรมไม่ได้ฝากไว้กับผู้ปฏิบัติหรือเครื่องมือคนใดเครื่องใด แต่ฝากไว้กับโครงสร้างและหลักการที่มั่นคงข้ามกาลเวลา

## 22. ธรรมาภิบาล (Governance)

โครงสร้างองค์กรนี้อยู่ภายใต้ธรรมาภิบาลที่กำกับการเปลี่ยนแปลงและความรับผิดชอบ

1. **การเปลี่ยนโครงสร้างต้องผ่านการอนุมัติของมนุษย์** — การจัดตั้งสำนักงานใหม่หรือการเปลี่ยนสายการบังคับบัญชาเป็นเรื่องระดับองค์กรที่มนุษย์ต้องอนุมัติ
2. **ทุกการเปลี่ยนแปลงต้องมีร่องรอย** — การเพิ่ม ลด หรือเปลี่ยนองค์ประกอบขององค์กรต้องบันทึกไว้ตรวจสอบได้
3. **อำนาจอนุมัติองค์ความรู้แยกจากอำนาจบริหารงาน** — ผู้บริหารงานองค์กรมีอำนาจบริหารการไหลของงาน แต่ไม่มีอำนาจอนุมัติองค์ความรู้ ซึ่งเป็นของมนุษย์
4. **โครงสร้างต้องตรวจสอบและกำกับได้เสมอ** — ไม่มีส่วนใดขององค์กรอยู่นอกสายการกำกับและการตรวจสอบ

## 23. ความสัมพันธ์กับ PKA-OPS-001 และเอกสารอ้างอิง

เอกสารฉบับนี้เป็นส่วนขยายของ [PKA-OPS-001](PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md) ภายในตระกูลปฏิบัติการ (PKA-OPS) โดย

- **PKA-OPS-001** จัดตั้งสำนักงานองค์ความรู้ดิจิทัลและวางหลักการระดับองค์กร — *องค์กรนี้คืออะไรและยึดหลักใด*
- **PKA-OPS-002** (ฉบับนี้) ลงรายละเอียด **โครงสร้างองค์กรและสายการบังคับบัญชา** — *องค์กรนี้มีรูปร่างอย่างไรและใครขึ้นกับใคร*
- **PKA-OPS-003** กำหนด **แบบจำลองขีดความสามารถขององค์กร** — *องค์กรทำอะไรได้*
- **PKA-OPS-004** (จะจัดทำต่อไป) จะลงรายละเอียด **หน้าที่ของแต่ละสำนักงาน** ตามหลัก Capability First — *แต่ละสำนักงานทำอะไรโดยละเอียด*

เอกสารฉบับนี้อ้างอิงและต้องสอดคล้องกับ

- **[PKA-STD-001 — ธรรมนูญโครงการ](../standards/PKA-STD-001-ธรรมนูญโครงการ.md)** (เอกสารแม่) — หลักการสูงสุด รวมถึงหลักการด้าน AI และหลักการตัดสินใจที่กำหนดให้มนุษย์รับผิดชอบสูงสุด
- **[PKA-ARC-001 — สถาปัตยกรรมองค์รวมของโครงการ](../architecture/PKA-ARC-001-สถาปัตยกรรมองค์รวมของโครงการ.md)** — ภาพรวมของระบบที่องค์กรนี้ดำเนินงานอยู่ภายใน
- **[PKA-OPS-001 — ธรรมนูญสำนักงานดิจิทัล](PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md)** — กฎบัตรองค์กรที่เอกสารฉบับนี้ขยายความ

> **ลำดับชั้นยังคงเดิม:** เมื่อใดที่เนื้อหาของ OPS-002 ขัดกับเอกสารระดับสูงกว่า ให้ยึดเอกสารระดับสูงกว่าเสมอ

## 24. สิ่งที่ยังไม่กำหนด (To Be Defined)

เอกสารฉบับนี้ตั้งใจ **ไม่กำหนด** เรื่องต่อไปนี้ เพื่อรักษาขอบเขตให้อยู่ที่ "โครงสร้างองค์กร" เท่านั้น

- หน้าที่ ภารกิจ ขอบเขต ปัจจัยนำเข้า และผลผลิตของแต่ละสำนักงานโดยละเอียด → **PKA-OPS-004 — Office Catalog**
- การออกแบบตัวแทนงานแต่ละตัว ความสามารถ และชุดคำสั่ง → เอกสาร PKA-OPS ฉบับถัด ๆ ไป
- กระแสงานเชิงปฏิบัติและขั้นตอนการทำงานข้ามสำนักงาน → เอกสาร PKA-OPS ฉบับถัด ๆ ไป
- จำนวนตัวแทนงานและหน่วยปฏิบัติที่แน่นอนในแต่ละสำนักงาน → กำหนดตามความเหมาะสมในภายหลัง
- รายละเอียดเชิงปฏิบัติเชิงเทคนิคทั้งหมด → อยู่นอกขอบเขตของตระกูล OPS

## 25. Roadmap

ลำดับการพัฒนาเอกสารในตระกูลปฏิบัติการ (ปรับปรุงให้ความสัมพันธ์ระหว่างขีดความสามารถกับสำนักงานถูกนิยามให้ชัดก่อนการออกแบบกระแสงานและตัวแทนงาน)

```
   PKA-OPS-001 : ธรรมนูญสำนักงานดิจิทัล              ✔ จัดทำแล้ว
        ▼
   PKA-OPS-002 : โครงสร้างองค์กรและสายการบังคับบัญชา    ◀ เอกสารฉบับนี้
        ▼
   PKA-OPS-003 : แบบจำลองขีดความสามารถขององค์กร        ✔ จัดทำแล้ว
        ▼
   PKA-OPS-004 : สารบบสำนักงาน                        ○ ขั้นถัดไป
        ▼
   PKA-OPS-005 : เมทริกซ์การมอบหมายขีดความสามารถ      ○ อนาคต
        ▼
   PKA-OPS-005A: สารบบหน้าที่ทางธุรกิจ                ○ อนาคต
        ▼
   PKA-OPS-006 : สถาปัตยกรรมกระแสงาน                  ○ อนาคต
        ▼
   PKA-OPS-007 : สถาปัตยกรรมเหตุการณ์                  ○ อนาคต
        ▼
   PKA-OPS-008 : สถาปัตยกรรมตัวแทนงาน                  ○ อนาคต
        ▼
   PKA-OPS-009 : สถาปัตยกรรมหน่วยปฏิบัติ                ○ อนาคต
```

| รหัส | ชื่อ | จุดมุ่งหมาย |
|---|---|---|
| **OPS-001** | Charter (ธรรมนูญสำนักงานดิจิทัล) | จัดตั้งองค์กรและวางหลักการระดับองค์กร |
| **OPS-002** | Organization Structure (โครงสร้างองค์กรและสายการบังคับบัญชา) | กำหนดโครงสร้างองค์กรและสายการบังคับบัญชา (ฉบับนี้) |
| **OPS-003** | Enterprise Capability Model (แบบจำลองขีดความสามารถขององค์กร) | กำหนดว่าองค์กรทำอะไรได้ |
| **OPS-004** | Office Catalog (สารบบสำนักงาน) | จับคู่ขีดความสามารถกับสำนักงานเจ้าของ และลงรายละเอียดหน้าที่ของแต่ละสำนักงาน |
| **OPS-005** | Capability Assignment Matrix (เมทริกซ์การมอบหมายขีดความสามารถ) | สะพานเชื่อมขีดความสามารถสู่การมอบหมายความรับผิดชอบและการอนุมัติ |
| **OPS-005A** | Business Function Catalog (สารบบหน้าที่ทางธุรกิจ) | นิยามหน้าที่ถาวรขององค์กร เชื่อมขีดความสามารถกับกระแสงาน |
| **OPS-006** | Workflow Architecture (สถาปัตยกรรมกระแสงาน) | กำหนดกระแสงานที่ทำหน้าที่ทางธุรกิจ |
| **OPS-007** | Event Architecture (สถาปัตยกรรมเหตุการณ์) | กำหนดเหตุการณ์ที่ใช้ประสานงานระหว่างสำนักงาน |
| **OPS-008** | Agent Architecture (สถาปัตยกรรมตัวแทนงาน) | กำหนดตัวแทนงานและภารกิจภายในสำนักงาน |
| **OPS-009** | Worker Architecture (สถาปัตยกรรมหน่วยปฏิบัติ) | กำหนดหน่วยปฏิบัติและงานหนึ่งหน่วย |
