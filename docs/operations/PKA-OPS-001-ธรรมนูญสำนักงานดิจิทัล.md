## 1. วัตถุประสงค์ (Purpose)

เอกสารฉบับนี้จัดตั้ง **สำนักงานองค์ความรู้ดิจิทัล (Digital Knowledge Office)** ของโครงการ PKA และวางกฎบัตรพื้นฐานว่าด้วยการจัดองค์กร การแบ่งหน้าที่ การประสานงาน และการกำกับดูแล เพื่อให้การทำงานทั้งหมดของโครงการ — ตั้งแต่การรวบรวมแหล่งข้อมูลต้นทางจนถึงการเผยแพร่องค์ความรู้ — ดำเนินไปอย่างมีระเบียบ ตรวจสอบได้ และสอดคล้องกับหลักการในเอกสารแม่ทุกขั้นตอน

เอกสารนี้มีไว้เพื่อ

1. กำหนดว่าโครงการ PKA จัดระเบียบงานในรูปของ **องค์กร (Office)** ไม่ใช่ในรูปของกลุ่มเครื่องมือที่กระจัดกระจาย
2. กำหนด **โครงสร้างการบังคับบัญชา** ที่ชัดเจน ตั้งแต่ผู้บริหารงานองค์กรลงไปจนถึงหน่วยปฏิบัติที่เล็กที่สุด
3. กำหนด **หน้าที่ ขอบเขต ปัจจัยนำเข้า และผลผลิต** ของแต่ละสำนักงานในระดับองค์กร โดยยังไม่ลงรายละเอียดเชิงปฏิบัติ
4. กำหนด **สิทธิ์การอนุมัติ** และ **เส้นแบ่งความรับผิดชอบ** ระหว่างมนุษย์กับผู้ช่วยอัจฉริยะอย่างเด็ดขาด
5. วางหลักการ **การประสานงานผ่านเหตุการณ์ (Event)** เพื่อให้องค์กรขยายได้ในอนาคตโดยไม่ต้องรื้อโครงสร้าง

> เมื่ออ่านเอกสารฉบับนี้จบ ผู้อ่านควรเข้าใจว่า **สำนักงานองค์ความรู้ดิจิทัลของ PKA คือองค์กรแบบใด ทำงานอย่างไร มีสำนักงานใดบ้าง ยึดหลักการใด** และเหตุใดองค์กรนี้จึงสามารถเติบโตเป็นระบบที่มีผู้ช่วยอัจฉริยะทำงานร่วมกันได้ในอนาคต โดยไม่ต้องเปลี่ยนแปลงโครงสร้างองค์กรที่วางไว้

## 2. ขอบเขต (Scope)

**อยู่ในขอบเขต**

- โครงสร้างองค์กรของสำนักงานองค์ความรู้ดิจิทัล และความสัมพันธ์ระหว่างสำนักงาน
- หลักการทำงาน หลักการประสานงาน และหลักการกำกับดูแลในระดับองค์กร
- บทบาทของ **ผู้บริหารงานองค์กร (Chief Knowledge Orchestrator)** สำนักงาน (Office) ตัวแทนงาน (Agent) และหน่วยปฏิบัติ (Worker)
- สิทธิ์การอนุมัติ เส้นแบ่งระหว่างมนุษย์กับผู้ช่วยอัจฉริยะ และหลักการมนุษย์ในวงจรการตัดสินใจ
- กระแสงานเชิงแนวคิด (Conceptual Workflow) ของการแปรแหล่งข้อมูลต้นทางสู่องค์ความรู้ที่เผยแพร่ได้

**อยู่นอกขอบเขต**

- รายละเอียดการทำงานภายในของแต่ละสำนักงานในระดับขั้นตอนปฏิบัติ (จะกำหนดในเอกสาร PKA-OPS ฉบับถัด ๆ ไป)
- การออกแบบตัวแทนงาน (Agent) แต่ละตัวในรายละเอียด
- เครื่องมือ เทคโนโลยี รูปแบบข้อมูล หรือสถาปัตยกรรมเชิงเทคนิคใด ๆ
- กระบวนการ ขั้นตอน หรือชุดคำสั่งที่นำไปปฏิบัติได้ทันที

> เอกสารฉบับนี้กำหนด **"องค์กร"** ไม่ใช่ **"การลงมือทำ"** เปรียบดั่งรัฐธรรมนูญที่กำหนดโครงสร้างของรัฐ แต่ไม่ใช่กฎหมายลูกหรือระเบียบปฏิบัติประจำวัน

## 3. วิสัยทัศน์ของสำนักงานดิจิทัล (Vision)

> เป็น **สำนักงานองค์ความรู้ดิจิทัล** ที่ทำงานอย่างเป็นระบบดุจหน่วยงานที่มีระเบียบ มีสำนักงานที่รับผิดชอบงานแต่ละด้านอย่างชัดเจน ประสานกันอย่างราบรื่น และกำกับดูแลด้วยหลักธรรมาภิบาล เพื่อแปรแหล่งข้อมูลต้นทางทางพระธรรมให้เป็นองค์ความรู้ที่ **ถูกต้อง ตรวจสอบได้ และเชื่อมโยงกัน** โดยมนุษย์เป็นผู้รับผิดชอบสูงสุดเสมอ

สำนักงานนี้มุ่งให้การจัดการองค์ความรู้พระธรรมเป็นงานที่มี *เจ้าภาพ* ชัดเจนในทุกขั้นตอน มี *เส้นทางของงาน* ที่สืบย้อนได้ และมี *ผู้รับผิดชอบ* ที่ระบุได้เสมอ ความเป็นระเบียบขององค์กรนี้เป็นเครื่องประกันว่าความถูกต้องของพระธรรมจะไม่หล่นหายระหว่างทาง

## 4. พันธกิจ (Mission)

เพื่อบรรลุวิสัยทัศน์ สำนักงานองค์ความรู้ดิจิทัลมีพันธกิจดังนี้

- **จัดระเบียบงานองค์ความรู้** ให้อยู่ในรูปของสำนักงานที่มีหน้าที่ชัดเจน ไม่ทับซ้อน และไม่มีช่องว่าง
- **รักษาเส้นทางขององค์ความรู้** ตั้งแต่แหล่งข้อมูลต้นทางจนถึงการเผยแพร่ ให้สืบย้อนได้ครบทุกขั้น
- **กำกับคุณภาพและความถูกต้อง** ของทุกผลผลิตก่อนส่งต่อไปยังสำนักงานถัดไป
- **ธำรงเส้นแบ่งความรับผิดชอบ** ระหว่างมนุษย์กับผู้ช่วยอัจฉริยะ มิให้ผู้ช่วยอัจฉริยะล่วงล้ำอำนาจการอนุมัติองค์ความรู้
- **ประสานงานระหว่างสำนักงาน** ผ่านการส่งต่องานที่มีระเบียบ เพื่อให้องค์กรทำงานเป็นหนึ่งเดียว
- **เปิดทางให้องค์กรเติบโต** โดยออกแบบหลักการประสานงานที่รองรับการขยายตัวในอนาคตได้โดยไม่ต้องรื้อโครงสร้าง

## 5. หลักการของสำนักงานดิจิทัล (Principles)

หลักการแม่บทของสำนักงาน ซึ่งทุกสำนักงาน ทุกตัวแทนงาน และทุกหน่วยปฏิบัติต้องยึดถือร่วมกัน

1. **องค์กรนำเครื่องมือ** — โครงการจัดระเบียบงานในรูปขององค์กรที่มีหน้าที่ชัดเจน ก่อนจะคิดถึงเครื่องมือใด ๆ เครื่องมือเป็นเพียงสิ่งที่รับใช้หน้าที่ขององค์กร
2. **หนึ่งหน้าที่ หนึ่งเจ้าภาพ** — งานแต่ละด้านมีสำนักงานที่รับผิดชอบเพียงสำนักงานเดียว ไม่มีงานใดไร้เจ้าภาพ และไม่มีงานใดมีเจ้าภาพซ้ำซ้อนจนปัดความรับผิดชอบกันได้
3. **มนุษย์รับผิดชอบสูงสุด** — การอนุมัติองค์ความรู้และการตัดสินใจที่กระทบความถูกต้องของพระธรรมเป็นอำนาจของมนุษย์เสมอ ผู้ช่วยอัจฉริยะไม่มีวันมีอำนาจนี้
4. **ตรวจสอบย้อนกลับได้ทั้งองค์กร** — ทุกผลผลิตของทุกสำนักงานต้องสืบย้อนได้ว่ามาจากแหล่งใด ผ่านสำนักงานใด และใครเป็นผู้อนุมัติ
5. **ส่งต่อผ่านเหตุการณ์** — สำนักงานรับงานจากเหตุการณ์ (Event) ไม่เรียกสำนักงานอื่นโดยตรง เพื่อให้องค์กรยืดหยุ่นและขยายได้
6. **คุณภาพก่อนส่งต่อ** — ไม่มีผลผลิตใดถูกส่งต่อไปยังสำนักงานถัดไป หากยังไม่ผ่านเกณฑ์คุณภาพของสำนักงานต้นทาง
7. **ความถูกต้องเหนือความเร็ว** — เมื่อความรวดเร็วในการผลิตองค์ความรู้ขัดกับความถูกต้องของพระธรรม ให้ความถูกต้องเป็นฝ่ายชนะเสมอ สอดคล้องกับหลักการในเอกสารแม่
8. **โปร่งใสในทุกขั้น** — ทุกสำนักงานต้องเปิดเผยที่มา สถานะ และข้อจำกัดของงานที่ตนผลิต ไม่กลบเกลื่อนความไม่แน่นอน

## 6. สำนักงานดิจิทัลคืออะไร

โครงการ PKA **ไม่ได้มองตนเองเป็นกลุ่มของผู้ช่วยอัจฉริยะหลายตัวที่ทำงานขนานกัน** หากแต่มองตนเองเป็น **องค์กรหนึ่งเดียว** ที่มีโครงสร้าง มีระเบียบ และมีสายการบังคับบัญชา — เรียกว่า **สำนักงานองค์ความรู้ดิจิทัล (Digital Knowledge Office)**

ความแตกต่างนี้สำคัญยิ่ง การมองเป็น "กลุ่มผู้ช่วยหลายตัว" ทำให้ไม่มีเจ้าภาพที่ชัดเจน ไม่มีสายความรับผิดชอบ และยากต่อการกำกับความถูกต้อง ในขณะที่การมองเป็น **"สำนักงาน"** ทำให้ทุกงานมีเจ้าภาพ มีผู้บังคับบัญชา และมีเส้นทางการอนุมัติที่ตรวจสอบได้

สำนักงานองค์ความรู้ดิจิทัลของ PKA จัดโครงสร้างเป็นสามชั้นที่ซ้อนกันลงไป

```
              │   รับผิดชอบงานหนึ่งด้านขององค์กร
              ▼
        ตัวแทนงาน (Agent)
              │   รับผิดชอบภารกิจหนึ่งภายในสำนักงาน
              ▼
        หน่วยปฏิบัติ (Worker)
                  ลงมือทำงานหนึ่งหน่วยตามที่ได้รับมอบหมาย
```

และเหนือสำนักงานทั้งหมดขึ้นไปมี **ผู้บริหารงานองค์กร (Chief Knowledge Orchestrator)** เป็นผู้กำกับ ประสาน และติดตามงานของทั้งองค์กร

> สำนักงานนี้เป็น **แนวคิดเชิงองค์กร** ไม่ใช่การพรรณนาเครื่องมือใด ๆ โครงสร้างนี้ออกแบบให้คงอยู่ได้แม้วิธีการทำงานเปลี่ยนไป และรองรับการเติบโตจากองค์กรที่มนุษย์ทำงานเป็นหลัก ไปสู่องค์กรที่มีผู้ช่วยอัจฉริยะทำงานร่วมในอนาคต โดย **โครงสร้างองค์กรไม่ต้องเปลี่ยน**

## 7. โครงสร้างองค์กร (Organizational Structure)

องค์กรประกอบด้วยองค์ประกอบสี่ระดับ เรียงจากบนลงล่าง

| ระดับ | องค์ประกอบ | หน้าที่โดยสังเขป |
|---|---|---|
| 1 | **ผู้บริหารงานองค์กร** (Chief Knowledge Orchestrator) | กำกับ ประสาน และติดตามงานของทั้งองค์กร จัดลำดับงาน จัดการเหตุการณ์ และจัดทำรายงาน |
| 2 | **สำนักงาน** (Office) | รับผิดชอบงานหนึ่งด้านขององค์กรอย่างเต็มตัว มีหน้าที่ ขอบเขต ปัจจัยนำเข้า และผลผลิตที่ชัดเจน |
| 3 | **ตัวแทนงาน** (Agent) | รับผิดชอบภารกิจเฉพาะภายในสำนักงานหนึ่ง ๆ |
| 4 | **หน่วยปฏิบัติ** (Worker) | ลงมือทำงานหนึ่งหน่วยที่เล็กที่สุดตามที่ได้รับมอบหมาย |

```
                ┌───────────────────────────────────┐
                │   Chief Knowledge Orchestrator     │
                │        (ผู้บริหารงานองค์กร)          │
                └───────────────────────────────────┘
                                  │
            ┌───────────────┬─────┴─────┬───────────────┐
            ▼               ▼           ▼               ▼
       ┌─────────┐    ┌─────────┐  ┌─────────┐     ┌─────────┐
       │ สำนักงาน │    │ สำนักงาน │  │ สำนักงาน │ ... │ สำนักงาน │
       └─────────┘    └─────────┘  └─────────┘     └─────────┘
            │
        ┌───┴───┐
        ▼       ▼
   ┌────────┐ ┌────────┐
   │ Agent  │ │ Agent  │
   └────────┘ └────────┘
        │
    ┌───┴───┐
    ▼       ▼
 ┌──────┐ ┌──────┐
 │Worker│ │Worker│
 └──────┘ └──────┘
```

หลักการของโครงสร้างนี้คือ **ความรับผิดชอบไหลลง คุณภาพไหลขึ้น** — คำสั่งและการมอบหมายไหลจากบนลงล่าง ส่วนผลผลิตที่ผ่านการตรวจคุณภาพแล้วไหลจากล่างขึ้นบนเพื่อรอการอนุมัติและส่งต่อ

## 8. ระดับการบังคับบัญชา (Command Levels)

สายการบังคับบัญชาขององค์กรมีสี่ระดับ และมีหลักว่า **แต่ละระดับรับผิดชอบต่อระดับที่อยู่เหนือตน**

1. **ผู้บริหารงานองค์กร** กำกับสำนักงานทั้งหมด แต่ไม่ลงไปทำงานขององค์ความรู้เอง
2. **สำนักงาน** รับมอบหมายจากผู้บริหารงานองค์กรผ่านเหตุการณ์ และกำกับตัวแทนงานภายในของตน
3. **ตัวแทนงาน** รับภารกิจจากสำนักงานของตน และมอบหมายงานย่อยให้หน่วยปฏิบัติ
4. **หน่วยปฏิบัติ** ลงมือทำงานหนึ่งหน่วย แล้วส่งผลกลับขึ้นไปตามสาย

**กฎของการบังคับบัญชา**

- งานข้ามสำนักงานต้องผ่านเหตุการณ์เสมอ ไม่มีตัวแทนงานหรือหน่วยปฏิบัติใดสั่งงานข้ามสำนักงานโดยตรง
- ความรับผิดชอบต่อผลผลิตเป็นของสำนักงานต้นทาง แม้งานจริงจะลงมือโดยหน่วยปฏิบัติ
- การอนุมัติที่กระทบความถูกต้องของพระธรรมไม่อยู่ในสายการบังคับบัญชาของผู้ช่วยอัจฉริยะ แต่สงวนไว้สำหรับมนุษย์เสมอ (ดูหัวข้อ [19](#19-human-in-the-loop--มนุษย์ในวงจรการตัดสินใจ))

## 9. สำนักงานทั้งหมด (The Offices)

องค์กรประกอบด้วยสำนักงานหลักดังนี้ แต่ละสำนักงานรับผิดชอบงานหนึ่งด้านในเส้นทางของการแปรแหล่งข้อมูลต้นทางสู่องค์ความรู้ที่เผยแพร่ได้

| สำนักงาน (ไทย) | Office (อังกฤษ) | บทบาทโดยสังเขป |
|---|---|---|
| สำนักแหล่งข้อมูลต้นทาง | Source Office | ดูแลแหล่งข้อมูลต้นทางและลำดับชั้นความน่าเชื่อถือ |
| สำนักคลังคัมภีร์ | Corpus Office | จัดระเบียบเนื้อหาคัมภีร์ให้เป็นคลังที่ใช้งานได้ |
| สำนักคลังศัพท์ | Lexicon Office | ดูแลศัพท์ คำนิยาม และอภิธานศัพท์มาตรฐาน |
| สำนักองค์ความรู้ | Knowledge Office | แปรเนื้อหาเป็นองค์ความรู้ที่มีโครงสร้าง |
| สำนักความสัมพันธ์ | Relationship Office | สร้างและดูแลความสัมพันธ์ระหว่างองค์ความรู้ |
| สำนักการอ้างอิง | Citation Office | กำกับการอ้างอิงให้ครบถ้วนและตรวจสอบได้ |
| สำนักคุณภาพ | Quality Office | ตรวจสอบคุณภาพและความถูกต้องของผลผลิต |
| สำนักวิจัย | Research Office | ค้นคว้าและเตรียมความรู้สนับสนุนสำนักงานอื่น |
| สำนักเผยแพร่ | Publishing Office | จัดเตรียมองค์ความรู้ที่อนุมัติแล้วเพื่อเผยแพร่ |
| สำนักสถาปัตยกรรม | Architecture Office | ดูแลความสอดคล้องเชิงโครงสร้างขององค์กรและองค์ความรู้ |
| สำนักปฏิบัติการ | Operations Office | ประสานการทำงาน ติดตามเหตุการณ์ และดูแลระเบียบขององค์กร |

> รายชื่อนี้เป็นสำนักงาน **หลัก** ขั้นต่ำขององค์กร องค์กรอาจจัดตั้งสำนักงานเพิ่มเติมในอนาคตตามความเหมาะสม โดยต้องคงหลักการ "หนึ่งหน้าที่ หนึ่งเจ้าภาพ" ไว้เสมอ

## 10. หน้าที่ของแต่ละสำนักงาน (Office Mandates)

แต่ละสำนักงานกำหนดด้วยสี่ด้าน — **ภารกิจ (Mission)** *สิ่งที่สำนักงานนี้มุ่งทำ*, **ขอบเขต (Scope)** *สิ่งที่อยู่ในความรับผิดชอบ*, **ปัจจัยนำเข้า (Input)** *สิ่งที่รับเข้ามาทำงาน*, และ **ผลผลิต (Output)** *สิ่งที่ส่งมอบออกไป* เอกสารฉบับนี้กำหนดเพียงระดับองค์กร **ยังไม่ลงรายละเอียดระดับตัวแทนงาน**

### 10.1 สำนักแหล่งข้อมูลต้นทาง (Source Office)

- **ภารกิจ:** ธำรงแหล่งข้อมูลต้นทางและลำดับชั้นความน่าเชื่อถือให้เป็นรากฐานที่มั่นคงขององค์กร
- **ขอบเขต:** การรับ ขึ้นทะเบียน และดูแลแหล่งข้อมูลต้นทาง การกำกับลำดับชั้นความน่าเชื่อถือตามเอกสารแม่
- **ปัจจัยนำเข้า:** แหล่งข้อมูลต้นทางทางพระธรรมและคัมภีร์
- **ผลผลิต:** แหล่งข้อมูลต้นทางที่ขึ้นทะเบียน ระบุลำดับชั้น และพร้อมให้สำนักงานอื่นอ้างอิง

### 10.2 สำนักคลังคัมภีร์ (Corpus Office)

- **ภารกิจ:** จัดระเบียบเนื้อหาคัมภีร์ให้เป็นคลังที่ค้นและอ้างอิงได้อย่างเป็นระบบ
- **ขอบเขต:** การจัดโครงสร้างเนื้อหา การแบ่งหมวด การรักษาความครบถ้วนของเนื้อหาต้นทาง
- **ปัจจัยนำเข้า:** แหล่งข้อมูลต้นทางที่ขึ้นทะเบียนจากสำนักแหล่งข้อมูลต้นทาง
- **ผลผลิต:** คลังเนื้อหาคัมภีร์ที่มีโครงสร้างชัดเจน รักษาความครบถ้วนของต้นฉบับ

### 10.3 สำนักคลังศัพท์ (Lexicon Office)

- **ภารกิจ:** ดูแลศัพท์ คำนิยาม และอภิธานศัพท์มาตรฐานให้สม่ำเสมอทั้งองค์กร
- **ขอบเขต:** การรวบรวมศัพท์ การกำกับคำนิยามที่ถูกต้อง การรักษาความสอดคล้องของการใช้คำ
- **ปัจจัยนำเข้า:** เนื้อหาจากคลังคัมภีร์ และศัพท์ที่สำนักงานอื่นร้องขอ
- **ผลผลิต:** อภิธานศัพท์มาตรฐานและคำนิยามที่ทุกสำนักงานใช้ร่วมกัน

### 10.4 สำนักองค์ความรู้ (Knowledge Office)

- **ภารกิจ:** แปรเนื้อหาคัมภีร์ให้เป็นองค์ความรู้ที่มีโครงสร้าง เข้าถึงและเข้าใจได้
- **ขอบเขต:** การจัดโครงสร้างองค์ความรู้ การเรียบเรียงคำอธิบายของโครงการที่แยกชัดจากพุทธพจน์
- **ปัจจัยนำเข้า:** เนื้อหาจากคลังคัมภีร์ และศัพท์มาตรฐานจากสำนักคลังศัพท์
- **ผลผลิต:** องค์ความรู้ที่มีโครงสร้าง พร้อมระบุสถานะและที่มาอย่างโปร่งใส

### 10.5 สำนักความสัมพันธ์ (Relationship Office)

- **ภารกิจ:** สร้างและดูแลความสัมพันธ์ระหว่างองค์ความรู้ให้ถูกต้องและมีหลักฐานรองรับ
- **ขอบเขต:** การกำหนดความเชื่อมโยงระหว่างหลักธรรม หมวดหมู่ และคัมภีร์ ตามแค็ตตาล็อกชนิดความสัมพันธ์
- **ปัจจัยนำเข้า:** องค์ความรู้ที่มีโครงสร้างจากสำนักองค์ความรู้
- **ผลผลิต:** เครือข่ายความสัมพันธ์ขององค์ความรู้ที่มีหลักฐานกำกับทุกเส้น

### 10.6 สำนักการอ้างอิง (Citation Office)

- **ภารกิจ:** กำกับให้ทุกองค์ความรู้มีการอ้างอิงที่ครบถ้วนและตรวจสอบย้อนกลับได้
- **ขอบเขต:** การตรวจความครบถ้วนของการอ้างอิง การรักษารูปแบบการอ้างอิงที่สม่ำเสมอ
- **ปัจจัยนำเข้า:** องค์ความรู้และความสัมพันธ์จากสำนักงานต้นทาง
- **ผลผลิต:** องค์ความรู้ที่มีสายการอ้างอิงครบถ้วน สืบย้อนถึงแหล่งข้อมูลต้นทางได้

### 10.7 สำนักคุณภาพ (Quality Office)

- **ภารกิจ:** ตรวจสอบคุณภาพและความถูกต้องของผลผลิตก่อนเข้าสู่การทบทวนของมนุษย์
- **ขอบเขต:** การตรวจตามเกณฑ์คุณภาพ การคืนงานที่ไม่ผ่านเกณฑ์ การบันทึกผลการตรวจ
- **ปัจจัยนำเข้า:** องค์ความรู้ที่มีการอ้างอิงครบจากสำนักการอ้างอิง
- **ผลผลิต:** ผลการตรวจคุณภาพ และองค์ความรู้ที่ผ่านเกณฑ์พร้อมเข้าสู่การทบทวน

### 10.8 สำนักวิจัย (Research Office)

- **ภารกิจ:** ค้นคว้าและเตรียมความรู้สนับสนุนเพื่อช่วยให้สำนักงานอื่นทำงานได้แม่นยำขึ้น
- **ขอบเขต:** การค้นคว้าเชิงลึก การรวบรวมข้อมูลประกอบ การเสนอแนวการตีความที่มีอยู่ โดยไม่ชี้ขาด
- **ปัจจัยนำเข้า:** คำถามและคำร้องขอจากสำนักงานอื่นและผู้บริหารงานองค์กร
- **ผลผลิต:** รายงานการค้นคว้าและข้อมูลสนับสนุนที่ระบุที่มาชัดเจน

### 10.9 สำนักเผยแพร่ (Publishing Office)

- **ภารกิจ:** จัดเตรียมองค์ความรู้ที่ได้รับอนุมัติแล้วให้พร้อมเผยแพร่อย่างเคารพต่อเนื้อหา
- **ขอบเขต:** การจัดรูปเพื่อเผยแพร่ การรักษาความครบถ้วนของการอ้างอิงในงานที่เผยแพร่
- **ปัจจัยนำเข้า:** องค์ความรู้ที่ผ่านการทบทวนและอนุมัติโดยมนุษย์
- **ผลผลิต:** องค์ความรู้ที่พร้อมเผยแพร่ พร้อมที่มาและการอ้างอิงครบถ้วน

### 10.10 สำนักสถาปัตยกรรม (Architecture Office)

- **ภารกิจ:** ดูแลความสอดคล้องเชิงโครงสร้างขององค์กรและขององค์ความรู้กับเอกสารระดับสถาปัตยกรรม
- **ขอบเขต:** การกำกับให้การจัดโครงสร้างเป็นไปตามสถาปัตยกรรมที่วางไว้ การทบทวนผลกระทบเชิงโครงสร้าง
- **ปัจจัยนำเข้า:** ข้อเสนอและผลผลิตที่มีนัยต่อโครงสร้างจากสำนักงานต่าง ๆ
- **ผลผลิต:** ข้อวินิจฉัยความสอดคล้องเชิงโครงสร้าง และข้อเสนอแนะเชิงสถาปัตยกรรม

### 10.11 สำนักปฏิบัติการ (Operations Office)

- **ภารกิจ:** ประสานการทำงานขององค์กร ติดตามเหตุการณ์ และดูแลให้องค์กรเดินตามระเบียบ
- **ขอบเขต:** การติดตามสถานะงาน การดูแลคิวงาน การบันทึกเหตุการณ์ และการสนับสนุนผู้บริหารงานองค์กร
- **ปัจจัยนำเข้า:** เหตุการณ์และสถานะงานจากทุกสำนักงาน
- **ผลผลิต:** ภาพรวมสถานะองค์กร รายงานการดำเนินงาน และการแจ้งเตือนเมื่อมีปัญหา

## 11. Chief Knowledge Orchestrator — ผู้บริหารงานองค์กร

**ผู้บริหารงานองค์กร (Chief Knowledge Orchestrator)** คือผู้กำกับงานของสำนักงานทั้งหมด เป็นตำแหน่งสูงสุดในสายการบังคับบัญชาของการดำเนินงาน

**สิ่งที่ผู้บริหารงานองค์กรเป็น**

ผู้บริหารงานองค์กรเป็น **ผู้บริหารกระแสงาน (Workflow)** — กำหนดว่างานควรไหลจากสำนักงานใดไปสำนักงานใด **ผู้จัดลำดับคิวงาน** — ตัดสินว่างานใดควรทำก่อนหลัง **ผู้ติดตามงาน** — เฝ้าดูความคืบหน้าของทุกสำนักงาน **ผู้จัดการเหตุการณ์** — รับและกระจายเหตุการณ์ให้สำนักงานที่เกี่ยวข้อง **ผู้จัดทำรายงาน** — สรุปสถานะและผลงานขององค์กร และ **ผู้ประสานงานระหว่างสำนักงาน** — ทำให้สำนักงานต่าง ๆ ทำงานสอดประสานเป็นหนึ่งเดียว

**สิ่งที่ผู้บริหารงานองค์กรไม่เป็น**

ผู้บริหารงานองค์กร **ไม่ใช่ผู้สร้างองค์ความรู้** — ไม่ลงไปเรียบเรียงเนื้อหา ไม่ตีความหลักธรรม และ **ไม่มีอำนาจอนุมัติองค์ความรู้** อำนาจการอนุมัติองค์ความรู้สงวนไว้สำหรับมนุษย์เสมอ บทบาทของผู้บริหารงานองค์กรคือ *การจัดการการไหลของงาน* ไม่ใช่ *การตัดสินเนื้อหา*

```
   ผู้บริหารงานองค์กร (Chief Knowledge Orchestrator)
        ├── บริหารกระแสงาน        (จัดเส้นทางการไหลของงาน)
        ├── จัดลำดับคิวงาน         (ตัดสินก่อนหลัง)
        ├── ติดตามงาน             (เฝ้าดูความคืบหน้า)
        ├── จัดการเหตุการณ์         (รับและกระจายเหตุการณ์)
        ├── จัดทำรายงาน           (สรุปสถานะองค์กร)
        └── ประสานงานระหว่างสำนักงาน (ทำให้องค์กรเป็นหนึ่งเดียว)
```

> ผู้บริหารงานองค์กรเปรียบดั่งเลขาธิการที่จัดการให้ทุกสำนักงานทำงานประสานกัน แต่ **ไม่ใช่ผู้ชี้ขาดเนื้อหาพระธรรม** การตัดสินเนื้อหาและการอนุมัติเป็นของมนุษย์ตามหลักการในเอกสารแม่

## 12. Agent คืออะไร

**ตัวแทนงาน (Agent)** คือหน่วยที่รับผิดชอบ **ภารกิจเฉพาะหนึ่ง** ภายในสำนักงาน ตัวแทนงานหนึ่งตัวมุ่งทำภารกิจเดียวให้สำเร็จ และอยู่ภายใต้การกำกับของสำนักงานที่ตนสังกัด

- ตัวแทนงานทำงาน **ภายในขอบเขตของสำนักงานเดียว** ไม่ข้ามไปทำงานของสำนักงานอื่น
- ตัวแทนงานรับภารกิจจากสำนักงานของตน และมอบหมายงานย่อยให้หน่วยปฏิบัติ
- ตัวแทนงาน **ไม่มีอำนาจอนุมัติองค์ความรู้** ผลงานของตัวแทนงานต้องผ่านคุณภาพและการทบทวนตามสายงาน

> เอกสารฉบับนี้กำหนดเพียงว่า "ตัวแทนงานคืออะไร" ในระดับแนวคิด การออกแบบตัวแทนงานแต่ละตัวในรายละเอียดเป็นเรื่องของเอกสาร PKA-OPS ฉบับถัดไป

## 13. Worker คืออะไร

**หน่วยปฏิบัติ (Worker)** คือหน่วยที่เล็กที่สุดในองค์กร ลงมือทำ **งานหนึ่งหน่วย** ตามที่ตัวแทนงานมอบหมาย แล้วส่งผลกลับขึ้นไปตามสาย

- หน่วยปฏิบัติทำงานที่ชัดเจนและจำกัดขอบเขต ไม่ตัดสินใจเชิงนโยบาย
- หน่วยปฏิบัติส่งผลงานกลับไปยังตัวแทนงานที่มอบหมาย ไม่ส่งข้ามสำนักงาน
- หน่วยปฏิบัติ **ไม่มีอำนาจอนุมัติองค์ความรู้** เช่นเดียวกับทุกองค์ประกอบที่ไม่ใช่มนุษย์ผู้ทบทวน

ความสัมพันธ์ระหว่างสามชั้น: **สำนักงานมอบภารกิจให้ตัวแทนงาน ตัวแทนงานมอบงานย่อยให้หน่วยปฏิบัติ หน่วยปฏิบัติลงมือทำแล้วส่งผลกลับ** เป็นวงจรที่ความรับผิดชอบไหลลงและผลงานไหลขึ้น

## 14. Queue คืออะไร

**คิวงาน (Queue)** คือลำดับของงานที่รอการดำเนินการ เป็นแนวคิดที่ทำให้องค์กรจัดการปริมาณงานได้อย่างมีระเบียบ โดยไม่ต้องทำทุกอย่างพร้อมกัน

- คิวงานทำให้ **จัดลำดับก่อนหลัง** ได้ตามความสำคัญ ความถูกต้องที่เร่งด่วนมาก่อนงานทั่วไป
- คิวงานทำให้องค์กร **รองรับงานปริมาณมาก** ได้โดยไม่สับสน งานที่ยังไม่ถึงคิวจะรออย่างเป็นระเบียบ
- คิวงานทำให้ **ติดตามสถานะ** ได้ว่ามีงานใดรออยู่ กำลังทำ หรือเสร็จแล้ว

> คิวงานในเอกสารนี้เป็น **หลักการเชิงองค์กร** ว่าด้วยการจัดลำดับงานที่รอ ไม่ใช่กลไกทางเทคนิคใด ๆ

## 15. Workflow คืออะไร (Conceptual Workflow)

**กระแสงาน (Workflow)** คือเส้นทางเชิงแนวคิดที่งานไหลผ่านสำนักงานต่าง ๆ ตั้งแต่แหล่งข้อมูลต้นทางจนถึงการเผยแพร่ กระแสงานหลักขององค์กรเป็นดังนี้

```
   แหล่งข้อมูลต้นทาง (Source)
            ▼
   คลังคัมภีร์ (Corpus)
            ▼
   คลังศัพท์ (Lexicon)
            ▼
   องค์ความรู้ (Knowledge)
            ▼
   ความสัมพันธ์ (Relationship)
            ▼
   การอ้างอิง (Citation)
            ▼
   คุณภาพ (Quality)
            ▼
   การทบทวนโดยมนุษย์ (Review)
            ▼
   การเผยแพร่ (Publish)
```

แต่ละขั้นรับงานจากขั้นก่อนหน้า เพิ่มคุณค่าในส่วนที่ตนรับผิดชอบ แล้วส่งต่อไปขั้นถัดไป โดย **ขั้นการทบทวนโดยมนุษย์ (Review) เป็นด่านบังคับก่อนการเผยแพร่เสมอ** ไม่มีองค์ความรู้ใดข้ามขั้นนี้ไปสู่การเผยแพร่ได้

> กระแสงานนี้เป็น **แนวคิด (Conceptual)** อธิบายลำดับของงานในระดับองค์กร **ไม่กล่าวถึงระบบหรือวิธีการทำงานเชิงเทคนิคใด ๆ** ลำดับนี้สะท้อนสายธารขององค์ความรู้ตามเอกสารแม่ ที่ทุกขั้นต้องรักษาความเชื่อมโยงกลับไปยังแหล่งข้อมูลต้นทาง

## 16. Event คืออะไร (Event-Driven Principle)

**เหตุการณ์ (Event)** คือสัญญาณที่บอกว่ามีบางสิ่งเกิดขึ้นและพร้อมให้สำนักงานที่เกี่ยวข้องรับไปทำงานต่อ องค์กรนี้ยึดหลัก **การประสานงานผ่านเหตุการณ์ (Event-Driven)**

หลักการสำคัญคือ

- **ทุกสำนักงานรับงานจากเหตุการณ์** ไม่เรียกสำนักงานอื่นโดยตรง เมื่อสำนักงานหนึ่งทำงานเสร็จ จะ "ประกาศเหตุการณ์" ว่างานพร้อมแล้ว สำนักงานถัดไปที่รอเหตุการณ์นั้นจึงรับไปทำต่อ
- **สำนักงานไม่ผูกติดกันโดยตรง** การประสานผ่านเหตุการณ์ทำให้สำนักงานหนึ่งไม่ต้องรู้รายละเอียดภายในของอีกสำนักงาน รู้เพียงว่ามีเหตุการณ์ใดที่ตนต้องรับ
- **องค์กรขยายได้โดยไม่ต้องรื้อโครงสร้าง** เมื่อจะเพิ่มสำนักงานใหม่ในอนาคต เพียงให้สำนักงานนั้นรับฟังเหตุการณ์ที่เกี่ยวข้อง โดยไม่กระทบสำนักงานเดิม

> หลักการประสานงานผ่านเหตุการณ์เป็น **เพียงแนวคิดเชิงองค์กร** เพื่อความยืดหยุ่นและการเติบโต **ไม่ใช่การนำไปปฏิบัติเชิงเทคนิค** การที่สำนักงานไม่เรียกกันโดยตรงคือสิ่งที่ทำให้องค์กรนี้รองรับการเปลี่ยนแปลงในระยะยาวได้

## 17. การส่งต่องาน (Handoff)

**การส่งต่องาน (Handoff)** คือการที่ผลผลิตของสำนักงานหนึ่งถูกส่งต่อไปยังสำนักงานถัดไปในกระแสงาน หลักการของการส่งต่องานมีดังนี้

1. **ส่งต่อเมื่อพร้อมเท่านั้น** — ผลผลิตต้องผ่านเกณฑ์คุณภาพของสำนักงานต้นทางก่อนจึงส่งต่อได้
2. **ส่งต่อผ่านเหตุการณ์** — การส่งต่อเกิดผ่านการประกาศเหตุการณ์ ไม่ใช่การเรียกสำนักงานปลายทางโดยตรง
3. **ส่งต่อพร้อมที่มา** — ทุกการส่งต่อต้องแนบที่มาและสถานะของงาน เพื่อให้สำนักงานปลายทางสืบย้อนได้
4. **ความรับผิดชอบยังคงอยู่** — สำนักงานต้นทางยังรับผิดชอบต่อคุณภาพของสิ่งที่ตนส่งต่อ แม้งานจะอยู่ในมือสำนักงานถัดไปแล้ว

## 18. การอนุมัติ (Approval)

**การอนุมัติ (Approval)** คือการตัดสินว่าผลผลิตหนึ่งผ่านเกณฑ์และพร้อมก้าวสู่ขั้นถัดไปหรือไม่ องค์กรนี้แยกการอนุมัติเป็นสองชนิดที่ต่างกันโดยสิ้นเชิง

- **การอนุมัติเชิงดำเนินงาน (Operational Approval)** — การยืนยันว่างานผ่านเกณฑ์ของขั้นตอนหนึ่ง เช่น ผ่านเกณฑ์คุณภาพ ผ่านความครบของการอ้างอิง การอนุมัติชนิดนี้อยู่ในสายงานปกติขององค์กร
- **การอนุมัติองค์ความรู้ (Knowledge Approval)** — การตัดสินว่าองค์ความรู้หนึ่งถูกต้องตามพระธรรมและพร้อมเผยแพร่ **การอนุมัติชนิดนี้เป็นอำนาจของมนุษย์เท่านั้น**

> **เส้นแบ่งที่ห้ามล่วงล้ำ:** ผู้ช่วยอัจฉริยะ ตัวแทนงาน หน่วยปฏิบัติ และผู้บริหารงานองค์กร อาจทำการอนุมัติเชิงดำเนินงานได้ แต่ **ไม่มีองค์ประกอบใดนอกจากมนุษย์ที่อนุมัติองค์ความรู้เพื่อเผยแพร่ได้** (ดูหัวข้อ [19](#19-human-in-the-loop--มนุษย์ในวงจรการตัดสินใจ) และ [20](#20-ธรรมาภิบาลปัญญาประดิษฐ์-ai-governance))

## 19. Human in the Loop — มนุษย์ในวงจรการตัดสินใจ

หลักการ **มนุษย์ในวงจรการตัดสินใจ (Human in the Loop)** เป็นหัวใจของธรรมาภิบาลในองค์กรนี้ และสืบเนื่องโดยตรงจากหลักการด้าน AI ในเอกสารแม่

**สิ่งที่ผู้ช่วยอัจฉริยะ (AI) ไม่มีสิทธิ์ทำ**

1. **ไม่มีสิทธิ์อนุมัติองค์ความรู้** — การตัดสินว่าองค์ความรู้พร้อมเผยแพร่เป็นอำนาจของมนุษย์เสมอ
2. **ไม่มีสิทธิ์เปลี่ยนแหล่งข้อมูลหลัก** — ลำดับชั้นและแหล่งข้อมูลต้นทางเป็นเรื่องที่มนุษย์เท่านั้นกำหนดและแก้ไข
3. **ไม่มีสิทธิ์สร้างหลักธรรมใหม่** — เป็นข้อห้ามเด็ดขาดตามเอกสารแม่ ผู้ช่วยอัจฉริยะเป็นผู้ช่วยค้นหา อธิบาย และเชื่อมโยงเท่านั้น

**สิ่งที่ต้องผ่านมนุษย์เสมอ**

- ทุกองค์ความรู้ต้องผ่าน **การทบทวนโดยมนุษย์ (Human Review)** ก่อนเผยแพร่ — เป็นด่านบังคับที่ข้ามไม่ได้
- การตัดสินใจที่กระทบความถูกต้องของพระธรรมต้องมีมนุษย์เป็นผู้รับผิดชอบสูงสุด

> มนุษย์ในวงจรการตัดสินใจไม่ใช่พิธีการ แต่เป็นหลักประกันสูงสุดของความถูกต้อง องค์กรนี้ออกแบบให้ **ไม่มีทางลัด** ที่ให้องค์ความรู้ออกสู่การเผยแพร่โดยไม่ผ่านสายตามนุษย์

## 20. ธรรมาภิบาลปัญญาประดิษฐ์ (AI Governance)

องค์กรนี้กำหนดกรอบธรรมาภิบาลสำหรับผู้ช่วยอัจฉริยะ เพื่อให้บทบาทของผู้ช่วยอัจฉริยะอยู่ในขอบเขตที่เอกสารแม่กำหนดไว้เสมอ

1. **ผู้ช่วยอัจฉริยะเป็นผู้ช่วย ไม่ใช่ผู้มีอำนาจ** — ทำหน้าที่ค้นหา อธิบาย เชื่อมโยง และเตรียมงาน แต่ไม่ตัดสินเนื้อหาพระธรรมและไม่อนุมัติองค์ความรู้
2. **ผลผลิตของผู้ช่วยอัจฉริยะอยู่ในลำดับชั้นต่ำสุด** — ตามลำดับชั้นความน่าเชื่อถือในเอกสารแม่ และต้องอ้างอิงกลับไปยังระดับที่สูงกว่าได้เสมอ
3. **โปร่งใสว่าสิ่งใดมาจากผู้ช่วยอัจฉริยะ** — ผู้ใช้และผู้ทบทวนต้องแยกออกได้เสมอว่าสิ่งใดเป็นผลของผู้ช่วยอัจฉริยะ สิ่งใดมาจากแหล่งข้อมูลต้นทาง
4. **ไม่กลบเกลื่อนความไม่แน่นอน** — เมื่อผู้ช่วยอัจฉริยะไม่มีข้อมูลหรือไม่แน่ใจ ต้องแสดงสถานะนั้นตามจริง
5. **อยู่ใต้การกำกับขององค์กร** — บทบาทของผู้ช่วยอัจฉริยะอยู่ภายใต้สายการบังคับบัญชาและการกำกับของสำนักงานที่เกี่ยวข้องเสมอ

## 21. ธรรมาภิบาลองค์ความรู้ (Knowledge Governance)

ธรรมาภิบาลองค์ความรู้กำกับว่าองค์ความรู้ในองค์กรนี้ถูกสร้าง ตรวจสอบ และดูแลอย่างไรให้คงความถูกต้องตลอดอายุการใช้งาน

1. **แหล่งข้อมูลต้นทางเดียวเป็นที่ตั้ง** — องค์ความรู้แต่ละชิ้นต้องสืบกลับไปยังแหล่งข้อมูลต้นทางเดียวได้ ไม่มีต้นฉบับซ้อนกัน
2. **แยกพุทธพจน์ออกจากคำอธิบาย** — เส้นแบ่งระหว่างเนื้อหาต้นทางกับคำอธิบายของโครงการต้องชัดเจนเสมอ
3. **ความสัมพันธ์ต้องมีหลักฐาน** — การเชื่อมโยงองค์ความรู้ทุกเส้นต้องมีหลักฐานรองรับ
4. **การเปลี่ยนแปลงต้องมีร่องรอย** — การแก้ไของค์ความรู้ต้องบันทึกเหตุผลและร่องรอยให้ทบทวนได้
5. **เนื้อหาที่สืบย้อนไม่ได้ยังไม่พร้อมเผยแพร่** — สอดคล้องกับหลักการ Data Lineage ในเอกสารแม่

## 22. การจัดการคุณภาพ (Quality Management)

การจัดการคุณภาพเป็นหน้าที่ที่กระจายอยู่ในทุกสำนักงาน และมี **สำนักคุณภาพ** เป็นผู้กำกับมาตรฐานกลาง

- **คุณภาพเป็นเงื่อนไขของการส่งต่อ** — ไม่มีผลผลิตใดส่งต่อได้หากยังไม่ผ่านเกณฑ์คุณภาพของขั้นตอนนั้น
- **เกณฑ์คุณภาพต้องชัดเจนและสม่ำเสมอ** — ทุกสำนักงานใช้เกณฑ์ที่สอดคล้องกัน เพื่อให้คุณภาพทั้งองค์กรอยู่ในระดับเดียวกัน
- **งานที่ไม่ผ่านต้องถูกคืน** — ผลผลิตที่ไม่ผ่านเกณฑ์ถูกส่งกลับไปแก้ไข พร้อมเหตุผลที่ชัดเจน ไม่ปล่อยผ่าน
- **คุณภาพวัดที่ความถูกต้องเป็นหลัก** — เกณฑ์คุณภาพสูงสุดคือความถูกต้องของพระธรรมและความครบของการอ้างอิง

## 23. การตรวจสอบ (Audit)

การตรวจสอบทำให้องค์กรนี้ตรวจทานตนเองได้ และทำให้ทุกการกระทำมีร่องรอยที่สืบย้อนได้

1. **ทุกขั้นมีร่องรอย** — งานที่ผ่านแต่ละสำนักงานต้องทิ้งร่องรอยว่าใครทำ ทำเมื่อใด และผ่านการอนุมัติชนิดใด
2. **สืบย้อนได้ถึงต้นทาง** — การตรวจสอบต้องตามรอยองค์ความรู้กลับไปถึงแหล่งข้อมูลต้นทางได้เสมอ
3. **ตรวจสอบการอนุมัติของมนุษย์** — ต้องตรวจได้ว่าองค์ความรู้ที่เผยแพร่ผ่านการทบทวนของมนุษย์จริง
4. **ผลการตรวจสอบนำไปปรับปรุง** — สิ่งที่พบจากการตรวจสอบต้องนำกลับเข้าสู่การพัฒนาอย่างต่อเนื่อง

## 24. ดัชนีชี้วัดผลงาน (KPI)

องค์กรวัดผลงานด้วยดัชนีที่สะท้อนคุณค่าหลักของโครงการ โดย **ความถูกต้องเป็นดัชนีสูงสุด** เหนือดัชนีด้านปริมาณหรือความเร็ว

| ด้าน | สิ่งที่วัด |
|---|---|
| ความถูกต้อง | สัดส่วนองค์ความรู้ที่ถูกต้องตามพระธรรมและไม่มีข้อคลาดเคลื่อน |
| การตรวจสอบย้อนกลับ | สัดส่วนองค์ความรู้ที่สืบย้อนถึงแหล่งข้อมูลต้นทางได้ครบ |
| ความครบของการอ้างอิง | สัดส่วนองค์ความรู้ที่มีการอ้างอิงครบถ้วน |
| การผ่านการทบทวนของมนุษย์ | สัดส่วนองค์ความรู้ที่เผยแพร่หลังผ่านการทบทวนของมนุษย์ |
| ความราบรื่นของการส่งต่อ | ความคล่องของการไหลของงานระหว่างสำนักงาน |

> ดัชนีด้านปริมาณและความเร็วมีไว้เพื่อช่วยบริหารงาน แต่ **ต้องไม่ถูกใช้ผลักดันจนกระทบความถูกต้อง** เมื่อดัชนีด้านปริมาณขัดกับความถูกต้อง ให้ยึดความถูกต้องเสมอ

## 25. หลักการความมั่นคงปลอดภัย (Security Principles)

องค์กรนี้ถือว่าความถูกต้องและความน่าเชื่อถือขององค์ความรู้เป็นสิ่งที่ต้องปกป้อง หลักการความมั่นคงปลอดภัยในระดับองค์กรมีดังนี้

1. **ปกป้องความครบถ้วนของแหล่งข้อมูลต้นทาง** — เนื้อหาต้นทางต้องไม่ถูกแก้ไขโดยไม่มีอำนาจ และต้องคงไว้ตามต้นฉบับ
2. **คุ้มครองเส้นแบ่งการอนุมัติ** — อำนาจการอนุมัติองค์ความรู้ของมนุษย์ต้องไม่ถูกล่วงล้ำหรือข้ามผ่านได้
3. **จำกัดสิทธิ์ตามหน้าที่** — แต่ละองค์ประกอบขององค์กรมีสิทธิ์เท่าที่หน้าที่ของตนต้องการ ไม่เกินกว่านั้น
4. **รักษาร่องรอยให้แก้ไขไม่ได้** — ร่องรอยการตรวจสอบต้องเชื่อถือได้และไม่ถูกลบหรือแก้โดยไม่มีร่องรอย
5. **โปร่งใสเมื่อมีเหตุผิดปกติ** — เมื่อพบเหตุที่กระทบความน่าเชื่อถือ ต้องเปิดเผยและจัดการตามระเบียบ

> หลักการเหล่านี้เป็น **หลักการระดับองค์กร** ว่าด้วยการปกป้องความถูกต้องและความน่าเชื่อถือ **ไม่ใช่มาตรการทางเทคนิคใด ๆ**

## 26. การพัฒนาอย่างต่อเนื่อง (Continuous Improvement)

องค์กรนี้ออกแบบให้เรียนรู้และพัฒนาตนเองอย่างต่อเนื่อง โดยไม่ละทิ้งหลักการที่วางไว้

1. **เรียนรู้จากการตรวจสอบ** — สิ่งที่พบจากการตรวจสอบและการทบทวนของมนุษย์ต้องนำกลับมาปรับปรุงการทำงาน
2. **ปรับปรุงโดยไม่ลดทอนความถูกต้อง** — การพัฒนาทุกอย่างต้องไม่ทำให้ความถูกต้องของพระธรรมอ่อนลง
3. **ขยายองค์กรอย่างมีระเบียบ** — การเพิ่มสำนักงานหรือบทบาทใหม่ต้องคงหลักการ "หนึ่งหน้าที่ หนึ่งเจ้าภาพ" และการประสานผ่านเหตุการณ์
4. **บันทึกการเปลี่ยนแปลง** — การเปลี่ยนแปลงเชิงองค์กรต้องมีร่องรอยและเหตุผลในประวัติการเปลี่ยนแปลงของเอกสารตระกูล OPS
5. **พร้อมเติบโตสู่อนาคต** — โครงสร้างองค์กรนี้ออกแบบให้รองรับการมีผู้ช่วยอัจฉริยะทำงานร่วมในอนาคต โดยไม่ต้องเปลี่ยนโครงสร้างองค์กร

## 27. ความสัมพันธ์กับเอกสารเดิม

ตระกูล **PKA-OPS** เป็น **ชั้นปฏิบัติการ (Operational Layer)** ของโครงการ ซึ่งอยู่ **คู่กับ** ตระกูลมาตรฐาน (Standards) สถาปัตยกรรม (Architecture) แบบจำลองโดเมน (Domain) และแหล่งอ้างอิง (Reference) โดย **ไม่แทนที่เอกสารใด** และไม่เปลี่ยนแปลงหลักการที่เอกสารเหล่านั้นวางไว้

OPS ตอบคำถามที่ต่างจากเอกสารตระกูลอื่น — ขณะที่ STD กำหนด *กฎ* ARC อธิบาย *ภาพรวมของระบบ* DOM กำหนด *แบบจำลองขององค์ความรู้* และ REF อธิบาย *แหล่งอ้างอิง* ตระกูล OPS ตอบว่า **"องค์กรจัดระเบียบและดำเนินงานอย่างไร"** เพื่อให้หลักการทั้งหมดถูกนำไปปฏิบัติได้จริงโดยมีเจ้าภาพและสายความรับผิดชอบที่ชัดเจน

เอกสารฉบับนี้อ้างอิงและต้องสอดคล้องกับเอกสารต่อไปนี้

- **[PKA-STD-001 — ธรรมนูญโครงการ](../standards/PKA-STD-001-ธรรมนูญโครงการ.md)** (เอกสารแม่) — หลักการสูงสุดทั้งหมด รวมถึงหลักการด้าน AI ลำดับชั้นแหล่งข้อมูล และหลักการตัดสินใจ
- **[PKA-STD-002 — มาตรฐานข้อมูล](../standards/PKA-STD-002-มาตรฐานข้อมูล.md)** — หลักการด้านข้อมูลที่สำนักงานในสายข้อมูลต้องยึดถือ
- **[PKA-STD-003 — มาตรฐานการอ้างอิง](../standards/PKA-STD-003-มาตรฐานการอ้างอิง.md)** — หลักการที่สำนักการอ้างอิงต้องปฏิบัติตาม
- **[PKA-ARC-001 — สถาปัตยกรรมองค์รวมของโครงการ](../architecture/PKA-ARC-001-สถาปัตยกรรมองค์รวมของโครงการ.md)** — ภาพรวมของระบบที่องค์กรนี้ดำเนินงานอยู่ภายใน
- **[PKA-ARC-004 — สถาปัตยกรรมเชิงความหมาย](../architecture/PKA-ARC-004-สถาปัตยกรรมเชิงความหมาย.md)** — กรอบเชิงความหมายที่สำนักความสัมพันธ์และสำนักองค์ความรู้ทำงานสอดคล้องด้วย

> **ลำดับชั้นยังคงเดิม:** เมื่อใดที่เนื้อหาของ OPS ขัดกับเอกสารระดับสูงกว่า (โดยเฉพาะเอกสารแม่) ให้ยึดเอกสารระดับสูงกว่าเสมอ OPS เป็นชั้นปฏิบัติการที่ **เคารพทุกระดับเหนือขึ้นไป**

## 28. นิยามคำศัพท์

| คำศัพท์ | ความหมาย |
|---|---|
| **สำนักงานองค์ความรู้ดิจิทัล (Digital Knowledge Office)** | องค์กรที่โครงการ PKA จัดตั้งขึ้นเพื่อดำเนินงานองค์ความรู้อย่างเป็นระบบ ประกอบด้วยสำนักงานหลายสำนักงานภายใต้ผู้บริหารงานองค์กร |
| **สำนักงาน (Office)** | หน่วยที่รับผิดชอบงานหนึ่งด้านขององค์กรอย่างเต็มตัว มีหน้าที่ ขอบเขต ปัจจัยนำเข้า และผลผลิตที่ชัดเจน |
| **ตัวแทนงาน (Agent)** | หน่วยที่รับผิดชอบภารกิจเฉพาะหนึ่งภายในสำนักงาน |
| **หน่วยปฏิบัติ (Worker)** | หน่วยที่เล็กที่สุด ลงมือทำงานหนึ่งหน่วยตามที่ได้รับมอบหมาย |
| **ผู้บริหารงานองค์กร (Chief Knowledge Orchestrator)** | ผู้กำกับ ประสาน และติดตามงานของทั้งองค์กร เป็นผู้บริหารกระแสงาน มิใช่ผู้สร้างหรืออนุมัติองค์ความรู้ |
| **คิวงาน (Queue)** | ลำดับของงานที่รอการดำเนินการ เป็นแนวคิดสำหรับการจัดลำดับงานในระดับองค์กร |
| **กระแสงาน (Workflow)** | เส้นทางเชิงแนวคิดที่งานไหลผ่านสำนักงานต่าง ๆ จากแหล่งข้อมูลต้นทางสู่การเผยแพร่ |
| **เหตุการณ์ (Event)** | สัญญาณที่บอกว่ามีบางสิ่งเกิดขึ้นและพร้อมให้สำนักงานที่เกี่ยวข้องรับไปทำงานต่อ |
| **การส่งต่องาน (Handoff)** | การส่งผลผลิตจากสำนักงานหนึ่งไปยังสำนักงานถัดไปผ่านเหตุการณ์ |
| **การอนุมัติเชิงดำเนินงาน (Operational Approval)** | การยืนยันว่างานผ่านเกณฑ์ของขั้นตอนหนึ่ง อยู่ในสายงานปกติขององค์กร |
| **การอนุมัติองค์ความรู้ (Knowledge Approval)** | การตัดสินว่าองค์ความรู้พร้อมเผยแพร่ เป็นอำนาจของมนุษย์เท่านั้น |
| **มนุษย์ในวงจรการตัดสินใจ (Human in the Loop)** | หลักการที่กำหนดให้มนุษย์เป็นผู้รับผิดชอบสูงสุดและเป็นด่านบังคับก่อนการเผยแพร่ |
| **ชั้นปฏิบัติการ (Operational Layer)** | ชั้นของเอกสารตระกูล OPS ที่อยู่คู่กับมาตรฐาน สถาปัตยกรรม โดเมน และแหล่งอ้างอิง โดยไม่แทนที่เอกสารใด |

## 29. ประวัติการเปลี่ยนแปลง

| Version | Date | Author | Change |
|---|---|---|---|
| Draft 0.1 | 2026-06-27 | ทีมโครงการ PKA | ฉบับร่างแรกของธรรมนูญสำนักงานดิจิทัล — จัดตั้งตระกูลเอกสาร PKA-OPS (ชั้นปฏิบัติการ); กำหนดโครงสร้างองค์กรสามชั้น (สำนักงาน → ตัวแทนงาน → หน่วยปฏิบัติ) และผู้บริหารงานองค์กร; นิยามสำนักงานหลัก 11 สำนักงานพร้อมภารกิจ/ขอบเขต/ปัจจัยนำเข้า/ผลผลิต; วางหลักการการประสานงานผ่านเหตุการณ์ การส่งต่องาน การอนุมัติ มนุษย์ในวงจรการตัดสินใจ ธรรมาภิบาล AI/องค์ความรู้ คุณภาพ การตรวจสอบ KPI ความมั่นคงปลอดภัย และการพัฒนาต่อเนื่อง ครบ 29 หัวข้อ |
