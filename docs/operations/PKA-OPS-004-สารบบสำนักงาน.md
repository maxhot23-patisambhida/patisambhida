## 1. วัตถุประสงค์ (Purpose)

เอกสารฉบับนี้รวบรวม **สารบบสำนักงาน (Office Catalog)** ของสำนักงานองค์ความรู้ดิจิทัล โดยลงรายละเอียดของแต่ละสำนักงานที่จัดตั้งไว้ใน [PKA-OPS-001](PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md) และวางโครงสร้างองค์กรไว้ใน [PKA-OPS-002](PKA-OPS-002-โครงสร้างองค์กรและสายการบังคับบัญชา.md) — แต่ออกแบบใหม่โดยยึด **ขีดความสามารถ (Capability)** จาก [PKA-OPS-003](PKA-OPS-003-แบบจำลองขีดความสามารถขององค์กร.md) เป็นจุดเริ่ม

เอกสารนี้มีไว้เพื่อ

1. ระบุว่าแต่ละสำนักงาน **รองรับขีดความสามารถใดบ้าง** และ **เป็นเจ้าของขีดความสามารถใด**
2. ทำให้ทุกสำนักงาน **สืบย้อนกลับไปยังขีดความสามารถใน OPS-003 ได้** อย่างครบถ้วน
3. ยืนยันว่า **ทุกขีดความสามารถมีสำนักงานเจ้าของเพียงหนึ่งเดียว** ตามหลักใน OPS-003
4. ลงรายละเอียด **ภารกิจ ขอบเขต ปัจจัยนำเข้า และผลผลิต** ของแต่ละสำนักงานในระดับแนวคิด

> เมื่ออ่านเอกสารฉบับนี้จบ ผู้อ่านควรเห็นชัดว่า **ขีดความสามารถแต่ละรายการขององค์กรมีสำนักงานใดรับผิดชอบ** และสำนักงานแต่ละแห่ง **รองรับและเป็นเจ้าของขีดความสามารถใด** โดยสืบย้อนกลับสู่ OPS-003 ได้ทุกจุด

## 2. ขอบเขต (Scope)

**อยู่ในขอบเขต**

- รายการสำนักงานทั้งหมดขององค์กร พร้อมขีดความสามารถที่รองรับและเป็นเจ้าของ
- ภารกิจ ขอบเขต ปัจจัยนำเข้า และผลผลิตของแต่ละสำนักงานในระดับแนวคิด
- การสืบย้อนความสัมพันธ์ระหว่างสำนักงานกับขีดความสามารถใน OPS-003

**อยู่นอกขอบเขต**

- การมอบหมายขีดความสามารถสู่การปฏิบัติและการอนุมัติโดยละเอียด (อยู่ใน **PKA-OPS-005 — Capability Assignment Matrix**)
- หน้าที่ทางธุรกิจ กระแสงาน เหตุการณ์ ตัวแทนงาน และหน่วยปฏิบัติ (หน้าที่ทางธุรกิจอยู่ใน OPS-005A; กระแสงานถึงหน่วยปฏิบัติอยู่ใน OPS-006 ถึง OPS-009)
- วิธีการทำงานภายในสำนักงาน ขั้นตอนปฏิบัติ และการวัดผลเชิงตัวเลข
- เครื่องมือ เทคโนโลยี วิธีพัฒนา หรือรายละเอียดเชิงปฏิบัติเชิงเทคนิคใด ๆ

> เอกสารฉบับนี้ตอบว่า **"สำนักงานใดเป็นเจ้าของขีดความสามารถใด"** ส่วนการมอบหมายเชิงลึก (เจ้าของ ผู้สนับสนุน ประเภทตัวแทนงานและหน่วยปฏิบัติในอนาคต และระดับการอนุมัติ) เป็นของ **PKA-OPS-005 — Capability Assignment Matrix**

## 3. หลักขีดความสามารถเป็นที่ตั้ง (Capability First)

สารบบฉบับนี้ออกแบบโดยยึดหลัก **"ขีดความสามารถมาก่อนสำนักงาน"** ตาม [PKA-OPS-003 หัวข้อ 4](PKA-OPS-003-แบบจำลองขีดความสามารถขององค์กร.md#4-เหตุใดองค์กรต้องมีขีดความสามารถก่อนสำนักงาน-capability-before-office)

1. **เริ่มจากขีดความสามารถ ไม่ใช่เริ่มจากสำนักงาน** — ก่อนอธิบายว่าสำนักงานทำอะไร เราระบุก่อนว่าสำนักงานนั้นรองรับขีดความสามารถใดขององค์กร
2. **ทุกสำนักงานต้องผูกกับขีดความสามารถ** — ไม่มีสำนักงานใดดำรงอยู่โดยไม่รองรับขีดความสามารถอย่างน้อยหนึ่งรายการ
3. **ทุกขีดความสามารถต้องมีเจ้าของ** — ขีดความสามารถทุกรายการใน OPS-003 ต้องมีสำนักงานเจ้าของเพียงหนึ่งเดียว ไม่มีขีดความสามารถใดไร้เจ้าภาพ
4. **สืบย้อนได้สองทาง** — จากสำนักงานสืบไปยังขีดความสามารถได้ และจากขีดความสามารถสืบกลับมายังสำนักงานเจ้าของได้

```
   OPS-003 : Capability  (องค์กรทำอะไรได้)
        │   ── ยึดเป็นที่ตั้ง ──
        ▼
   OPS-004 : Office      (สำนักงานใดเป็นเจ้าของและรองรับ)
```

> หลักนี้ทำให้สารบบสำนักงานไม่ใช่เพียง "รายชื่อหน่วยงาน" แต่เป็น **แผนที่ความรับผิดชอบ** ที่บอกว่าความสามารถขององค์กรถูกมอบหมายให้ใครดูแล

## 4. โครงสร้างของแต่ละรายการในสารบบ (Catalog Entry Structure)

ทุกรายการสำนักงานในสารบบนี้อธิบายด้วยโครงสร้างเดียวกันเจ็ดด้าน โดยเริ่มจากขีดความสามารถเสมอ

| ด้าน | คำอธิบาย |
|---|---|
| **ขีดความสามารถที่เป็นเจ้าของ** (Owned Capabilities) | ขีดความสามารถที่สำนักงานนี้เป็นเจ้าของเพียงผู้เดียว |
| **ขีดความสามารถที่รองรับ** (Supported Capabilities) | ขีดความสามารถที่สำนักงานนี้สนับสนุน แม้ไม่ได้เป็นเจ้าของ |
| **ภารกิจ** (Mission) | สิ่งที่สำนักงานมุ่งทำ |
| **ขอบเขต** (Scope) | สิ่งที่อยู่ในความรับผิดชอบ |
| **ปัจจัยนำเข้า** (Input) | สิ่งที่รับเข้ามาทำงาน |
| **ผลผลิต** (Output) | สิ่งที่ส่งมอบออกไป |
| **สืบย้อนสู่ OPS-003** (Trace) | การอ้างอิงกลับไปยังขีดความสามารถต้นทางใน OPS-003 |

> หากสำนักงานใดเป็นเจ้าของ **ขีดความสามารถย่อย (Sub Capability)** ตามลำดับชั้นใน [OPS-003 หัวข้อ 5](PKA-OPS-003-แบบจำลองขีดความสามารถขององค์กร.md#5-ลำดับชั้นของขีดความสามารถ-capability-hierarchy) จะระบุไว้ในด้าน "ขีดความสามารถที่เป็นเจ้าของ" พร้อมขีดความสามารถแม่ที่ขีดความสามารถย่อยนั้นสังกัด

## 5. ภาพรวมความเป็นเจ้าของขีดความสามารถ (Capability Ownership Overview)

ตารางนี้แสดงว่าขีดความสามารถทั้งสิบสองรายการใน [OPS-003 หัวข้อ 15](PKA-OPS-003-แบบจำลองขีดความสามารถขององค์กร.md#15-แผนที่ขีดความสามารถ-capability-map) มีสำนักงานใดเป็นเจ้าของ — ทุกรายการมีเจ้าของเพียงหนึ่งเดียว

| ขีดความสามารถ (OPS-003) | ประเภท | สำนักงานเจ้าของ |
|---|---|---|
| Knowledge Acquisition (การรับองค์ความรู้) | หลัก | สำนักแหล่งข้อมูลต้นทาง |
| Knowledge Processing (การประมวลองค์ความรู้) | หลัก | สำนักองค์ความรู้ |
| Knowledge Search (การค้นหาองค์ความรู้) | หลัก | สำนักองค์ความรู้ |
| Knowledge Validation (การตรวจสอบองค์ความรู้) | หลัก | สำนักการอ้างอิง |
| Knowledge Relationship (การเชื่อมโยงองค์ความรู้) | หลัก | สำนักความสัมพันธ์ |
| Knowledge Publishing (การเผยแพร่องค์ความรู้) | หลัก | สำนักเผยแพร่ |
| Knowledge Analytics (การวิเคราะห์องค์ความรู้) | สนับสนุน | สำนักวิจัย |
| Knowledge Operations (การดำเนินงานองค์ความรู้) | สนับสนุน | สำนักปฏิบัติการ |
| AI Assistance (การช่วยเหลือด้วยผู้ช่วยอัจฉริยะ) | สนับสนุน | สำนักปฏิบัติการ |
| Quality Management (การจัดการคุณภาพ) | สนับสนุน | สำนักคุณภาพ |
| Knowledge Governance (การกำกับองค์ความรู้) | ธรรมาภิบาล | สำนักคุณภาพ |
| Human Collaboration (การทำงานร่วมกับมนุษย์) | ธรรมาภิบาล | สำนักคุณภาพ |

นอกจากขีดความสามารถระดับบนทั้งสิบสองรายการ ยังมีสำนักงานที่เป็นเจ้าของ **ขีดความสามารถย่อย** เพื่อให้ทุกสำนักงานผูกกับขีดความสามารถอย่างชัดเจน

| ขีดความสามารถย่อย | ขีดความสามารถแม่ (OPS-003) | สำนักงานเจ้าของ |
|---|---|---|
| การจัดโครงสร้างคลังคัมภีร์ (Corpus Structuring) | Knowledge Processing | สำนักคลังคัมภีร์ |
| การจัดมาตรฐานศัพท์ (Lexical Standardization) | Knowledge Processing | สำนักคลังศัพท์ |
| การกำกับความสอดคล้องเชิงโครงสร้าง (Structural Consistency) | Knowledge Governance | สำนักสถาปัตยกรรม |

## 6. สารบบสำนักงาน (The Office Catalog)

### 6.1 สำนักแหล่งข้อมูลต้นทาง (Source Office)

- **ขีดความสามารถที่เป็นเจ้าของ:** Knowledge Acquisition (การรับองค์ความรู้)
- **ขีดความสามารถที่รองรับ:** Knowledge Validation, Knowledge Governance
- **ภารกิจ:** ธำรงแหล่งข้อมูลต้นทางและลำดับชั้นความน่าเชื่อถือให้เป็นรากฐานที่มั่นคงขององค์กร
- **ขอบเขต:** การรับ ขึ้นทะเบียน และดูแลแหล่งข้อมูลต้นทาง การกำกับลำดับชั้นความน่าเชื่อถือตามเอกสารแม่
- **ปัจจัยนำเข้า:** แหล่งข้อมูลต้นทางทางพระธรรมและคัมภีร์
- **ผลผลิต:** แหล่งข้อมูลต้นทางที่ขึ้นทะเบียน ระบุลำดับชั้น และพร้อมให้สำนักงานอื่นอ้างอิง
- **สืบย้อนสู่ OPS-003:** Knowledge Acquisition (หลัก)

### 6.2 สำนักคลังคัมภีร์ (Corpus Office)

- **ขีดความสามารถที่เป็นเจ้าของ:** การจัดโครงสร้างคลังคัมภีร์ (Corpus Structuring) — ขีดความสามารถย่อยภายใต้ Knowledge Processing
- **ขีดความสามารถที่รองรับ:** Knowledge Acquisition, Knowledge Processing
- **ภารกิจ:** จัดระเบียบเนื้อหาคัมภีร์ให้เป็นคลังที่ค้นและอ้างอิงได้อย่างเป็นระบบ
- **ขอบเขต:** การจัดโครงสร้างเนื้อหา การแบ่งหมวด การรักษาความครบถ้วนของเนื้อหาต้นทาง
- **ปัจจัยนำเข้า:** แหล่งข้อมูลต้นทางที่ขึ้นทะเบียนจากสำนักแหล่งข้อมูลต้นทาง
- **ผลผลิต:** คลังเนื้อหาคัมภีร์ที่มีโครงสร้างชัดเจน รักษาความครบถ้วนของต้นฉบับ
- **สืบย้อนสู่ OPS-003:** Knowledge Processing (ผ่านขีดความสามารถย่อย Corpus Structuring)

### 6.3 สำนักคลังศัพท์ (Lexicon Office)

- **ขีดความสามารถที่เป็นเจ้าของ:** การจัดมาตรฐานศัพท์ (Lexical Standardization) — ขีดความสามารถย่อยภายใต้ Knowledge Processing
- **ขีดความสามารถที่รองรับ:** Knowledge Processing, Knowledge Search
- **ภารกิจ:** ดูแลศัพท์ คำนิยาม และอภิธานศัพท์มาตรฐานให้สม่ำเสมอทั้งองค์กร
- **ขอบเขต:** การรวบรวมศัพท์ การกำกับคำนิยามที่ถูกต้อง การรักษาความสอดคล้องของการใช้คำ
- **ปัจจัยนำเข้า:** เนื้อหาจากคลังคัมภีร์ และศัพท์ที่สำนักงานอื่นร้องขอ
- **ผลผลิต:** อภิธานศัพท์มาตรฐานและคำนิยามที่ทุกสำนักงานใช้ร่วมกัน
- **สืบย้อนสู่ OPS-003:** Knowledge Processing (ผ่านขีดความสามารถย่อย Lexical Standardization)

### 6.4 สำนักองค์ความรู้ (Knowledge Office)

- **ขีดความสามารถที่เป็นเจ้าของ:** Knowledge Processing (การประมวลองค์ความรู้), Knowledge Search (การค้นหาองค์ความรู้)
- **ขีดความสามารถที่รองรับ:** Knowledge Relationship, Knowledge Validation
- **ภารกิจ:** แปรเนื้อหาคัมภีร์ให้เป็นองค์ความรู้ที่มีโครงสร้าง เข้าถึงและเข้าใจได้ และทำให้ค้นหาได้
- **ขอบเขต:** การจัดโครงสร้างองค์ความรู้ การเรียบเรียงคำอธิบายของโครงการที่แยกชัดจากพุทธพจน์ และการทำให้องค์ความรู้เข้าถึงได้ผ่านการค้นหา
- **ปัจจัยนำเข้า:** เนื้อหาจากคลังคัมภีร์ และศัพท์มาตรฐานจากสำนักคลังศัพท์
- **ผลผลิต:** องค์ความรู้ที่มีโครงสร้าง พร้อมระบุสถานะและที่มาอย่างโปร่งใส และเข้าถึงได้ผ่านการค้นหา
- **สืบย้อนสู่ OPS-003:** Knowledge Processing (หลัก), Knowledge Search (หลัก)

### 6.5 สำนักความสัมพันธ์ (Relationship Office)

- **ขีดความสามารถที่เป็นเจ้าของ:** Knowledge Relationship (การเชื่อมโยงองค์ความรู้)
- **ขีดความสามารถที่รองรับ:** Knowledge Processing, Knowledge Analytics
- **ภารกิจ:** สร้างและดูแลความสัมพันธ์ระหว่างองค์ความรู้ให้ถูกต้องและมีหลักฐานรองรับ
- **ขอบเขต:** การกำหนดความเชื่อมโยงระหว่างหลักธรรม หมวดหมู่ และคัมภีร์ ตามแค็ตตาล็อกชนิดความสัมพันธ์
- **ปัจจัยนำเข้า:** องค์ความรู้ที่มีโครงสร้างจากสำนักองค์ความรู้
- **ผลผลิต:** เครือข่ายความสัมพันธ์ขององค์ความรู้ที่มีหลักฐานกำกับทุกเส้น
- **สืบย้อนสู่ OPS-003:** Knowledge Relationship (หลัก)

### 6.6 สำนักการอ้างอิง (Citation Office)

- **ขีดความสามารถที่เป็นเจ้าของ:** Knowledge Validation (การตรวจสอบองค์ความรู้)
- **ขีดความสามารถที่รองรับ:** Knowledge Governance, Quality Management
- **ภารกิจ:** กำกับให้ทุกองค์ความรู้มีการอ้างอิงที่ครบถ้วนและตรวจสอบย้อนกลับได้
- **ขอบเขต:** การตรวจความครบถ้วนของการอ้างอิง การรักษารูปแบบการอ้างอิงที่สม่ำเสมอ การยืนยันการสืบย้อนถึงแหล่งข้อมูลต้นทาง
- **ปัจจัยนำเข้า:** องค์ความรู้และความสัมพันธ์จากสำนักงานต้นทาง
- **ผลผลิต:** องค์ความรู้ที่มีสายการอ้างอิงครบถ้วน สืบย้อนถึงแหล่งข้อมูลต้นทางได้
- **สืบย้อนสู่ OPS-003:** Knowledge Validation (หลัก)

### 6.7 สำนักคุณภาพ (Quality Office)

- **ขีดความสามารถที่เป็นเจ้าของ:** Quality Management (การจัดการคุณภาพ), Knowledge Governance (การกำกับองค์ความรู้), Human Collaboration (การทำงานร่วมกับมนุษย์)
- **ขีดความสามารถที่รองรับ:** Knowledge Validation
- **ภารกิจ:** ตรวจสอบคุณภาพและความถูกต้องของผลผลิต กำกับธรรมาภิบาลขององค์ความรู้ และธำรงบทบาทมนุษย์ในวงจรการตัดสินใจ
- **ขอบเขต:** การตรวจตามเกณฑ์คุณภาพ การคืนงานที่ไม่ผ่านเกณฑ์ การกำกับการอนุมัติองค์ความรู้ และการรักษาด่านการทบทวนของมนุษย์ก่อนเผยแพร่
- **ปัจจัยนำเข้า:** องค์ความรู้ที่มีการอ้างอิงครบจากสำนักการอ้างอิง
- **ผลผลิต:** ผลการตรวจคุณภาพ องค์ความรู้ที่ผ่านเกณฑ์ และการยืนยันว่าผ่านการทบทวนของมนุษย์
- **สืบย้อนสู่ OPS-003:** Quality Management (สนับสนุน), Knowledge Governance (ธรรมาภิบาล), Human Collaboration (ธรรมาภิบาล)

> สำนักคุณภาพเป็น **ศูนย์กลางธรรมาภิบาล** ขององค์กร อำนาจการอนุมัติองค์ความรู้ยังเป็นของมนุษย์เสมอตามเอกสารแม่ — สำนักคุณภาพ *กำกับ* ให้กระบวนการอนุมัติของมนุษย์เกิดขึ้นจริง แต่ไม่ได้อนุมัติแทนมนุษย์

### 6.8 สำนักวิจัย (Research Office)

- **ขีดความสามารถที่เป็นเจ้าของ:** Knowledge Analytics (การวิเคราะห์องค์ความรู้)
- **ขีดความสามารถที่รองรับ:** Knowledge Processing, Knowledge Relationship
- **ภารกิจ:** ค้นคว้าและวิเคราะห์เพื่อช่วยให้สำนักงานอื่นทำงานได้แม่นยำขึ้น
- **ขอบเขต:** การค้นคว้าเชิงลึก การวิเคราะห์ภาพรวมและคุณภาพขององค์ความรู้ การเสนอแนวการตีความที่มีอยู่โดยไม่ชี้ขาด
- **ปัจจัยนำเข้า:** คำถามและคำร้องขอจากสำนักงานอื่นและผู้บริหารงานองค์กร
- **ผลผลิต:** รายงานการค้นคว้าและการวิเคราะห์ที่ระบุที่มาชัดเจน
- **สืบย้อนสู่ OPS-003:** Knowledge Analytics (สนับสนุน)

### 6.9 สำนักเผยแพร่ (Publishing Office)

- **ขีดความสามารถที่เป็นเจ้าของ:** Knowledge Publishing (การเผยแพร่องค์ความรู้)
- **ขีดความสามารถที่รองรับ:** Knowledge Governance
- **ภารกิจ:** จัดเตรียมองค์ความรู้ที่ได้รับอนุมัติแล้วให้พร้อมเผยแพร่อย่างเคารพต่อเนื้อหา
- **ขอบเขต:** การจัดรูปเพื่อเผยแพร่ การรักษาความครบถ้วนของการอ้างอิงในงานที่เผยแพร่
- **ปัจจัยนำเข้า:** องค์ความรู้ที่ผ่านการทบทวนและอนุมัติโดยมนุษย์
- **ผลผลิต:** องค์ความรู้ที่พร้อมเผยแพร่ พร้อมที่มาและการอ้างอิงครบถ้วน
- **สืบย้อนสู่ OPS-003:** Knowledge Publishing (หลัก)

### 6.10 สำนักสถาปัตยกรรม (Architecture Office)

- **ขีดความสามารถที่เป็นเจ้าของ:** การกำกับความสอดคล้องเชิงโครงสร้าง (Structural Consistency) — ขีดความสามารถย่อยภายใต้ Knowledge Governance
- **ขีดความสามารถที่รองรับ:** Knowledge Relationship, Knowledge Operations
- **ภารกิจ:** ดูแลความสอดคล้องเชิงโครงสร้างขององค์กรและขององค์ความรู้กับเอกสารระดับสถาปัตยกรรม
- **ขอบเขต:** การกำกับให้การจัดโครงสร้างเป็นไปตามสถาปัตยกรรมที่วางไว้ การทบทวนผลกระทบเชิงโครงสร้าง
- **ปัจจัยนำเข้า:** ข้อเสนอและผลผลิตที่มีนัยต่อโครงสร้างจากสำนักงานต่าง ๆ
- **ผลผลิต:** ข้อวินิจฉัยความสอดคล้องเชิงโครงสร้าง และข้อเสนอแนะเชิงสถาปัตยกรรม
- **สืบย้อนสู่ OPS-003:** Knowledge Governance (ผ่านขีดความสามารถย่อย Structural Consistency)

### 6.11 สำนักปฏิบัติการ (Operations Office)

- **ขีดความสามารถที่เป็นเจ้าของ:** Knowledge Operations (การดำเนินงานองค์ความรู้), AI Assistance (การช่วยเหลือด้วยผู้ช่วยอัจฉริยะ)
- **ขีดความสามารถที่รองรับ:** Knowledge Governance, Quality Management
- **ภารกิจ:** ประสานการทำงานขององค์กร ติดตามเหตุการณ์ ดูแลระเบียบ และกำกับการใช้ผู้ช่วยอัจฉริยะภายใต้ธรรมาภิบาล
- **ขอบเขต:** การติดตามสถานะงาน การดูแลคิวงาน การบันทึกเหตุการณ์ การสนับสนุนผู้บริหารงานองค์กร และการประสานการใช้ผู้ช่วยอัจฉริยะให้อยู่ในขอบเขตที่กำหนด
- **ปัจจัยนำเข้า:** เหตุการณ์และสถานะงานจากทุกสำนักงาน
- **ผลผลิต:** ภาพรวมสถานะองค์กร รายงานการดำเนินงาน และการแจ้งเตือนเมื่อมีปัญหา
- **สืบย้อนสู่ OPS-003:** Knowledge Operations (สนับสนุน), AI Assistance (สนับสนุน)

> **ข้อกำกับ AI:** สำนักปฏิบัติการเป็นเจ้าของขีดความสามารถ AI Assistance ในแง่ของ *การประสานและกำกับ* การใช้ผู้ช่วยอัจฉริยะเท่านั้น ผู้ช่วยอัจฉริยะยังคงอยู่ในลำดับชั้นต่ำสุดและ **ไม่มีสิทธิ์อนุมัติองค์ความรู้** ตามเอกสารแม่และ [PKA-OPS-001](PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md)

## 7. เจ้าของและผู้สนับสนุน (Owner vs Supporter)

สารบบนี้แยกบทบาทของสำนักงานต่อขีดความสามารถเป็นสองชนิด เพื่อรักษาหลัก "หนึ่งขีดความสามารถ หนึ่งเจ้าของ" โดยยังเปิดให้เกิดความร่วมมือ

| บทบาท | ความหมาย | จำนวนต่อขีดความสามารถ |
|---|---|---|
| **เจ้าของ** (Owner) | รับผิดชอบให้ขีดความสามารถทำงานได้และมีคุณภาพ | หนึ่งสำนักงานเท่านั้น |
| **ผู้สนับสนุน** (Supporter) | ช่วยให้ขีดความสามารถที่สำนักงานอื่นเป็นเจ้าของทำงานได้ดีขึ้น | มีได้หลายสำนักงาน |

หลักการ

1. **ความเป็นเจ้าของไม่ซ้ำ** — แต่ละขีดความสามารถมีสำนักงานเจ้าของเพียงหนึ่งเดียว ตาม [OPS-003 หัวข้อ 6](PKA-OPS-003-แบบจำลองขีดความสามารถขององค์กร.md#6-ความเป็นเจ้าของขีดความสามารถ-capability-ownership)
2. **การสนับสนุนไม่ลดทอนความเป็นเจ้าของ** — แม้หลายสำนักงานสนับสนุนขีดความสามารถเดียวกัน ความรับผิดชอบสูงสุดยังเป็นของสำนักงานเจ้าของ
3. **ทุกสำนักงานเป็นเจ้าของอย่างน้อยหนึ่งขีดความสามารถ** — ไม่ว่าจะเป็นขีดความสามารถระดับบนหรือขีดความสามารถย่อย เพื่อให้ไม่มีสำนักงานใดดำรงอยู่โดยปราศจากความรับผิดชอบ

> รายละเอียดของการมอบหมาย (เจ้าของ ผู้สนับสนุน ประเภทตัวแทนงานและหน่วยปฏิบัติในอนาคต และระดับการอนุมัติ) จะกำหนดเป็นเมทริกซ์เต็มใน **PKA-OPS-005 — Capability Assignment Matrix**

## 8. การสืบย้อนกลับสู่ OPS-003 (Traceability)

ตารางสืบย้อนนี้ยืนยันว่า **ทุกขีดความสามารถใน OPS-003 มีสำนักงานเจ้าของในสารบบนี้** และในทางกลับกัน ทุกสำนักงานในสารบบนี้สืบย้อนกลับไปยังขีดความสามารถใน OPS-003 ได้

| ขีดความสามารถ (OPS-003 §15) | สำนักงานเจ้าของ (OPS-004 §6) | สำนักงานผู้สนับสนุน |
|---|---|---|
| Knowledge Acquisition | สำนักแหล่งข้อมูลต้นทาง (6.1) | สำนักคลังคัมภีร์ |
| Knowledge Processing | สำนักองค์ความรู้ (6.4) | สำนักคลังคัมภีร์, สำนักคลังศัพท์, สำนักวิจัย |
| Knowledge Search | สำนักองค์ความรู้ (6.4) | สำนักคลังศัพท์ |
| Knowledge Validation | สำนักการอ้างอิง (6.6) | สำนักองค์ความรู้, สำนักคุณภาพ |
| Knowledge Relationship | สำนักความสัมพันธ์ (6.5) | สำนักองค์ความรู้, สำนักวิจัย, สำนักสถาปัตยกรรม |
| Knowledge Publishing | สำนักเผยแพร่ (6.9) | — |
| Knowledge Analytics | สำนักวิจัย (6.8) | สำนักความสัมพันธ์ |
| Knowledge Operations | สำนักปฏิบัติการ (6.11) | สำนักสถาปัตยกรรม |
| AI Assistance | สำนักปฏิบัติการ (6.11) | — |
| Quality Management | สำนักคุณภาพ (6.7) | สำนักการอ้างอิง, สำนักปฏิบัติการ |
| Knowledge Governance | สำนักคุณภาพ (6.7) | สำนักแหล่งข้อมูลต้นทาง, สำนักการอ้างอิง, สำนักเผยแพร่, สำนักสถาปัตยกรรม, สำนักปฏิบัติการ |
| Human Collaboration | สำนักคุณภาพ (6.7) | — |

> **ผลการสืบย้อน:** ขีดความสามารถระดับบนทั้ง 12 รายการมีเจ้าของครบถ้วน และสำนักงานทั้ง 11 แห่งเป็นเจ้าของขีดความสามารถระดับบนหรือขีดความสามารถย่อยอย่างน้อยหนึ่งรายการ — **ไม่มีขีดความสามารถใดไร้เจ้าภาพ และไม่มีสำนักงานใดไร้ความรับผิดชอบ**

## 9. ช่องว่างและสำนักงานในอนาคต (Gaps & Future Offices)

สารบบฉบับนี้บันทึกข้อสังเกตเชิงโครงสร้างที่ควรพิจารณาต่อ โดยยังไม่ตัดสินใจในฉบับนี้

1. **ขีดความสามารถ Knowledge Search ยังไม่มีสำนักงานเฉพาะ** — ปัจจุบันสำนักองค์ความรู้เป็นเจ้าของทั้งการประมวลและการค้นหา หากปริมาณงานด้านการค้นหาขยายตัว องค์กรอาจพิจารณาจัดตั้ง **สำนักการค้นหา (Search Office)** ในอนาคต ตามหลักการจัดตั้งสำนักงานใหม่ใน [OPS-002 หัวข้อ 17](PKA-OPS-002-โครงสร้างองค์กรและสายการบังคับบัญชา.md#17-การจัดตั้งสำนักงานใหม่-establishing-a-new-office)
2. **ขีดความสามารถด้านธรรมาภิบาลกระจุกที่สำนักคุณภาพ** — สำนักคุณภาพเป็นเจ้าของขีดความสามารถด้านธรรมาภิบาลและคุณภาพหลายรายการ หากภาระมากเกินไป องค์กรอาจแยก **สำนักธรรมาภิบาล (Governance Office)** ออกมาในอนาคต
3. **การเติบโตต้องไม่เปลี่ยนโครงสร้าง** — การจัดตั้งสำนักงานใหม่ใด ๆ ต้องเป็นการ "เติม" ไม่ใช่การ "รื้อ" ตามหลักการเติบโตใน [OPS-002 หัวข้อ 20](PKA-OPS-002-โครงสร้างองค์กรและสายการบังคับบัญชา.md#20-การเติบโตขององค์กร-organization-growth)

> การตัดสินใจจัดตั้งสำนักงานใหม่เป็นเรื่องระดับองค์กรที่ต้องผ่านการอนุมัติของมนุษย์ และจะสะท้อนในสารบบฉบับปรับปรุงเมื่อเกิดขึ้นจริง

## 10. ความสัมพันธ์กับเอกสารอ้างอิง

เอกสารฉบับนี้ต่อยอดจากเอกสารปฏิบัติการก่อนหน้าโดยตรง

- **PKA-OPS-001** จัดตั้งสำนักงานและวางหลักการระดับองค์กร — *องค์กรนี้คืออะไร*
- **PKA-OPS-002** กำหนดโครงสร้างองค์กรและสายการบังคับบัญชา — *องค์กรนี้มีรูปร่างอย่างไร*
- **PKA-OPS-003** กำหนดแบบจำลองขีดความสามารถ — *องค์กรทำอะไรได้*
- **PKA-OPS-004** (ฉบับนี้) จับคู่ขีดความสามารถกับสำนักงานเจ้าของ — *สำนักงานใดเป็นเจ้าของความสามารถใด*
- **PKA-OPS-005** (จะจัดทำต่อไป) จะลงรายละเอียดความรับผิดชอบเป็นเมทริกซ์ — *ใครรับผิดชอบ สนับสนุน ทบทวน และอนุมัติในแต่ละด้าน*

เอกสารฉบับนี้อ้างอิงและต้องสอดคล้องกับ

- **[PKA-STD-001 — ธรรมนูญโครงการ](../standards/PKA-STD-001-ธรรมนูญโครงการ.md)** (เอกสารแม่) — หลักการสูงสุด รวมถึงการให้มนุษย์รับผิดชอบสูงสุด
- **[PKA-OPS-001 — ธรรมนูญสำนักงานดิจิทัล](PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md)** — รายชื่อสำนักงานและหลักการประสานงาน
- **[PKA-OPS-002 — โครงสร้างองค์กรและสายการบังคับบัญชา](PKA-OPS-002-โครงสร้างองค์กรและสายการบังคับบัญชา.md)** — โครงสร้างองค์กรที่สำนักงานเหล่านี้สังกัด
- **[PKA-OPS-003 — แบบจำลองขีดความสามารถขององค์กร](PKA-OPS-003-แบบจำลองขีดความสามารถขององค์กร.md)** — ขีดความสามารถที่สารบบนี้จับคู่กับสำนักงาน

> **ลำดับชั้นยังคงเดิม:** เมื่อใดที่เนื้อหาของ OPS-004 ขัดกับเอกสารระดับสูงกว่า ให้ยึดเอกสารระดับสูงกว่าเสมอ

## 11. Roadmap

ลำดับการพัฒนาเอกสารในตระกูลปฏิบัติการ

```
   PKA-OPS-001 : ธรรมนูญสำนักงานดิจิทัล              ✔ จัดทำแล้ว
        ▼
   PKA-OPS-002 : โครงสร้างองค์กรและสายการบังคับบัญชา    ✔ จัดทำแล้ว
        ▼
   PKA-OPS-003 : แบบจำลองขีดความสามารถขององค์กร        ✔ จัดทำแล้ว
        ▼
   PKA-OPS-004 : สารบบสำนักงาน                        ◀ เอกสารฉบับนี้
        ▼
   PKA-OPS-005 : เมทริกซ์การมอบหมายขีดความสามารถ      ○ ขั้นถัดไป
        ▼
   PKA-OPS-005A: สารบบหน้าที่ทางธุรกิจ                ○ อนาคต
        ▼
   PKA-OPS-006 : สถาปัตยกรรมกระแสงาน                  ○ อนาคต
        ▼
   PKA-OPS-007 : สถาปัตยกรรมเหตุการณ์                  ○ อนาคต
        ▼
   PKA-OPS-008 : สถาปัตยกรรมตัวแทนงาน                  ○ อนาคต
        ▼
   PKA-OPS-009 : สถาปัตยกรรมหน่วยปฏิบัติ                ○ อนาคต
```

| รหัส | ชื่อ | จุดมุ่งหมาย |
|---|---|---|
| **OPS-004** | Office Catalog (สารบบสำนักงาน) | จับคู่ขีดความสามารถกับสำนักงานเจ้าของ (ฉบับนี้) |
| **OPS-005** | Capability Assignment Matrix (เมทริกซ์การมอบหมายขีดความสามารถ) | สะพานเชื่อมขีดความสามารถสู่การมอบหมายความรับผิดชอบและการอนุมัติ |
| **OPS-005A** | Business Function Catalog (สารบบหน้าที่ทางธุรกิจ) | นิยามหน้าที่ถาวรขององค์กร เชื่อมขีดความสามารถกับกระแสงาน |
| **OPS-006** | Workflow Architecture (สถาปัตยกรรมกระแสงาน) | กำหนดกระแสงานที่ทำหน้าที่ทางธุรกิจ |
| **OPS-007** | Event Architecture (สถาปัตยกรรมเหตุการณ์) | กำหนดเหตุการณ์ที่ใช้ประสานงานระหว่างสำนักงาน |
| **OPS-008** | Agent Architecture (สถาปัตยกรรมตัวแทนงาน) | กำหนดตัวแทนงานและภารกิจภายในสำนักงาน |
| **OPS-009** | Worker Architecture (สถาปัตยกรรมหน่วยปฏิบัติ) | กำหนดหน่วยปฏิบัติและงานหนึ่งหน่วย |
