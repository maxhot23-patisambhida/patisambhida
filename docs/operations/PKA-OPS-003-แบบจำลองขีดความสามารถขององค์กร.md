## 1. วัตถุประสงค์ (Purpose)

เอกสารฉบับนี้กำหนด **แบบจำลองขีดความสามารถขององค์กร (Enterprise Capability Model)** ของสำนักงานองค์ความรู้ดิจิทัล โดยอธิบายว่าองค์กร PKA **สามารถทำอะไรได้** ในระดับแนวคิดที่เป็นอิสระจากผู้ทำและวิธีทำ

เอกสารนี้มีไว้เพื่อ

1. นิยาม **ขีดความสามารถ (Capability)** และแยกให้ชัดจากแนวคิดที่มักสับสนกัน เช่น พันธกิจ หน้าที่ กระแสงาน งาน และกิจกรรม
2. วางหลักว่า **ขีดความสามารถมาก่อนสำนักงาน** — องค์กรต้องรู้ว่าตน "ทำอะไรได้" ก่อนจะจัดว่า "ใครรับผิดชอบ"
3. กำหนด **ลำดับชั้น การเป็นเจ้าของ วงจรชีวิต ระดับวุฒิภาวะ และการจำแนกประเภท** ของขีดความสามารถ
4. วาง **แผนที่ขีดความสามารถระดับสูง (Capability Map)** ของทั้งองค์กร
5. เชื่อมขีดความสามารถเข้ากับโครงสร้างองค์กรที่กำหนดไว้ใน [PKA-OPS-002](PKA-OPS-002-โครงสร้างองค์กรและสายการบังคับบัญชา.md)

> เมื่ออ่านเอกสารฉบับนี้จบ ผู้อ่านควรเข้าใจว่า **องค์กร PKA มีขีดความสามารถใดบ้าง ขีดความสามารถเหล่านั้นจัดลำดับและกำกับอย่างไร** โดยไม่จำเป็นต้องรู้ว่าสำนักงานใดทำ หรือทำด้วยวิธีใด

## 2. ขอบเขต (Scope)

**อยู่ในขอบเขต**

- นิยามและคุณสมบัติของขีดความสามารถในระดับองค์กร
- ลำดับชั้น การเป็นเจ้าของ ความเป็นอิสระ วงจรชีวิต ระดับวุฒิภาวะ และการจำแนกประเภทของขีดความสามารถ
- การพึ่งพาระหว่างขีดความสามารถ ธรรมาภิบาล และการจัดการการเปลี่ยนแปลง
- หลักการออกแบบขีดความสามารถ และแผนที่ขีดความสามารถระดับสูง

**อยู่นอกขอบเขต**

- รายการสำนักงานและหน้าที่โดยละเอียด (อยู่ใน **PKA-OPS-004 — Office Catalog**)
- หน้าที่ทางธุรกิจ กระแสงาน เหตุการณ์ ตัวแทนงาน และหน่วยปฏิบัติโดยละเอียด (หน้าที่ทางธุรกิจอยู่ใน OPS-005A; กระแสงานถึงหน่วยปฏิบัติอยู่ใน OPS-006 ถึง OPS-009)
- วิธีการทำงาน ขั้นตอนปฏิบัติ หรือการวัดผลเชิงตัวเลขที่เจาะจง
- เครื่องมือ เทคโนโลยี รูปแบบข้อมูล หรือรายละเอียดเชิงปฏิบัติเชิงเทคนิคใด ๆ

> เอกสารฉบับนี้ตอบเฉพาะคำถามว่า **"องค์กรทำอะไรได้"** ส่วนคำถามว่า **"ใครทำ"** เป็นของ OPS-002 และ OPS-004 และคำถามว่า **"ทำอย่างไร"** เป็นของ OPS-006 เป็นต้นไป

## 3. ขีดความสามารถคืออะไร (What is a Capability)

**ขีดความสามารถ (Capability)** คือ *สิ่งที่องค์กรสามารถกระทำได้* อย่างมั่นคงและทำซ้ำได้ โดยไม่ขึ้นกับว่าใครเป็นผู้กระทำหรือกระทำด้วยวิธีใด ขีดความสามารถตอบคำถามว่า **"องค์กรทำอะไรได้"** เพียงเท่านั้น

ความสำคัญอยู่ที่การแยกขีดความสามารถออกจากแนวคิดอื่นที่มักปะปนกัน

| แนวคิด | ตอบคำถาม | ลักษณะ |
|---|---|---|
| **ขีดความสามารถ (Capability)** | องค์กร *ทำอะไรได้* | คงทน เป็นอิสระจากผู้ทำและวิธีทำ |
| **พันธกิจ (Mission)** | องค์กร *มุ่งทำเพื่ออะไร* | เจตจำนงและความมุ่งหมาย |
| **หน้าที่ (Function)** | หน่วยงานหนึ่ง *รับผิดชอบอะไร* | ผูกกับสำนักงานหรือหน่วยงาน |
| **กระแสงาน (Workflow)** | งานหนึ่ง *ไหลอย่างไร* | ลำดับขั้นของการทำงาน |
| **งาน (Task)** | *ต้องทำสิ่งใดให้เสร็จ* | หน่วยงานที่มอบหมายได้ |
| **กิจกรรม (Activity)** | *กำลังลงมือทำสิ่งใด* | การกระทำที่เกิดขึ้นจริง |

ความต่างที่เป็นหัวใจ

- **ขีดความสามารถอยู่เหนือและคงทนกว่า** พันธกิจบอกว่าองค์กรมุ่งไปทางใด แต่ขีดความสามารถบอกว่าองค์กรมีกำลังทำสิ่งใดได้จริง
- **ขีดความสามารถเป็นอิสระจากหน้าที่** หน้าที่ผูกกับสำนักงาน แต่ขีดความสามารถดำรงอยู่แม้สำนักงานจะถูกจัดใหม่
- **ขีดความสามารถไม่ใช่กระแสงาน** กระแสงานเป็น *วิธี* ที่ขีดความสามารถถูกใช้ ขีดความสามารถหนึ่งอาจถูกใช้ผ่านกระแสงานได้หลายแบบ
- **ขีดความสามารถไม่ใช่งานหรือกิจกรรม** งานและกิจกรรมเป็นการลงมือที่เกิดขึ้นชั่วคราว ส่วนขีดความสามารถคือศักยภาพที่คงอยู่

> เปรียบได้ว่า ขีดความสามารถคือ *"สิ่งที่องค์กรเก่ง"* ส่วนสำนักงาน กระแสงาน ตัวแทนงาน และหน่วยปฏิบัติ คือ *"วิธีที่องค์กรนำความเก่งนั้นไปใช้"*

## 4. เหตุใดองค์กรต้องมีขีดความสามารถก่อนสำนักงาน (Capability Before Office)

องค์กรนี้ออกแบบโดยยึดหลัก **"ขีดความสามารถมาก่อนโครงสร้าง"** ด้วยเหตุผลดังนี้

1. **ขีดความสามารถคงทนกว่าโครงสร้าง** — การจัดสำนักงานอาจเปลี่ยนได้ตามเวลา แต่สิ่งที่องค์กรต้องทำได้ยังคงเดิม การเริ่มจากขีดความสามารถจึงให้รากฐานที่มั่นคงกว่า
2. **ป้องกันการออกแบบที่ผูกกับผู้ทำ** — หากเริ่มจากสำนักงานก่อน องค์กรจะถูกออกแบบรอบ "ผู้ทำ" แทนที่จะรอบ "สิ่งที่ต้องทำได้" ซึ่งทำให้เปราะบางเมื่อผู้ทำเปลี่ยน
3. **ทำให้มอบหมายความเป็นเจ้าของได้ชัด** — เมื่อรู้ขีดความสามารถทั้งหมดก่อน องค์กรจึงมอบหมายให้สำนักงานรับผิดชอบได้อย่างครบถ้วน ไม่มีขีดความสามารถใดไร้เจ้าภาพ
4. **รองรับการเติบโต** — เมื่อองค์กรขยาย ขีดความสามารถเป็นกรอบที่บอกว่าควรเพิ่มสำนักงานหรือตัวแทนงานใดเพื่อรองรับสิ่งที่องค์กรต้องทำได้

> หลักนี้สอดคล้องกับ [PKA-OPS-002](PKA-OPS-002-โครงสร้างองค์กรและสายการบังคับบัญชา.md) ที่ว่า **โครงสร้างองค์กรเป็นสิ่งถาวร** — และขีดความสามารถคือสิ่งที่กำหนดว่าโครงสร้างถาวรนั้นต้องรองรับอะไรบ้าง

## 5. ลำดับชั้นของขีดความสามารถ (Capability Hierarchy)

ขีดความสามารถจัดเป็นลำดับชั้น โดยขีดความสามารถระดับสูงแตกย่อยลงไปจนเชื่อมกับการลงมือทำจริง

```
            ┌─────────────────────────────────────┐
            │           Capability                │
            │        (ขีดความสามารถ)               │  ← องค์กรทำอะไรได้
            └─────────────────────────────────────┘
                              │
                              ▼
            ┌─────────────────────────────────────┐
            │         Sub Capability              │
            │      (ขีดความสามารถย่อย)             │  ← แตกความสามารถเป็นด้านย่อย
            └─────────────────────────────────────┘
                              │
                              ▼
            ┌─────────────────────────────────────┐
            │        Business Function            │
            │        (หน้าที่ทางงาน)               │  ← หน้าที่ที่รองรับความสามารถ
            └─────────────────────────────────────┘
                              │
                              ▼
            ┌─────────────────────────────────────┐
            │            Workflow                 │
            │           (กระแสงาน)                │  ← วิธีที่ความสามารถถูกใช้
            └─────────────────────────────────────┘
                              │
                              ▼
            ┌─────────────────────────────────────┐
            │             Agent                   │
            │           (ตัวแทนงาน)               │  ← ผู้รับผิดชอบภารกิจ
            └─────────────────────────────────────┘
                              │
                              ▼
            ┌─────────────────────────────────────┐
            │             Worker                  │
            │          (หน่วยปฏิบัติ)             │  ← ผู้ลงมือทำงานหนึ่งหน่วย
            └─────────────────────────────────────┘
```

| ระดับ | องค์ประกอบ | ตอบคำถาม |
|---|---|---|
| 1 | **ขีดความสามารถ** (Capability) | องค์กรทำอะไรได้ในภาพรวม |
| 2 | **ขีดความสามารถย่อย** (Sub Capability) | ความสามารถนั้นประกอบด้วยด้านย่อยใด |
| 3 | **หน้าที่ทางธุรกิจ** (Business Function) | มีหน้าที่ถาวรใดที่รองรับความสามารถนั้น |
| 4 | **กระแสงาน** (Workflow) | หน้าที่นั้นถูกทำผ่านลำดับงานใด |
| 5 | **ตัวแทนงาน** (Agent) | ใครรับผิดชอบภารกิจในกระแสงาน |
| 6 | **หน่วยปฏิบัติ** (Worker) | ใครลงมือทำงานหนึ่งหน่วย |

> สามระดับบน (ขีดความสามารถ → ขีดความสามารถย่อย → หน้าที่ทางธุรกิจ) เป็น **เชิงแนวคิด** ตอบว่า *องค์กรทำอะไรได้* ส่วนสามระดับล่าง (กระแสงาน → ตัวแทนงาน → หน่วยปฏิบัติ) เป็น **เชิงปฏิบัติ** ตอบว่า *ทำอย่างไรและใครทำ* เอกสารฉบับนี้รับผิดชอบเฉพาะสามระดับบน
>
> **หมายเหตุ:** ชั้น **หน้าที่ทางธุรกิจ (Business Function)** ถูกกำหนดรายละเอียดไว้ใน **[PKA-OPS-005A — สารบบหน้าที่ทางธุรกิจ](PKA-OPS-005A-สารบบหน้าที่ทางธุรกิจ.md)** เพื่อรักษาการแยกหน้าที่ของเอกสาร — เอกสารฉบับนี้นิยามเพียงว่าหน้าที่ทางธุรกิจเป็นชั้นหนึ่งในลำดับชั้น ส่วนสารบบและหลักการของหน้าที่ทางธุรกิจอยู่ใน OPS-005A

## 6. ความเป็นเจ้าของขีดความสามารถ (Capability Ownership)

ทุกขีดความสามารถต้องมีเจ้าของที่ชัดเจน โดยยึดหลัก

1. **หนึ่งขีดความสามารถ มีสำนักงานเจ้าของเพียงหนึ่งเดียว** — ไม่มีขีดความสามารถใดมีสำนักงานเจ้าภาพซ้ำซ้อนจนปัดความรับผิดชอบกันได้
2. **หนึ่งสำนักงาน ดูแลได้หลายขีดความสามารถ** — สำนักงานหนึ่งเป็นเจ้าของได้หลายขีดความสามารถที่อยู่ในความรับผิดชอบของตน
3. **ความเป็นเจ้าของคือความรับผิดชอบ ไม่ใช่กรรมสิทธิ์** — สำนักงานเจ้าของรับผิดชอบให้ขีดความสามารถนั้นทำงานได้และมีคุณภาพ มิใช่ครอบครองไว้ผู้เดียว

```
   Office A ──┬── Capability 1
              ├── Capability 2
              └── Capability 3

   Office B ──┬── Capability 4
              └── Capability 5

   (หนึ่งสำนักงานเป็นเจ้าของได้หลายขีดความสามารถ —
    แต่ขีดความสามารถหนึ่งมีสำนักงานเจ้าของเพียงหนึ่ง)
```

> การจับคู่ขีดความสามารถกับสำนักงานเจ้าของโดยละเอียดจะปรากฏใน **PKA-OPS-004 — Office Catalog** เอกสารฉบับนี้กำหนดเพียง *หลักการ* ของความเป็นเจ้าของ

## 7. ขีดความสามารถไม่ผูกกับสิ่งใด (Capability Independence)

ขีดความสามารถเป็นแนวคิดที่ **เป็นอิสระ** จากผู้ทำและวิธีทำโดยสิ้นเชิง

- **ไม่ผูกกับบุคคล** — ขีดความสามารถคงอยู่แม้บุคคลผู้ปฏิบัติจะเปลี่ยน
- **ไม่ผูกกับผู้ช่วยอัจฉริยะ (AI)** — ขีดความสามารถไม่ขึ้นกับผู้ช่วยอัจฉริยะรุ่นใด เปลี่ยนผู้ช่วยอัจฉริยะได้โดยขีดความสามารถไม่เปลี่ยน
- **ไม่ผูกกับตัวแทนงาน (Agent)** — ตัวแทนงานเป็นผู้รับภารกิจในการใช้ขีดความสามารถ แต่ไม่ใช่ตัวขีดความสามารถ
- **ไม่ผูกกับกระแสงาน (Workflow)** — กระแสงานเป็นวิธีใช้ขีดความสามารถ ขีดความสามารถเดียวอาจถูกใช้ผ่านกระแสงานหลายแบบ

> ความเป็นอิสระนี้คือสิ่งที่ทำให้ขีดความสามารถ **คงทน** สอดคล้องกับหลักการ "องค์กรถาวร กำลังพลทดแทนได้" ใน [PKA-OPS-002 หัวข้อ 21](PKA-OPS-002-โครงสร้างองค์กรและสายการบังคับบัญชา.md#21-องค์กรถาวร-กำลังพลทดแทนได้-stable-organization--replaceable-workforce) — เปลี่ยนผู้ทำและวิธีทำได้ แต่สิ่งที่องค์กรทำได้ยังคงเดิม

## 8. วงจรชีวิตของขีดความสามารถ (Capability Lifecycle)

ขีดความสามารถแต่ละรายการมีวงจรชีวิตที่ชัดเจน ตั้งแต่ถูกเสนอจนถูกยกเลิก

```
   Proposal      ข้อเสนอ        — เสนอให้องค์กรพิจารณาว่าควรมีความสามารถนี้
       ▼
   Approved      อนุมัติแล้ว     — ผ่านการอนุมัติให้เป็นขีดความสามารถขององค์กร
       ▼
   Operational   ใช้งานจริง      — ทำงานได้จริงและมีสำนักงานเจ้าของดูแล
       ▼
   Deprecated    เลิกสนับสนุน    — ยังคงอยู่แต่กำลังถูกทยอยเลิก
       ▼
   Retired       ปลดระวาง        — ยุติการใช้งาน คงบันทึกไว้เพื่อการอ้างอิง
```

| สถานะ | ความหมาย |
|---|---|
| **Proposal (ข้อเสนอ)** | มีการเสนอว่าองค์กรควรมีขีดความสามารถนี้ ยังไม่ผ่านการอนุมัติ |
| **Approved (อนุมัติแล้ว)** | ได้รับอนุมัติให้เป็นขีดความสามารถขององค์กร และมอบหมายเจ้าของแล้ว |
| **Operational (ใช้งานจริง)** | ทำงานได้จริง มีสำนักงานเจ้าของดูแล และอยู่ในการกำกับตามปกติ |
| **Deprecated (เลิกสนับสนุน)** | กำลังถูกทยอยเลิก ยังใช้ได้แต่ไม่ส่งเสริมให้พึ่งพาต่อ |
| **Retired (ปลดระวาง)** | ยุติการใช้งานแล้ว คงบันทึกไว้เพื่อการสืบย้อนและการอ้างอิง |

> การเปลี่ยนสถานะในวงจรชีวิตทุกครั้งต้องมีร่องรอยและการอนุมัติของมนุษย์ตามหลักธรรมาภิบาลในหัวข้อ [12](#12-ธรรมาภิบาลขีดความสามารถ-capability-governance)

## 9. ระดับวุฒิภาวะของขีดความสามารถ (Capability Maturity)

ขีดความสามารถแต่ละรายการมีระดับวุฒิภาวะตั้งแต่ระดับ 1 ถึง 5 ซึ่งสะท้อนความมั่นคงและคุณภาพของความสามารถนั้น

| ระดับ | ชื่อ | ลักษณะ |
|---|---|---|
| **1** | เริ่มต้น (Initial) | มีความสามารถนี้อยู่บ้าง แต่ยังไม่เป็นระบบ ผลลัพธ์ไม่สม่ำเสมอ |
| **2** | กำหนดรูปแล้ว (Defined) | มีการกำหนดขอบเขตและความรับผิดชอบชัดเจน แต่ยังพึ่งการกำกับใกล้ชิด |
| **3** | มั่นคง (Established) | ทำงานได้สม่ำเสมอ มีเจ้าของและการกำกับที่ชัด ผลลัพธ์เชื่อถือได้ |
| **4** | วัดและกำกับได้ (Managed) | มีการติดตามและประเมินอย่างเป็นระบบ สามารถปรับปรุงบนฐานของหลักฐาน |
| **5** | พัฒนาต่อเนื่อง (Optimizing) | มีการปรับปรุงอย่างต่อเนื่อง รักษาคุณภาพสูงและปรับตัวต่อความเปลี่ยนแปลงได้ |

หลักการของระดับวุฒิภาวะ

- ระดับวุฒิภาวะเป็นเครื่องมือ **ประเมินและพัฒนา** ไม่ใช่เครื่องมือตัดสินคุณค่าของสำนักงาน
- ขีดความสามารถที่กระทบความถูกต้องของพระธรรมโดยตรงควรได้รับการยกระดับวุฒิภาวะเป็นลำดับต้น
- การยกระดับวุฒิภาวะต้องไม่แลกมาด้วยการลดทอนความถูกต้อง ตามหลักการในเอกสารแม่

## 10. การจำแนกประเภทขีดความสามารถ (Capability Classification)

ขีดความสามารถจำแนกได้เป็นสี่ประเภทตามบทบาทที่มีต่อองค์กร

| ประเภท | ความหมาย | ตัวอย่างบทบาท |
|---|---|---|
| **ขีดความสามารถหลัก** (Core Capability) | ความสามารถที่เป็นแก่นของพันธกิจองค์กร | การรวบรวม ประมวล ตรวจสอบ และเผยแพร่องค์ความรู้ |
| **ขีดความสามารถสนับสนุน** (Supporting Capability) | ความสามารถที่ช่วยให้ความสามารถหลักทำงานได้ | การจัดการคุณภาพ การวิเคราะห์ การประสานงาน |
| **ขีดความสามารถด้านธรรมาภิบาล** (Governance Capability) | ความสามารถในการกำกับ ควบคุม และตรวจสอบองค์กร | การกำกับองค์ความรู้ การอนุมัติของมนุษย์ การตรวจสอบ |
| **ขีดความสามารถด้านนวัตกรรม** (Innovation Capability) | ความสามารถในการพัฒนาและปรับปรุงองค์กรในระยะยาว | การค้นคว้า การทดลองแนวทางใหม่ การพัฒนาต่อเนื่อง |

หลักการของการจำแนกประเภท

- ขีดความสามารถหนึ่งจัดอยู่ใน **ประเภทหลักเพียงประเภทเดียว** เพื่อความชัดเจนของบทบาท
- **ขีดความสามารถด้านธรรมาภิบาลต้องมีเสมอ** และไม่อาจถูกลดทอน เพราะเป็นหลักประกันความถูกต้อง
- ขีดความสามารถด้านนวัตกรรมต้องไม่ล่วงล้ำหลักการของเอกสารแม่ — นวัตกรรมต้องไม่สร้างหลักธรรมใหม่

## 11. การพึ่งพาระหว่างขีดความสามารถ (Capability Dependency)

ขีดความสามารถบางรายการต้องพึ่งพาขีดความสามารถอื่นจึงจะทำงานได้สมบูรณ์ ความสัมพันธ์เชิงพึ่งพานี้ต้องชัดเจนและตรวจสอบได้

```
   Knowledge Acquisition  (การรับองค์ความรู้)
            │
            ▼
   Knowledge Processing   (การประมวลองค์ความรู้)
            │
            ▼
   Knowledge Validation   (การตรวจสอบองค์ความรู้)
            │
            ├────────────► Knowledge Relationship (การเชื่อมโยงองค์ความรู้)
            │
            ▼
   Knowledge Publishing   (การเผยแพร่องค์ความรู้)

   ── ขีดความสามารถที่กำกับทุกขั้น ──
   Knowledge Governance · Quality Management · Human Collaboration
```

หลักการของการพึ่งพา

1. **การพึ่งพาต้องชัดเจน** — ทุกความสัมพันธ์เชิงพึ่งพาต้องระบุได้ว่าขีดความสามารถใดพึ่งขีดความสามารถใด
2. **ห้ามพึ่งพาเป็นวงวน** — ขีดความสามารถต้องไม่พึ่งพากันเป็นวงจรปิดที่ไม่มีต้นทาง
3. **ขีดความสามารถด้านธรรมาภิบาลและคุณภาพกำกับทุกขั้น** — มิใช่อยู่ปลายสาย แต่กำกับตลอดสายของการพึ่งพา
4. **การพึ่งพาต้องไม่ทำลายความเป็นเจ้าของ** — แม้พึ่งพากัน แต่ละขีดความสามารถยังมีสำนักงานเจ้าของของตน

## 12. ธรรมาภิบาลขีดความสามารถ (Capability Governance)

ขีดความสามารถทุกรายการอยู่ภายใต้ธรรมาภิบาลที่กำหนดผู้รับผิดชอบในแต่ละบทบาท

| บทบาท | ผู้รับผิดชอบ | หน้าที่ |
|---|---|---|
| **ผู้อนุมัติ** (Approver) | มนุษย์ (ผู้อำนวยการสูงสุดหรือผู้ที่ได้รับมอบหมาย) | อนุมัติการมีอยู่ การเปลี่ยนสถานะ และการเปลี่ยนแปลงของขีดความสามารถ |
| **เจ้าของ** (Owner) | สำนักงานเจ้าของ | รับผิดชอบให้ขีดความสามารถทำงานได้และมีคุณภาพ |
| **ผู้ทบทวน** (Reviewer) | สำนักคุณภาพและผู้ทบทวนที่เป็นมนุษย์ | ทบทวนความถูกต้อง คุณภาพ และความสอดคล้องกับหลักการ |

หลักการของธรรมาภิบาล

1. **การอนุมัติขีดความสามารถเป็นของมนุษย์เสมอ** — ผู้ช่วยอัจฉริยะไม่มีสิทธิ์อนุมัติการมีอยู่หรือการเปลี่ยนแปลงของขีดความสามารถ
2. **ทุกขีดความสามารถต้องมีเจ้าของ** — ไม่มีขีดความสามารถใดที่ไร้สำนักงานเจ้าภาพ
3. **การทบทวนต้องเป็นอิสระ** — ผู้ทบทวนต้องไม่ใช่ผู้เดียวกับผู้ลงมือ เพื่อรักษาความเป็นกลาง
4. **ทุกการกำกับต้องมีร่องรอย** — การอนุมัติ การมอบความเป็นเจ้าของ และการทบทวน ต้องบันทึกและตรวจสอบได้

## 13. การจัดการการเปลี่ยนแปลงขีดความสามารถ (Capability Change Management)

องค์กรเปลี่ยนแปลงขีดความสามารถได้ห้าลักษณะ โดยทุกลักษณะต้องผ่านธรรมาภิบาลในหัวข้อ [12](#12-ธรรมาภิบาลขีดความสามารถ-capability-governance)

| ลักษณะ | ความหมาย | ข้อพึงระวัง |
|---|---|---|
| **เพิ่ม** (Add) | จัดตั้งขีดความสามารถใหม่ | ต้องไม่ซ้ำหรือทับซ้อนกับขีดความสามารถเดิม |
| **แก้ไข** (Modify) | ปรับขอบเขตหรือคุณสมบัติของขีดความสามารถเดิม | ต้องไม่เปลี่ยนแก่นจนกลายเป็นความสามารถอื่น |
| **รวม** (Merge) | รวมขีดความสามารถหลายรายการเป็นหนึ่ง | ต้องรักษาความครบถ้วนของสิ่งที่เคยทำได้ |
| **แยก** (Split) | แยกขีดความสามารถหนึ่งเป็นหลายรายการ | ต้องกำหนดเจ้าของของแต่ละรายการใหม่ให้ชัด |
| **ยกเลิก** (Retire) | ปลดระวางขีดความสามารถ | ต้องตรวจว่าไม่มีขีดความสามารถอื่นพึ่งพาอยู่ |

หลักการของการจัดการการเปลี่ยนแปลง

- ทุกการเปลี่ยนแปลงต้อง **เริ่มจากข้อเสนอ** และผ่านการอนุมัติของมนุษย์
- การเปลี่ยนแปลงต้อง **ไม่ทำลายการพึ่งพา** ของขีดความสามารถอื่นโดยไม่ได้จัดการล่วงหน้า
- การเปลี่ยนแปลงต้อง **มีร่องรอย** ในประวัติของขีดความสามารถและของเอกสารตระกูล OPS

## 14. หลักการออกแบบขีดความสามารถ (Capability Design Principles)

ขีดความสามารถทุกรายการต้องเป็นไปตามหลักการออกแบบอย่างน้อยสิบห้าประการต่อไปนี้

1. **วัดได้ (Measurable)** — ต้องประเมินได้ว่าขีดความสามารถนั้นทำงานได้ดีเพียงใด
2. **ตรวจสอบได้ (Auditable)** — การทำงานและผลของขีดความสามารถต้องสืบย้อนและตรวจสอบได้
3. **ไม่ซ้ำกัน (Unique)** — แต่ละขีดความสามารถต้องไม่ซ้ำกับขีดความสามารถอื่น
4. **ไม่ทับซ้อน (Non-overlapping)** — ขอบเขตของขีดความสามารถต้องไม่ทับซ้อนกันจนสับสนความรับผิดชอบ
5. **ไม่อิงเทคโนโลยี** — ขีดความสามารถต้องไม่นิยามด้วยเครื่องมือหรือวิธีเชิงเทคนิคใด ๆ
6. **ไม่อิงผู้ช่วยอัจฉริยะ (AI-Independent)** — ขีดความสามารถต้องไม่ขึ้นกับผู้ช่วยอัจฉริยะรุ่นใด
7. **ไม่อิงบุคคล (Person-Independent)** — ขีดความสามารถต้องคงอยู่แม้ผู้ปฏิบัติเปลี่ยน
8. **ขยายได้ (Scalable)** — รองรับปริมาณงานที่เพิ่มขึ้นได้โดยไม่เปลี่ยนนิยาม
9. **เปลี่ยนเจ้าของได้ (Transferable Ownership)** — เปลี่ยนสำนักงานเจ้าของได้ โดยยังเป็นขีดความสามารถเดิม
10. **มีขอบเขตชัด (Well-bounded)** — ต้องระบุได้ชัดว่าสิ่งใดอยู่ในและสิ่งใดอยู่นอกขีดความสามารถ
11. **คงทน (Durable)** — ต้องคงอยู่ข้ามการเปลี่ยนแปลงของโครงสร้างและเครื่องมือ
12. **เป็นอิสระจากกระแสงาน (Workflow-Independent)** — ต้องไม่ผูกกับลำดับงานแบบใดแบบหนึ่ง
13. **สอดคล้องกับเอกสารแม่ (Aligned)** — ต้องไม่ขัดกับหลักการใน PKA-STD-001
14. **มีเจ้าของเดียว (Single Ownership)** — ต้องมีสำนักงานเจ้าของเพียงหนึ่งเดียว
15. **กำกับดูแลได้ (Governable)** — ต้องอยู่ภายใต้ธรรมาภิบาลและการอนุมัติของมนุษย์
16. **อธิบายด้วยภาษาขององค์กร (Business-Articulated)** — นิยามด้วยภาษาของงานองค์ความรู้ ไม่ใช่ภาษาเชิงเทคนิค
17. **เคารพความถูกต้องของพระธรรม (Dhamma-Respecting)** — ต้องไม่มีขีดความสามารถใดที่นำไปสู่การบิดเบือนพระธรรม

> หลักการเหล่านี้คือเกณฑ์ที่ใช้ตัดสินว่าสิ่งหนึ่งควรเป็น "ขีดความสามารถ" หรือไม่ สิ่งที่ไม่ผ่านเกณฑ์เหล่านี้ไม่ควรถูกจัดเป็นขีดความสามารถขององค์กร

## 15. แผนที่ขีดความสามารถ (Capability Map)

แผนที่ขีดความสามารถระดับสูงของสำนักงานองค์ความรู้ดิจิทัล แสดงเฉพาะ **ขีดความสามารถ** โดยยังไม่ลงรายละเอียดว่าสำนักงานใดเป็นเจ้าของ

```
   ┌───────────────────────────── PKA Capability Map ─────────────────────────────┐
   │                                                                              │
   │   ── ขีดความสามารถหลัก (Core) ──                                              │
   │   • Knowledge Acquisition    (การรับองค์ความรู้)                              │
   │   • Knowledge Processing     (การประมวลองค์ความรู้)                           │
   │   • Knowledge Validation     (การตรวจสอบองค์ความรู้)                          │
   │   • Knowledge Relationship   (การเชื่อมโยงองค์ความรู้)                        │
   │   • Knowledge Search         (การค้นหาองค์ความรู้)                            │
   │   • Knowledge Publishing     (การเผยแพร่องค์ความรู้)                          │
   │                                                                              │
   │   ── ขีดความสามารถสนับสนุน (Supporting) ──                                    │
   │   • Knowledge Analytics      (การวิเคราะห์องค์ความรู้)                        │
   │   • Knowledge Operations     (การดำเนินงานองค์ความรู้)                        │
   │   • AI Assistance            (การช่วยเหลือด้วยผู้ช่วยอัจฉริยะ)                 │
   │   • Quality Management        (การจัดการคุณภาพ)                              │
   │                                                                              │
   │   ── ขีดความสามารถด้านธรรมาภิบาล (Governance) ──                              │
   │   • Knowledge Governance     (การกำกับองค์ความรู้)                            │
   │   • Human Collaboration      (การทำงานร่วมกับมนุษย์)                          │
   │                                                                              │
   └──────────────────────────────────────────────────────────────────────────────┘
```

| ขีดความสามารถ | ความหมายโดยสังเขป | ประเภท |
|---|---|---|
| **Knowledge Acquisition** (การรับองค์ความรู้) | รับและขึ้นทะเบียนแหล่งข้อมูลต้นทางและเนื้อหาคัมภีร์ | หลัก |
| **Knowledge Processing** (การประมวลองค์ความรู้) | แปรเนื้อหาให้เป็นองค์ความรู้ที่มีโครงสร้าง | หลัก |
| **Knowledge Validation** (การตรวจสอบองค์ความรู้) | ตรวจความถูกต้องและความครบของการอ้างอิง | หลัก |
| **Knowledge Relationship** (การเชื่อมโยงองค์ความรู้) | สร้างและดูแลความสัมพันธ์ระหว่างองค์ความรู้ | หลัก |
| **Knowledge Search** (การค้นหาองค์ความรู้) | ทำให้เข้าถึงองค์ความรู้ผ่านการค้นหา | หลัก |
| **Knowledge Publishing** (การเผยแพร่องค์ความรู้) | จัดเตรียมและเผยแพร่องค์ความรู้ที่อนุมัติแล้ว | หลัก |
| **Knowledge Analytics** (การวิเคราะห์องค์ความรู้) | วิเคราะห์ภาพรวมและคุณภาพขององค์ความรู้ | สนับสนุน |
| **Knowledge Operations** (การดำเนินงานองค์ความรู้) | ประสานและติดตามการดำเนินงานขององค์กร | สนับสนุน |
| **AI Assistance** (การช่วยเหลือด้วยผู้ช่วยอัจฉริยะ) | ใช้ผู้ช่วยอัจฉริยะค้นหา อธิบาย และเชื่อมโยง ภายใต้การกำกับ | สนับสนุน |
| **Quality Management** (การจัดการคุณภาพ) | กำกับคุณภาพและความถูกต้องทั่วทั้งองค์กร | สนับสนุน |
| **Knowledge Governance** (การกำกับองค์ความรู้) | กำกับความถูกต้องและการอนุมัติองค์ความรู้ | ธรรมาภิบาล |
| **Human Collaboration** (การทำงานร่วมกับมนุษย์) | ธำรงบทบาทมนุษย์ในวงจรการตัดสินใจและการอนุมัติ | ธรรมาภิบาล |

> แผนที่นี้เป็นระดับสูง แสดงเพียง *ขีดความสามารถ* เท่านั้น การจับคู่ขีดความสามารถกับสำนักงานเจ้าของจะปรากฏใน **PKA-OPS-004 — Office Catalog**

## 16. ความสัมพันธ์จากขีดความสามารถสู่หน่วยปฏิบัติ

ขีดความสามารถเชื่อมลงสู่การลงมือทำจริงผ่านโครงสร้างองค์กรที่กำหนดไว้ใน [PKA-OPS-002](PKA-OPS-002-โครงสร้างองค์กรและสายการบังคับบัญชา.md)

```
   Capability   (ขีดความสามารถ)   ← องค์กรทำอะไรได้
        │
        ▼
   Office       (สำนักงาน)        ← ใครเป็นเจ้าของความสามารถ
        │
        ▼
   Workflow     (กระแสงาน)        ← ความสามารถถูกใช้ผ่านลำดับงานใด
        │
        ▼
   Agent        (ตัวแทนงาน)       ← ใครรับผิดชอบภารกิจ
        │
        ▼
   Worker       (หน่วยปฏิบัติ)    ← ใครลงมือทำงานหนึ่งหน่วย
```

- **ขีดความสามารถ** บอกว่าองค์กร *ทำอะไรได้* — เป็นจุดเริ่มของทุกอย่าง
- **สำนักงาน** เป็น *เจ้าของ* ขีดความสามารถ และนำมันไปใช้
- **กระแสงาน** เป็น *วิธี* ที่ขีดความสามารถถูกใช้
- **ตัวแทนงาน** และ **หน่วยปฏิบัติ** เป็น *ผู้ลงมือ* ตามกระแสงานนั้น

> เส้นนี้แสดงว่าขีดความสามารถอยู่ **บนสุด** ของห่วงโซ่ — ทุกสิ่งที่อยู่ใต้ลงไปล้วนเป็นวิธีนำขีดความสามารถไปใช้ การออกแบบจึงต้องเริ่มจากขีดความสามารถเสมอ ตามหลักในหัวข้อ [4](#4-เหตุใดองค์กรต้องมีขีดความสามารถก่อนสำนักงาน-capability-before-office)

## 17. ความสัมพันธ์กับเอกสารอ้างอิง

เอกสารฉบับนี้เป็นส่วนหนึ่งของตระกูลปฏิบัติการ (PKA-OPS) และวางตัวอยู่เหนือเอกสารปฏิบัติการฉบับถัด ๆ ไปในเชิงแนวคิด

- **PKA-OPS-001** จัดตั้งองค์กรและวางหลักการระดับองค์กร — *องค์กรนี้คืออะไร*
- **PKA-OPS-002** กำหนดโครงสร้างองค์กรและสายการบังคับบัญชา — *องค์กรนี้มีรูปร่างอย่างไร ใครขึ้นกับใคร*
- **PKA-OPS-003** (ฉบับนี้) กำหนดแบบจำลองขีดความสามารถ — *องค์กรนี้ทำอะไรได้*
- **PKA-OPS-004 เป็นต้นไป** จะลงรายละเอียดว่า *ใครทำและทำอย่างไร*

เอกสารฉบับนี้อ้างอิงและต้องสอดคล้องกับ

- **[PKA-STD-001 — ธรรมนูญโครงการ](../standards/PKA-STD-001-ธรรมนูญโครงการ.md)** (เอกสารแม่) — หลักการสูงสุด รวมถึงการให้มนุษย์รับผิดชอบสูงสุดและห้ามผู้ช่วยอัจฉริยะสร้างหลักธรรมใหม่
- **[PKA-OPS-001 — ธรรมนูญสำนักงานดิจิทัล](PKA-OPS-001-ธรรมนูญสำนักงานดิจิทัล.md)** — กฎบัตรองค์กรที่กำหนดสำนักงานและหลักการประสานงาน
- **[PKA-OPS-002 — โครงสร้างองค์กรและสายการบังคับบัญชา](PKA-OPS-002-โครงสร้างองค์กรและสายการบังคับบัญชา.md)** — โครงสร้างองค์กรที่ขีดความสามารถเชื่อมลงไปใช้

> **ลำดับชั้นยังคงเดิม:** เมื่อใดที่เนื้อหาของ OPS-003 ขัดกับเอกสารระดับสูงกว่า ให้ยึดเอกสารระดับสูงกว่าเสมอ

## 18. Roadmap

ลำดับการพัฒนาเอกสารในตระกูลปฏิบัติการต่อจากฉบับนี้

```
   PKA-OPS-001 : ธรรมนูญสำนักงานดิจิทัล              ✔ จัดทำแล้ว
        ▼
   PKA-OPS-002 : โครงสร้างองค์กรและสายการบังคับบัญชา    ✔ จัดทำแล้ว
        ▼
   PKA-OPS-003 : แบบจำลองขีดความสามารถขององค์กร        ◀ เอกสารฉบับนี้
        ▼
   PKA-OPS-004 : Office Catalog        (สารบบสำนักงาน)       ○ ขั้นถัดไป
        ▼
   PKA-OPS-005 : Capability Assignment Matrix (เมทริกซ์การมอบหมายขีดความสามารถ) ○ อนาคต
        ▼
   PKA-OPS-005A: Business Function Catalog (สารบบหน้าที่ทางธุรกิจ) ○ อนาคต
        ▼
   PKA-OPS-006 : Workflow Architecture (สถาปัตยกรรมกระแสงาน)   ○ อนาคต
        ▼
   PKA-OPS-007 : Event Architecture    (สถาปัตยกรรมเหตุการณ์)  ○ อนาคต
        ▼
   PKA-OPS-008 : Agent Architecture    (สถาปัตยกรรมตัวแทนงาน)  ○ อนาคต
        ▼
   PKA-OPS-009 : Worker Architecture   (สถาปัตยกรรมหน่วยปฏิบัติ) ○ อนาคต
```

| รหัส | ชื่อ | จุดมุ่งหมาย |
|---|---|---|
| **OPS-004** | Office Catalog (สารบบสำนักงาน) | จับคู่ขีดความสามารถกับสำนักงานเจ้าของ และลงรายละเอียดหน้าที่ของแต่ละสำนักงาน |
| **OPS-005** | Capability Assignment Matrix (เมทริกซ์การมอบหมายขีดความสามารถ) | สะพานเชื่อมขีดความสามารถสู่การมอบหมายความรับผิดชอบและการอนุมัติ |
| **OPS-005A** | Business Function Catalog (สารบบหน้าที่ทางธุรกิจ) | นิยามหน้าที่ถาวรขององค์กร เชื่อมขีดความสามารถกับกระแสงาน |
| **OPS-006** | Workflow Architecture (สถาปัตยกรรมกระแสงาน) | กำหนดกระแสงานที่ทำหน้าที่ทางธุรกิจ |
| **OPS-007** | Event Architecture (สถาปัตยกรรมเหตุการณ์) | กำหนดเหตุการณ์ที่ใช้ประสานงานระหว่างสำนักงาน |
| **OPS-008** | Agent Architecture (สถาปัตยกรรมตัวแทนงาน) | กำหนดตัวแทนงานและภารกิจภายในสำนักงาน |
| **OPS-009** | Worker Architecture (สถาปัตยกรรมหน่วยปฏิบัติ) | กำหนดหน่วยปฏิบัติและงานหนึ่งหน่วย |
