# vision/ — เอกสารวิสัยทัศน์ (PKA-VIS) — Level 0

โฟลเดอร์นี้บรรจุเอกสารตระกูล **วิสัยทัศน์ (`PKA-VIS-*`)** ซึ่งเป็นเอกสาร **ระดับสูงสุด (Level 0)** ของโครงการ — เป็น **ชั้นวิสัยทัศน์ (Vision Layer)** ที่อยู่ **เหนือ** ตระกูลมาตรฐาน (PKA-STD)

## หน้าที่ของตระกูล VIS

- กำหนด **ภาพอนาคตและเจตจำนงสูงสุด** ของทั้งระบบ — *เรากำลังสร้างอะไร และเพื่ออะไร*
- เป็นที่มาของเจตนารมณ์ที่ [PKA-STD-001 — ธรรมนูญโครงการ](../standards/PKA-STD-001-ธรรมนูญโครงการ.md) นำไปกำหนดเป็นหลักการผูกพัน
- **ไม่ใช่มาตรฐาน (STD), ไม่ใช่สถาปัตยกรรม (ARC), ไม่ใช่ปฏิบัติการ (OPS)** และ **ไม่กล่าวถึงเทคโนโลยีหรือรายละเอียดเชิงเทคนิค**

## ตำแหน่งในลำดับชั้น

```
   Level 0  วิสัยทัศน์ (Vision)   ← โฟลเดอร์นี้ (PKA-VIS)
        ▼
   Level 1  มาตรฐาน (Standards)   PKA-STD (รวมธรรมนูญ STD-001)
        ▼
   Level 2  สถาปัตยกรรม (Architecture)
        ▼
   ... Domain · Reference · Operations · Knowledge
```

- ตระกูล VIS เป็น **ชั้นสูงสุด** — กำหนดทิศทางที่ทุกตระกูลด้านล่างต้องไม่ขัดแย้ง
- **PKA-STD-001 ยังคงเป็นเอกสารแม่ (Master Document)** ที่เอกสารเชิงปฏิบัติทุกฉบับอ้างอิง ส่วน VIS เป็นวิสัยทัศน์ที่อยู่เหนือขึ้นไป
