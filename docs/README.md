# PKA Documentation Portal
## สารบบเอกสารโครงการ Patisambhidamagga Knowledge Architecture (PKA)

> เอกสารฉบับนี้เป็น **ประตูทางเข้า (Documentation Portal)** ของเอกสารทั้งหมดในโครงการ
> ใช้เป็นจุดเริ่มต้นในการค้นหา ทำความเข้าใจลำดับชั้น และเพิ่มเอกสารใหม่ให้สอดคล้องกับมาตรฐานเดียวกันทั้งโครงการ

| รายการ | รายละเอียด |
|---|---|
| ชื่อเอกสาร | PKA Documentation Portal |
| ระดับเอกสาร | สารบบ (Index) |
| เอกสารแม่ | [PKA-STD-001 — ธรรมนูญโครงการ](standards/PKA-STD-001-ธรรมนูญโครงการ.md) |
| สถานะ | Draft 1.2 |
| ภาษา | ไทย |

> **หลักสำคัญ:** เอกสารทุกฉบับในโครงการต้องอ้างอิง **PKA-STD-001 (เอกสารแม่ / Master Document)** และต้องไม่ขัดแย้งกับหลักการที่กำหนดไว้ในเอกสารนั้น เอกสารทุกฉบับยังคงอยู่ในระดับวิสัยทัศน์ สถาปัตยกรรม และมาตรฐาน จึง **ไม่กล่าวถึงเทคโนโลยีหรือรายละเอียดการ Implement**

## 1. ภาพรวม

โครงการ PKA มุ่งสร้างสถาปัตยกรรมความรู้สำหรับการศึกษาพระธรรม โดยมีปฏิสัมภิทามรรคและพระไตรปิฎกเป็นรากฐาน เอกสารชุดนี้เป็นโครงสร้างพื้นฐาน (Documentation Foundation) ที่ออกแบบให้รองรับการพัฒนาโครงการในระยะยาว และทำให้เอกสารทุกฉบับในอนาคตมีโครงสร้าง การตั้งชื่อ และลำดับชั้นเดียวกัน

อ่านเอกสารแม่ก่อนเสมอ: **[PKA-STD-001 — ธรรมนูญโครงการ](standards/PKA-STD-001-ธรรมนูญโครงการ.md)** เป็นที่มาของหลักการทั้งหมดที่เอกสารอื่นนำไปขยายความ

## 3. ลำดับชั้นของเอกสาร (Document Hierarchy)

เอกสารในโครงการจัดเป็น **7 ระดับ (Document Level)** โดย **ระดับที่ต่ำกว่าต้องไม่ขัดกับระดับที่สูงกว่าเสมอ** — โดยมี **ชั้นวิสัยทัศน์ (PKA-VIS)** เป็นระดับสูงสุด (Level 0) และ **PKA-STD-001 ยังคงเป็นเอกสารแม่ (Master Document)** ที่เอกสารเชิงปฏิบัติทุกฉบับอ้างอิง

```
   Level 0  ─  วิสัยทัศน์ (Vision)
              PKA-VIS-001 — วิสัยทัศน์สำนักงานใหญ่ดิจิทัล
                        │
                        ▼
   Level 1  ─  มาตรฐาน (Standards)
              PKA-STD-001 (ธรรมนูญ / เอกสารแม่) · PKA-STD-002 …
                        │
                        ▼
   Level 2  ─  สถาปัตยกรรม (Architecture)
              PKA-ARC-*
                        │
                        ▼
   Level 3  ─  แบบจำลองโดเมน (Domain)
              PKA-DOM-*
                        │
                        ▼
   Level 4  ─  แหล่งอ้างอิง (Reference)
              PKA-REF-*
                        │
                        ▼
   ───────  ─  ปฏิบัติการ/องค์กร (Operations)
              PKA-OPS-*  (ชั้นปฏิบัติการ — สำนักงานดิจิทัล)
                        │
                        ▼
   Level 5  ─  องค์ความรู้ (Knowledge)
              (กำหนดในเฟสถัดไป)
                        │
                        ▼
   Level 6  ─  บันทึกการตัดสินใจ (Decision Records)
              ADR-*
```

| Level | ชื่อ | หน้าที่ | ตระกูล / ตัวอย่าง |
|---|---|---|---|
| **0** | วิสัยทัศน์ (Vision) | กำหนดภาพอนาคตและเจตนารมณ์สูงสุดของทั้งระบบ เปลี่ยนได้ยากที่สุด | PKA-VIS-* |
| **1** | มาตรฐาน (Standards) | กำหนด *กฎ* กลางที่ทุกส่วนต้องปฏิบัติตาม (รวมธรรมนูญ STD-001 เอกสารแม่) | PKA-STD-* |
| **2** | สถาปัตยกรรม (Architecture) | อธิบาย *ภาพรวม* ของระบบและการเชื่อมโยง | PKA-ARC-* |
| **3** | แบบจำลองโดเมน (Domain) | กำหนด *แบบจำลอง* ขององค์ความรู้ | PKA-DOM-* |
| **4** | แหล่งอ้างอิง (Reference) | อธิบาย *แหล่งอ้างอิง* และคัมภีร์ต้นทาง | PKA-REF-* |
| **—** | ปฏิบัติการ/องค์กร (Operations) | จัดตั้งองค์กรและการดำเนินงาน (ชั้นปฏิบัติการ ขวางทุกระดับ) | PKA-OPS-* |
| **5** | องค์ความรู้ (Knowledge) | เก็บ *องค์ความรู้จริง* | (กำหนดในเฟสถัดไป) |
| **6** | บันทึกการตัดสินใจ (Decision Records) | บันทึกเหตุผลของการตัดสินใจ (ขวางทุกระดับ) | ADR-* |

> **ชั้นปฏิบัติ:** คู่มือปฏิบัติ (GUIDE) และการนำไปปฏิบัติ (Implementation) เป็นชั้นปฏิบัติที่อยู่ **ใต้** ระดับเอกสารแนวคิดทั้งเจ็ด และต้องเคารพทุกระดับเหนือขึ้นไป

**กฎของลำดับชั้น**
1. **เอกสารระดับล่างต้องไม่ขัดกับระดับบน** — เมื่อเกิดความขัดแย้ง ให้ยึดเอกสารระดับสูงกว่าเสมอ
2. เอกสารเชิงปฏิบัติทุกฉบับต้องระบุ "เอกสารแม่: PKA-STD-001" ในส่วนหัว — ยกเว้นชั้นวิสัยทัศน์ (PKA-VIS) ซึ่งเป็นระดับสูงสุดและเป็นที่มาของเจตนารมณ์ที่ STD-001 นำไปกำหนด
3. การเปลี่ยนแปลงเอกสารระดับสูงอาจกระทบเอกสารระดับล่าง — ต้องทบทวนผลกระทบทุกครั้ง
4. แต่ละตระกูลคงหน้าที่ของตน — **มาตรฐาน (STD) และแบบจำลองโดเมน (DOM) ต้องไม่ถูกย้ายเข้าสถาปัตยกรรม (ARC)**

## 4. มาตรฐานการตั้งชื่อ (Naming Convention)

### 4.1 รหัสเอกสาร

| คำนำหน้า | ประเภท | โฟลเดอร์ | รูปแบบเลข | ตัวอย่าง |
|---|---|---|---|---|
| `PKA-VIS-NNN` | วิสัยทัศน์ (Vision) | `vision/` | 3 หลัก เริ่มที่ `001` | `PKA-VIS-001` |
| `PKA-STD-NNN` | มาตรฐาน (Standards) | `standards/` | 3 หลัก เริ่มที่ `001` | `PKA-STD-001` |
| `PKA-ARC-NNN` | สถาปัตยกรรม (Architecture) | `architecture/` | 3 หลัก เริ่มที่ `001` | `PKA-ARC-001` |
| `PKA-DOM-NNN` | แบบจำลองโดเมน (Domain Model) | `domain/` | 3 หลัก เริ่มที่ `001` | `PKA-DOM-001` |
| `PKA-REF-NNN` | แหล่งอ้างอิง (Reference) | `reference/` | 3 หลัก เริ่มที่ `001` | `PKA-REF-001` |
| `PKA-KNW-NNN` | องค์ความรู้จริง (Knowledge Asset) | `knowledge/` | 4 หลัก เริ่มที่ `0001` | `PKA-KNW-0001` |
| `ADR-NNNN` | บันทึกการตัดสินใจ (Decision Record) | `decisions/` | 4 หลัก เริ่มที่ `0001` | `ADR-0001` |
| `PKA-REV-NNN` | เอกสารประเมิน (Review) | `reviews/` | 3 หลัก เริ่มที่ `001` | `PKA-REV-001` |
| `PKA-OPS-NNN` | ปฏิบัติการ (Operations) | `operations/` | 3 หลัก เริ่มที่ `001` | `PKA-OPS-001` |
| `GUIDE-NNN` | คู่มือปฏิบัติ (Guide) | `development/` | 3 หลัก เริ่มที่ `001` | `GUIDE-001` |

- หมายเลขเรียงตามลำดับการสร้าง **ห้ามนำหมายเลขที่เคยใช้แล้วกลับมาใช้ซ้ำ** แม้เอกสารนั้นจะถูกยกเลิก
- เอกสารที่ถูกยกเลิกให้คงหมายเลขไว้และเปลี่ยนสถานะเป็น *Deprecated* หรือ *Superseded*

### 4.2 ชื่อไฟล์

รูปแบบ: `<รหัสเอกสาร>-<ชื่อย่อภาษาไทย>.md`

ตัวอย่าง: `PKA-STD-002-มาตรฐานข้อมูล.md`

- ใช้ชื่อย่อภาษาไทยที่สื่อความหมายต่อจากรหัส
- หนึ่งไฟล์ต่อหนึ่งเอกสาร
- ไฟล์ดัชนีของแต่ละโฟลเดอร์ใช้ชื่อ `README.md`

### 4.3 สถานะเอกสาร (Status)

| สถานะ | ความหมาย |
|---|---|
| `Outline` | โครงร่าง — กำหนด Purpose/Scope/สารบัญ แต่ยังไม่เขียนรายละเอียดเต็ม |
| `Draft` | ฉบับร่าง — มีเนื้อหาแล้ว อยู่ระหว่างทบทวน |
| `Active` | บังคับใช้ — ผ่านการทบทวนและถือเป็นมาตรฐานปัจจุบัน |
| `Deprecated` | เลิกใช้ — ไม่บังคับใช้แล้ว แต่คงไว้เพื่อการอ้างอิง |
| `Superseded` | ถูกแทนที่ — มีเอกสารใหม่มาแทน (ต้องระบุว่าแทนที่ด้วยฉบับใด) |

## 6. การเพิ่มเอกสารใหม่

1. เลือกประเภทและคำนำหน้าให้ถูกต้องตาม [Naming Convention](#4-มาตรฐานการตั้งชื่อ-naming-convention)
2. คัดลอกเทมเพลตจาก `templates/` ที่ตรงกับประเภทเอกสาร
3. ใส่หมายเลขถัดไปที่ยังไม่เคยใช้ และวางไฟล์ในโฟลเดอร์ที่ถูกต้อง
4. ระบุ **เอกสารแม่: PKA-STD-001** ในส่วนหัวเสมอ
5. ตรวจสอบว่าเนื้อหาไม่ขัดแย้งกับเอกสารระดับสูงกว่า
6. เพิ่มรายการในทะเบียนเอกสารของพอร์ทัลฉบับนี้
7. เริ่มที่สถานะ `Outline` หรือ `Draft` ตามความพร้อม
